# 财务-采购记账模块功能解析文档

## 1. 模块架构

### 1.1 技术栈

- **前端**：Vue 3 + Composition API + Element Plus + Axios
- **后端**：Spring Boot 2.x + MyBatis-Plus + Spring Security
- **数据库**：MySQL
- **API设计**：RESTful API
- **数据传输**：JSON格式

### 1.2 模块结构

#### 前端结构
```
wimoor-ui/src/views/erp/finance/paymentRequest/
├── index.vue          # 请款单主组件
└── components/
    ├── table.vue      # 请款单表格组件
    └── details.vue    # 请款单详情组件
```

#### 后端结构
```
wimoor-erp/erp-boot/src/main/java/com/wimoor/erp/purchase/
├── controller/
│   ├── PurchaseFinanceFormController.java  # 采购财务表单控制器
│   └── PurchaseFormController.java         # 采购单控制器
```

## 3. 后端实现

### 3.1 采购财务表单控制器（PurchaseFinanceFormController.java）

#### 核心功能
- 请款单列表查询
- 请款单详情查询
- 请款单审核
- 请款单驳回
- 付款方式更新
- 备注更新
- 付款处理

#### 关键API接口

**1. 查询请款单列表**
- **接口**：`POST /api/v1/purchase_finance_form/list`
- **功能**：根据条件查询请款单列表，支持分页
- **参数**：`PurchaseFinanceListDTO`（包含供应商ID、支付方式、状态、日期范围等查询条件）
- **返回**：`IPage<Map<String, Object>>`（分页的请款单列表）

**2. 查询请款单详情**
- **接口**：`GET /api/v1/purchase_finance_form/getdetail`
- **功能**：查询请款单的详细信息
- **参数**：`id`（请款单ID）
- **返回**：`Map<String, Object>`（请款单详细信息）

**3. 审核请款单**
- **接口**：`POST /api/v1/purchase_finance_form/approve`
- **功能**：审核通过请款单
- **参数**：`List<String>`（请款单ID列表）
- **返回**：`Map<String, Object>`（操作结果）

**4. 驳回请款单**
- **接口**：`POST /api/v1/purchase_finance_form/approveReturn`
- **功能**：审核驳回请款单
- **参数**：`List<String>`（请款单ID列表）
- **返回**：`Map<String, Object>`（操作结果）

**5. 更新付款方式**
- **接口**：`POST /api/v1/purchase_finance_form/updatePay`
- **功能**：更新请款单的付款方式
- **参数**：`FinanceFormPayMethDTO`（包含付款方式、支付账号和请款单ID列表）
- **返回**：`Map<String, Object>`（操作结果）

**6. 更新备注**
- **接口**：`GET /api/v1/purchase_finance_form/updateRemark`
- **功能**：更新请款单的备注
- **参数**：`id`（请款单ID）、`remark`（备注内容）
- **返回**：`Map<String, Object>`（操作结果）

**7. 获取请款单数据**
- **接口**：`POST /api/v1/purchase_finance_form/getData`
- **功能**：获取请款单的详细数据
- **参数**：`List<String>`（请款单ID列表）
- **返回**：`List<Map<String, Object>>`（请款单详细数据）

**8. 处理付款**
- **接口**：`POST /api/v1/purchase_finance_form/paymentForm`
- **功能**：处理请款单的付款
- **参数**：`List<String>`（请款单ID列表）
- **返回**：`Map<String, Object>`（操作结果）

### 3.2 采购财务表单服务（IPurchaseFinanceFormService）

- `findByCondition()`：根据条件查询请款单列表

## 4. 业务逻辑流程

### 4.1 请款单查询流程

1. 前端设置查询条件（供应商、支付方式、日期范围等）
2. 前端调用`purchaseFinlistApi.list()`方法
3. 后端接收请求，解析参数
4. 后端调用`purchaseFinanceFormService.findByCondition()`查询请款单列表
5. 后端返回分页的请款单数据给前端
6. 前端展示请款单列表

### 4.2 请款单审核流程

1. 前端选择待审核的请款单
2. 前端点击「审核通过」或「审核驳回」按钮
3. 前端调用`purchaseFinlistApi.approve()`或`purchaseFinlistApi.approveReturn()`方法
4. 后端接收请求，解析参数
5. 后端调用`purchaseFinanceFormService.approve()`或`purchaseFinanceFormService.approveReturn()`处理审核
6. 后端返回操作结果给前端
7. 前端显示操作结果提示，刷新请款单列表

### 4.3 请款单付款流程

1. 前端选择待付款的请款单
2. 前端点击「付款」按钮
3. 前端调用`purchaseFinlistApi.getData()`获取付款详情
4. 后端返回付款详情数据给前端
5. 前端显示付款详情对话框
6. 前端点击「确认」按钮
7. 前端调用`purchaseFinlistApi.paymentForm()`处理付款
8. 后端接收请求，解析参数
9. 后端调用`purchaseFinanceFormService.paymentForm()`处理付款
10. 后端返回操作结果给前端
11. 前端显示操作结果提示，刷新请款单列表

### 4.4 付款方式修改流程

1. 前端选择需要修改付款方式的请款单
2. 前端点击「修改付款方式」按钮
3. 前端调用`faccountApi.getPaymentMethod()`获取支付方式列表
4. 后端返回支付方式列表给前端
5. 前端显示批量修改付款方式对话框
6. 前端选择支付方式，调用`loadPaymentAccount()`加载支付账号
7. 前端点击「确认」按钮
8. 前端调用`purchaseFinlistApi.updatePay()`更新付款方式
9. 后端接收请求，解析参数
10. 后端调用`purchaseFinanceFormService.updatePay()`更新付款方式
11. 后端返回操作结果给前端
12. 前端显示操作结果提示，刷新请款单列表
