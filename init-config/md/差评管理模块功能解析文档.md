# 差评管理模块功能解析文档

## 1. 模块架构

### 1.1 技术栈

- **前端**：Vue 3 + Composition API + Element Plus
- **后端**：Spring Boot 2.x + MyBatis-Plus
- **数据库**：MySQL
- **API设计**：RESTful API

### 1.2 模块结构


## 2. 前端实现

### 2.1 主页面组件 (index.vue)

#### 核心功能

- 展示评论列表，支持分页
- 显示评论详细信息，包括站点、订单、产品、评分、评论内容等
- 集成搜索和筛选功能

#### 关键代码

```vue
<template>
  <div class="main-sty">
    <div class="con-header">
      <MyHeader @getdata="loadData"></MyHeader>
    </div>
    <el-row>
      <GlobalTable ref="globalTable" :tableData="tableData" height="calc(100vh - 254px)" @loadTable="loadTableData" :stripe="true" style="width: 100%;margin-bottom:16px;">
        <template #field>
          <!-- 表格列定义 -->
          <el-table-column prop="marketname" label="站点" width="100">
            <template #default="scope">
              <span>{{scope.row.groupname}}</span> <br />
              <span>{{scope.row.marketname}}</span>
            </template>
          </el-table-column>
          <!-- 其他列定义 -->
        </template>
      </GlobalTable>
    </el-row>
  </div>
</template>

<script setup>
import { ref, reactive, toRefs } from 'vue';
import GlobalTable from "@/components/Table/GlobalTable/index.vue";
import orderFeedbackApi from "@/api/amazon/performance/orderFeedbackApi.js";
import MyHeader from "./header.vue"

let state = reactive({
  tableData: { records: [], total: 0 },
  queryParams: {},
});

const { tableData } = toRefs(state);

function loadData(params) {
  globalTable.value.loadTable(params);
}

function loadTableData(params) {
  state.queryParams = params;
  orderFeedbackApi.list(params).then((res) => {
    if (res.data) {
      state.tableData.records = res.data.records;
      state.tableData.total = res.data.total;
    } else {
      state.tableData.records = [];
      state.tableData.total = 0;
    }
  })
}
</script>
```

### 2.2 搜索和筛选组件 (header.vue)

#### 核心功能

- 站点选择器，支持按亚马逊站点筛选
- 关键词搜索框，支持搜索评论内容、产品名称或SKU

#### 关键代码

```vue
<template>
  <div class="con-header">
    <el-row>
      <el-space>
        <Group @change="groupChange" />
        <el-input v-model="searchKeywords" clearable @input="searchConfirm" placeholder="请输入" class="input-with-select">
          <template #append>
            <el-button @click="searchConfirm">
              <el-icon style="font-size: 16px;align-itmes:center">
                <search />
              </el-icon>
            </el-button>
          </template>
        </el-input>
      </el-space>
    </el-row>
  </div>
</template>

<script setup>
import { reactive, toRefs } from 'vue';
import Group from '@/components/header/group.vue';
import { getCountryCode } from '@/utils/index.js';

const emit = defineEmits(['getdata']);
let state = reactive({
  queryParams: {},
  searchKeywords: "",
});

const { queryParams, searchKeywords } = toRefs(state);

function groupChange(obj) {
  state.queryParams.groupid = obj.groupid;
  state.queryParams.marketplaceid = obj.marketplaceid;
  state.queryParams.country = getCountryCode(obj.marketplaceid);
  state.queryParams.search = "";
  changeData();
}

function searchConfirm() {
  changeData();
}

function changeData() {
  state.queryParams.search = state.searchKeywords;
  emit('getdata', state.queryParams);
}
</script>
```

## 3. 后端实现

### 3.1 控制器 (AmzOrderFeedbackController.java)

#### 核心功能

- 处理前端请求，调用服务层获取评论数据
- 解析请求参数，设置用户信息

#### 关键代码

```java
@Api(tags = "Coupon接口")
@RestController
@RequestMapping("/api/v0/rpt/performance/orderfeedback")
@RequiredArgsConstructor
public class AmzOrderFeedbackController {
    final IAmzOrderFeedbackService iAmzOrderFeedbackService;
    
    @ApiOperation(value = "刷新接受信息的Destination")
    @PostMapping("/list")
    public Result<?> listAction(@RequestBody PerformanceReportListDTO dto) {
        UserInfo userInfo = UserInfoContext.get();
        dto.setShopid(userInfo.getCompanyid());
        return Result.success(iAmzOrderFeedbackService.findByCondition(dto));
    }
}
```

## 5. 功能流程

### 5.1 数据加载流程

1. 用户进入差评管理页面
2. 前端默认加载所有评论数据
3. 前端调用 `orderFeedbackApi.list()` 方法向后端请求数据
4. 后端 `AmzOrderFeedbackController` 接收请求，获取用户信息
5. 后端调用 `AmzOrderFeedbackService.findByCondition()` 方法查询数据
6. 服务层调用 Mapper 执行 SQL 查询
7. 数据库返回查询结果
8. 后端将结果封装为分页数据返回给前端
9. 前端接收到数据后更新表格显示

### 5.2 筛选流程

1. 用户选择站点或输入搜索关键词
2. 前端调用 `changeData()` 方法触发数据重新加载
3. 前端将筛选参数传递给 `loadTableData()` 方法
4. 前端重新调用 API 请求数据
5. 后端根据筛选参数查询数据并返回
6. 前端更新表格显示筛选结果

## 6. 性能优化

- **分页查询**：使用 MyBatis-Plus 的分页功能，避免一次性加载大量数据
- **索引优化**：数据库表应该在常用查询字段上建立索引，如 `amazon_order_id`、`feeddate`、`rating` 等
- **缓存策略**：对于频繁访问的数据，可以考虑使用缓存减少数据库查询
- **异步加载**：前端使用异步请求，避免阻塞页面渲染

## 7. 安全性

- **权限控制**：通过 `UserInfoContext` 获取用户信息，确保用户只能访问自己店铺的评论数据
- **输入验证**：前端和后端都应该对输入参数进行验证，防止恶意输入
- **SQL注入防护**：使用 MyBatis-Plus 的参数化查询，防止 SQL 注入攻击
- **XSS防护**：对评论内容进行 HTML 转义，防止 XSS 攻击

## 8. 扩展建议

- **评论回复功能**：在系统内直接添加评论回复功能，无需跳转到亚马逊后台
- **评论分析**：添加评论趋势分析和关键词提取功能，帮助卖家了解产品问题
- **自动通知**：当出现新的差评时，通过邮件或系统通知提醒相关人员
- **批量操作**：支持批量导出评论数据，方便进行离线分析
- **多维度筛选**：增加更多筛选条件，如评分范围、评论长度等

差评管理模块是一个功能完善的亚马逊评论管理工具，通过前后端的紧密配合，为卖家提供了便捷的差评监控和处理能力。该模块采用了现代化的技术栈，具有良好的性能和可扩展性。
