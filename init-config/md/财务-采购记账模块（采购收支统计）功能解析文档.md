# 财务-采购记账模块（采购收支统计）功能解析文档

## 1. 模块架构

### 1.1 技术栈

- **前端**：Vue 3 + Composition API + Element Plus + Axios + ECharts（图表展示）
- **后端**：Spring Boot 2.x + MyBatis-Plus + Spring Security
- **数据库**：MySQL
- **API设计**：RESTful API
- **数据传输**：JSON格式
- **Excel导出**：SXSSFWorkbook（支持大数量数据导出）

### 1.2 模块结构

#### 前端结构
```
wimoor-ui/src/views/erp/purchase/report/summary/
└── index.vue          # 采购统计主组件
```

#### 后端结构
```
wimoor-erp/erp-boot/src/main/java/com/wimoor/erp/purchase/
├── controller/
│   └── PurchaseFormController.java  # 采购单控制器（包含采购统计相关接口）
├── pojo/
│   └── dto/
│       └── PurchaseFormDTO.java     # 采购表单DTO
└── service/
    └── IPurchaseFormService.java    # 采购单服务接口
```

## 2. 前端实现

### 2.1 采购统计主组件（index.vue）

#### 核心功能
- 采购收支统计数据展示
- 支持按SKU和供应商维度汇总
- 多条件筛选（仓库、日期范围、关键词搜索）
- 数据导出功能
- 合计行计算

#### 关键代码分析

**模板结构**：
- 使用Element Plus的Tabs组件实现按SKU和供应商汇总的切换
- 使用Warehouse组件选择仓库
- 使用Datepicker组件选择日期范围
- 使用el-input输入搜索关键词
- 使用GlobalTable组件展示采购统计数据
- 支持表格排序和合计功能

**数据管理**：
- 使用reactive和ref管理组件状态
- 定义queryParams对象管理查询参数
- 定义summary对象管理合计数据

**核心方法**：
- `handleQuery()`：处理查询操作，根据条件加载采购统计数据
- `loadTableData()`：加载表格数据，调用`listApi.getPurchaseFormReport()`获取采购统计数据
- `downLoadExcel()`：导出采购统计数据，调用`listApi.downExcelPurchaseFormReport()`
- `wareChange()`：处理仓库选择变化
- `changedate()`：处理日期范围变化
- `getSummaries()`：计算表格合计行数据

**图表展示**：
- 使用ECharts库实现年度和月度收支统计图表
- 支持多种图表类型：折线图、柱状图、饼图等
- 图表数据来源于后端API返回的统计数据

### 2.2 API调用

**核心API调用**：
- `getPurchaseFormReport()`：获取采购统计数据
- `downExcelPurchaseFormReport()`：导出采购统计数据

## 3. 后端实现

### 3.1 采购单控制器（PurchaseFormController.java）

#### 核心功能
- 采购统计数据查询
- 采购统计数据导出
- 采购统计合计计算

#### 关键API接口

**1. 获取采购统计数据**
- **接口**：`POST /api/v1/purchase_form/getPurchaseFormReport`
- **功能**：根据条件查询采购统计数据，支持按SKU和供应商维度汇总
- **参数**：`PurchaseFormDTO`（包含仓库ID、日期范围、搜索关键词、汇总类型等查询条件）
- **返回**：`IPage<Map<String, Object>>`（分页的采购统计数据）

**2. 导出采购统计数据**
- **接口**：`POST /api/v1/purchase_form/downExcelPurchaseFormReport`
- **功能**：导出采购统计数据为Excel文件
- **参数**：`PurchaseFormDTO`（包含导出条件）
- **返回**：Excel文件流

### 3.2 采购单服务（IPurchaseFormService）

#### 核心方法
- `purchaseFormReport()`：查询采购统计数据
- `purchaseFormReportTotal()`：计算采购统计合计数据
- `setPurchaseFormReportExcelBook()`：生成采购统计Excel文件

## 4. 业务逻辑流程

### 4.1 采购统计查询流程

1. 前端设置查询条件（汇总维度、仓库、日期范围、搜索关键词）
2. 前端调用`listApi.getPurchaseFormReport()`方法
3. 后端接收请求，解析参数
4. 后端调用`purchaseFormService.purchaseFormReport()`查询采购统计数据
5. 后端调用`purchaseFormService.purchaseFormReportTotal()`计算合计数据
6. 后端返回分页的采购统计数据和合计数据给前端
7. 前端展示采购统计数据，包括合计行

### 4.2 采购统计导出流程

1. 前端点击导出按钮
2. 前端调用`listApi.downExcelPurchaseFormReport()`方法
3. 后端接收请求，解析导出条件
4. 后端调用`purchaseFormService.setPurchaseFormReportExcelBook()`生成Excel文件
5. 后端设置响应头，返回Excel文件流
6. 前端接收文件流，下载Excel文件

### 4.3 图表数据生成流程

1. 前端选择时间范围（年度或月度）
2. 前端调用`listApi.getPurchaseFormReport()`方法获取采购统计数据
3. 后端返回采购统计数据给前端
4. 前端处理数据，转换为图表所需格式
5. 前端使用ECharts库生成图表
6. 前端展示图表，直观呈现采购收支趋势

## 5. 数据模型

### 5.1 核心数据传输对象（DTO）

**PurchaseFormDTO**：采购表单DTO
- `page`：分页信息
- `warehouseid`：仓库ID
- `ftype`：汇总类型（sku：按SKU汇总，supplier：按供应商汇总）
- `search`：搜索关键词
- `fromDate`：开始日期
- `toDate`：结束日期

### 5.2 数据结构

**采购统计数据结构**：
- `sku`：产品SKU（按SKU汇总时）
- `supplier`：供应商名称（按供应商汇总时）
- `orderamount`：采购数量
- `totalorderprice`：采购金额
- `itemprice`：采购单价
- `weight`：预估重量
- `totalcostfee`：付款金额
- `waitPay`：待付款金额
- `pprice`：实际单价
- `totalshipfee`：采购运费
- `totalotherfee`：其它费用
- `lesspay`：付款差额
- `actual_totalin`：入库(不含退货)
- `totalin`：入库(含退货)
- `needin`：待入库数
- `totalre`：退货数
- `lessrec`：入库差额
- `summary`：合计数据

**合计数据结构**：
- `orderamount`：采购数量合计
- `totalorderprice`：采购金额合计
- `totalcostfee`：付款金额合计
- `waitPay`：待付款金额合计
- `totalshipfee`：采购运费合计
- `totalotherfee`：其它费用合计
- `actual_totalin`：入库(不含退货)合计
- `totalin`：入库(含退货)合计
- `needin`：待入库数合计
- `totalre`：退货数合计
