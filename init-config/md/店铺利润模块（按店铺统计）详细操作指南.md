# 店铺利润模块（按店铺统计）详细操作指南

## 1. 模块概述

店铺利润模块是Wimoor系统中用于按店铺和站点分别统计各项店铺费用和利润的核心财务管理功能。通过该模块，您可以清晰地了解每个店铺和站点的销售情况、费用结构和利润状况，为企业的财务分析和决策提供有力支持。

### 1.1 核心功能

- **店铺和站点分类统计**：按店铺和站点分别统计各项费用和利润
- **多币种支持**：支持人民币(CNY)、美元(USD)、站点币种三种币种切换
- **日期范围选择**：支持自定义日期范围查询
- **数据导出**：支持导出店铺利润报表为Excel文件
- **结算日期和转账日期切换**：支持按结算日期或转账日期统计
- **数据汇总**：自动汇总店铺和站点的各项费用和利润

### 1.2 适用场景

- **财务分析**：分析不同店铺和站点的盈利能力
- **成本控制**：识别成本过高的环节，优化成本结构
- **业务决策**：基于数据做出合理的业务决策
- **绩效评估**：评估不同店铺和站点的经营绩效

1. 打开浏览器，输入Wimoor系统的登录地址
2. 输入您的用户名和密码
3. 点击"登录"按钮进入系统

## 4. 详细操作步骤

### 4.1 查看店铺利润数据

**操作步骤**：

1. **选择店铺**：
   - 点击店铺选择器的下拉菜单
   - 在弹出的店铺列表中，选择您要查看的店铺
   - 店铺选择后，站点选择器会自动更新为该店铺对应的站点

2. **选择站点**：
   - 点击站点选择器的下拉菜单
   - 在弹出的站点列表中，选择您要查看的站点
   - 如果只选择店铺不选择站点，将显示该店铺所有站点的汇总数据

3. **设置日期范围**：
   - 点击日期范围选择器
   - 在弹出的日历中，选择查询的开始日期
   - 选择查询的结束日期
   - 点击"确定"按钮
   - 默认日期范围为最近7天

4. **选择币种**：
   - 点击币种选择器的下拉菜单
   - 选择您要查看的币种：
     - 人民币(CNY)：以人民币显示所有金额
     - 美元(USD)：以美元显示所有金额
     - 站点币种(LOCAL)：以各站点的本地币种显示金额

5. **选择日期类型**：
   - 点击日期类型选择器的下拉菜单
   - 选择结算日期或转账日期：
     - 结算日期：按平台结算日期统计数据
     - 转账日期：按实际转账日期统计数据

6. **点击查询**：
   - 点击"查询"按钮
   - 系统会根据设置的条件显示店铺利润数据

7. **查看数据**：
   - 在数据表格中查看各项费用和利润数据
   - 点击表格列标题可以对数据进行排序
   - 通过分页控件查看更多数据

### 4.2 导出利润报表

**操作步骤**：

1. **设置查询条件**：
   - 按照上述步骤设置好店铺、站点、日期范围、币种和日期类型

2. **点击导出**：
   - 点击"导出"按钮
   - 系统会生成Excel格式的利润报表
   - 报表会自动下载到您的电脑

3. **查看报表**：
   - 打开下载的Excel文件
   - 查看详细的利润数据
   - 可以对数据进行进一步分析或打印

## 5. 数据说明

### 5.1 主要数据字段说明

| 字段名称 | 说明 | 计算方式 |
|---------|------|----------|
| 销售额 | 店铺在所选时间段内的销售总额 | 所有订单的销售金额总和 |
| 退款金额 | 店铺在所选时间段内的退款总额 | 所有退款订单的金额总和 |
| FBA费用 | 店铺在所选时间段内的FBA费用总额 | 亚马逊收取的FBA物流费用总和 |
| 销售佣金 | 店铺在所选时间段内的销售佣金总额 | 亚马逊收取的销售佣金总和 |
| 仓储费 | 店铺在所选时间段内的仓储费用总额 | 亚马逊收取的仓储费用总和 |
| 广告费 | 店铺在所选时间段内的广告费用总额 | 亚马逊广告费用总和 |
| 采购成本 | 店铺在所选时间段内的采购成本总额 | 产品采购成本总和 |
| 利润 | 店铺在所选时间段内的利润 | 销售额 - 退款金额 - FBA费用 - 销售佣金 - 仓储费 - 广告费 - 采购成本 |
| 利润率 | 店铺在所选时间段内的利润率 | 利润 ÷ (销售额 - 退款金额) × 100% |

### 5.2 币种说明

| 币种类型 | 说明 | 适用场景 |
|---------|------|----------|
| 人民币(CNY) | 以人民币显示所有金额 | 中国企业财务核算 |
| 美元(USD) | 以美元显示所有金额 | 国际业务分析 |
| 站点币种(LOCAL) | 以各站点的本地币种显示金额 | 本地业务分析 |

### 5.3 日期类型说明

| 日期类型 | 说明 | 适用场景 |
|---------|------|----------|
| 结算日期 | 按平台结算日期统计数据 | 财务核算，与平台结算周期一致 |
| 转账日期 | 按实际转账日期统计数据 | 现金流分析，与实际到账时间一致 |

## 6. 常见问题及解决方案

### 6.1 数据不显示

**问题**：选择店铺和站点后，表格中没有显示数据

**解决方案**：
1. **检查日期范围**：确保选择的日期范围内有数据。如果是新店铺，可能还没有产生销售数据
2. **检查店铺和站点选择**：确保正确选择了店铺和站点
3. **检查网络连接**：确保网络连接正常，尝试刷新页面
4. **检查权限**：确保您有查看该店铺数据的权限
5. **联系管理员**：如果以上方法都无效，请联系系统管理员

### 6.2 导出失败

**问题**：点击导出按钮后，报表没有下载

**解决方案**：
1. **检查浏览器设置**：确保浏览器没有阻止下载
2. **检查网络连接**：确保网络连接正常
3. **减少数据量**：尝试缩小日期范围，减少需要导出的数据量
4. **使用Chrome浏览器**：推荐使用Chrome浏览器进行导出操作
5. **联系管理员**：如果以上方法都无效，请联系系统管理员

### 6.3 数据不准确

**问题**：显示的利润数据与预期不符

**解决方案**：
1. **检查日期范围**：确保日期范围设置正确
2. **检查币种选择**：确保选择了正确的币种
3. **检查日期类型**：确保选择了正确的日期类型（结算日期/转账日期）
4. **检查数据同步**：确保系统数据已与平台同步
5. **联系管理员**：如果数据仍然不准确，请联系系统管理员检查数据同步情况

### 6.4 页面加载缓慢

**问题**：页面加载或查询数据时速度较慢

**解决方案**：
1. **缩小日期范围**：尝试缩小日期范围，减少数据量
2. **检查网络连接**：确保网络连接正常
3. **避开高峰期**：避免在系统高峰期使用
4. **清理浏览器缓存**：清理浏览器缓存后重新登录
5. **联系管理员**：如果问题持续存在，请联系系统管理员优化系统性能

### 6.5 权限不足

**问题**：无法查看某些店铺的数据

**解决方案**：
1. **检查权限**：确保您有查看该店铺数据的权限
2. **联系管理员**：如果需要查看更多店铺的数据，请联系系统管理员为您添加相应权限

## 7. 操作技巧

### 7.1 快速查看数据

- **使用默认日期范围**：系统默认显示最近7天的数据，适合快速查看近期业绩
- **使用快捷键**：按Enter键可以快速提交查询
- **使用排序功能**：点击表格列标题可以快速排序数据，找出表现最好或最差的店铺

### 7.2 高效导出数据

- **选择合适的日期范围**：根据需要选择合适的日期范围，避免导出过多数据
- **使用站点币种**：如果需要与当地团队共享数据，选择站点币种导出
- **定期导出**：定期导出数据进行备份和分析

### 7.3 数据分析技巧

- **对比分析**：选择不同的时间段进行对比，分析业务趋势
- **多维度分析**：结合其他模块的数据进行多维度分析
- **重点关注**：重点关注利润率低的店铺，分析原因并优化

## 8. 常见操作场景

### 8.1 月度财务分析

**操作步骤**：
1. 设置日期范围为上个月的1号到最后一天
2. 选择店铺和所有站点
3. 选择人民币币种
4. 点击查询
5. 查看各店铺和站点的利润数据
6. 导出报表进行详细分析

### 8.2 多店铺对比

**操作步骤**：
1. 设置日期范围为最近30天
2. 不选择具体店铺和站点（显示所有店铺汇总数据）
3. 选择美元币种
4. 点击查询
5. 点击"店铺"列标题排序，查看各店铺的表现
6. 分析表现最好和最差的店铺

### 8.3 成本分析

**操作步骤**：
1. 设置日期范围为最近一个季度
2. 选择特定店铺和站点
3. 选择站点币种
4. 点击查询
5. 分析各项费用占比
6. 识别成本过高的环节

**Q：为什么某些店铺的数据没有显示？**

A：目前系统支持按店铺和站点分别查看数据，您可以通过导出报表后在Excel中进行多店铺对比。
