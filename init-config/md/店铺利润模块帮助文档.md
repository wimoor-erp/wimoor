# 店铺利润模块帮助文档

## 1. 模块概述

店铺利润模块是Wimoor系统中用于按店铺和站点分别统计各项店铺费用和利润的核心财务管理功能。该模块支持按店铺和站点分类汇总销售额、销售佣金、FBA费用、退款金额、仓储费、广告费、采购成本等费用，并计算出店铺利润，为企业的财务分析和决策提供有力支持。

### 1.1 核心功能

- **店铺和站点分类统计**：按店铺和站点分别统计各项费用和利润
- **多币种支持**：支持人民币(CNY)、美元(USD)、站点币种三种币种切换
- **日期范围选择**：支持自定义日期范围查询
- **数据导出**：支持导出店铺利润报表
- **结算日期和转账日期切换**：支持按结算日期或转账日期统计
- **数据汇总**：自动汇总店铺和站点的各项费用和利润

### 1.2 技术架构

- **前端**：Vue 3、Composition API、Element Plus、Axios
- **后端**：Spring Boot 2.x、MyBatis-Plus、Spring Security
- **数据库**：MySQL
- **API风格**：RESTful API
- **数据传输**：JSON格式
- **汇率转换**：内置汇率转换服务

const settlementAccRptApi = {
  // 获取店铺利润数据
  getSettlementOverAll(params) {
    return axios.post('/api/v1/settlement/getSettlementOverAll', params);
  },
  
  // 导出店铺利润数据
  downloadSettlementOverAll(params) {
    return axios({
      url: '/api/v1/settlement/downloadSettlementOverAll',
      method: 'post',
      responseType: 'blob',
      data: params
    }).then(res => {
      const url = window.URL.createObjectURL(new Blob([res.data]));
      const link = document.createElement('a');
      link.href = url;
      link.setAttribute('download', `店铺利润报表_${new Date().getTime()}.xlsx`);
      document.body.appendChild(link);
      link.click();
      document.body.removeChild(link);
    });
  }
};

export default settlementAccRptApi;

@PostMapping("/getSettlementOverAll")
public Result<?> getSettlementOverAllAction(@RequestBody AmzSettlementDTO dto) {
    UserInfo user = UserInfoContext.get();
    
    String marketplaceid = dto.getMarketplaceid();
    String currency = dto.getCurrency();
    String fromDate = dto.getFromDate();
    String toDate = dto.getEndDate();
    String groupid = dto.getGroupid();
    String marketplace_name = dto.getMarketplace_name();
    String datetype = dto.getDatetype();
    String search = dto.getSearch();
    
    Map<String, Object> map = new HashMap<String, Object>();
    SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd");
    
    // 处理日期参数
    if (StrUtil.isNotEmpty(fromDate)) {
        fromDate = fromDate.replace("/", "-");
        map.put("fromDate", fromDate.trim());
    } else {
        Calendar cal = Calendar.getInstance();
        cal.add(Calendar.DAY_OF_MONTH, -7);
        fromDate = GeneralUtil.formatDate(cal.getTime(), sdf);
        map.put("fromDate", fromDate);
    }
    
    if (StrUtil.isNotEmpty(toDate)) {
        toDate = toDate.replace("/", "-");
        map.put("endDate", toDate.trim());
    } else {
        toDate = GeneralUtil.formatDate(new Date(), sdf);
        map.put("endDate", toDate);
    }
    
    // 处理其他参数
    if (StrUtil.isEmpty(marketplaceid) || "all".equals(marketplaceid)) {
        marketplaceid = null;
    } else {
        Marketplace mark = marketplaceService.findMapByMarketplaceId().get(marketplaceid);
        map.put("country", mark.getMarket());
    }
    
    if (StrUtil.isEmpty(marketplace_name) || "all".equals(marketplace_name)) {
        marketplace_name = null;
    }
    
    if (StrUtil.isEmpty(groupid) || "all".equals(groupid)) {
        groupid = null;
    }
    
    if (StrUtil.isEmpty(search)) {
        search = null;
    } else {
        search = "%" + search.trim() + "%";
    }
    
    if (user.isLimit(UserLimitDataType.operations)) {
        map.put("myself", user.getId());
    }
    
    if (StrUtil.isEmpty(datetype)) {
        map.put("datetype", null);
    } else {
        map.put("datetype", datetype);
    }
    
    map.put("groupid", groupid);
    map.put("shopid", user.getCompanyid());
    map.put("marketplaceid", marketplaceid);
    
    // 处理币种
    Marketplace market = marketplaceService.findMapByMarketplaceId().get(marketplaceid);
    if (market != null) {
        map.put("marketplace_name", market.getPointName());
        if ("market".equals(currency)) {
            currency = market.getCurrency();
        }
    } else {
        map.put("marketplace_name", null);
    }
    
    map.put("currency", currency);
    map.put("search", search);
    
    // 获取授权信息
    if (groupid != null && marketplaceid != null) {
        AmazonAuthority auth = amazonAuthorityService.selectByGroupAndMarket(groupid, marketplaceid);
        if (auth != null) {
            map.put("amazonAuthId", auth.getId());
            map.put("sellerid", auth.getSellerid());
        }
    }
    
    // 调用服务获取数据
    List<Map<String, Object>> list = iAmzSettlementReportService.getSettlementGroupOverAll(map);
    if (list != null) {
        return Result.success(list);
    } else {
        return Result.success();
    }
}

## 8. 常见问题及解决方案

### 8.1 数据准确性问题

**问题**：店铺利润数据与实际不符
**原因**：
- 数据未及时更新
- 币种转换错误
- 日期范围选择错误

**解决方案**：
- 确保数据已经同步到系统
- 检查币种设置是否正确
- 确认日期范围选择是否正确

### 8.2 数据加载缓慢

**问题**：查询店铺利润数据时加载缓慢
**原因**：
- 数据量过大
- 查询条件过于复杂
- 服务器性能不足

**解决方案**：
- 缩小查询日期范围
- 增加筛选条件，减少查询数据量
- 优化服务器配置

### 8.3 导出失败

**问题**：导出店铺利润数据时失败
**原因**：
- 数据量过大
- 网络连接中断
- 服务器内存不足

**解决方案**：
- 缩小导出数据范围
- 确保网络连接稳定
- 增加服务器内存

### 8.4 币种转换错误

**问题**：币种转换后的数据不正确
**原因**：
- 汇率数据过期
- 转换逻辑错误

**解决方案**：
- 确保汇率数据及时更新
- 检查币种转换配置

## 9. 总结

店铺利润模块是Wimoor系统中一个功能强大的财务管理工具，通过按店铺和站点分类统计各项费用和利润，为企业提供了全面的财务分析能力。该模块采用前后端分离架构，前端使用Vue 3和Element Plus构建了美观、易用的用户界面，后端使用Spring Boot和MyBatis-Plus构建了高效、可靠的API接口。

通过该模块，企业可以：
- 全面了解店铺和站点的利润情况
- 分析不同店铺和站点的费用结构
- 优化成本结构，提高盈利能力
- 做出数据驱动的业务决策

店铺利润模块的设计考虑了企业的实际需求，提供了灵活的配置选项和丰富的功能，能够满足不同企业的财务管理需求。同时，该模块的代码结构清晰，易于维护和扩展，为后续功能的迭代和优化奠定了良好的基础。
