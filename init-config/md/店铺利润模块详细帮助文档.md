# 店铺利润模块详细帮助文档

## 1. 模块概述

店铺利润模块是Wimoor系统的核心财务管理功能，用于查询、分析和管理店铺及产品的利润数据。该模块支持按多维度（如SKU、MSKU、ASIN、父ASIN、品类、店铺等）查看利润数据，同时提供币种切换、数据导出、公式配置等功能，为企业的财务分析和决策提供有力支持。

### 1.1 核心功能

- **多维度分析**：支持按SKU、MSKU、ASIN、父ASIN、品类、店铺等多维度查看利润数据
- **币种切换**：支持在人民币(CNY)、美元(USD)、站点币种之间切换查看
- **数据筛选**：支持按店铺、国家、日期范围、负责人、品类等多条件筛选
- **关键词搜索**：支持按MSKU、ASIN、父ASIN、SKU、产品名称等关键词搜索
- **数据导出**：支持导出利润报表数据
- **重算功能**：支持手动触发数据重算
- **公式配置**：支持自定义毛利润计算公式
- **数据汇总**：提供订单量、销量、销售额、利润等汇总数据
- **退款率计算**：自动计算退款率，帮助分析产品质量

### 1.2 技术架构

- **前端**：Vue 3、Composition API、Element Plus、Axios
- **后端**：Spring Boot 2.x、MyBatis-Plus、Spring Security
- **数据库**：MySQL
- **API风格**：RESTful API
- **数据传输**：JSON格式
- **汇率转换**：内置汇率转换服务
- **表达式引擎**：Aviator表达式引擎（用于利润计算公式）

@Api(tags = "账期月报")
@RestController
@RequiredArgsConstructor
@RequestMapping("/api/v1/settlementSummarySkuMonth")
public class SettlementSummarySkuMonthController {
    final IAmzSettlementSummarySkuMonthService iAmzSettlementSummarySkuMonthService;
    final IMarketplaceService marketplaceService;
    final IAmazonSettlementAnalysisService amazonSettlementAnalysisAgentService;
    
    @PostMapping("/list")
    public Result<IPage<Map<String, Object>>> listAction(@RequestBody AmzSettlementDTO dto) {
        UserInfo user = UserInfoContext.get();
        String shopid = user.getCompanyid();
        String marketplaceid = dto.getMarketplaceid();
        String currency = dto.getCurrency();
        String fromDate = dto.getFromDate();
        String toDate = dto.getEndDate();
        String groupid = dto.getGroupid();
        String pointname = dto.getMarketplace_name();
        String datetype = dto.getDatetype();
        String parentasin = dto.getParentasin();
        String searchType = dto.getFatype();
        String search = dto.getSearch();
        Map<String, Object> map = new HashMap<String, Object>();
        map.put("fromDate", fromDate.trim().subSequence(0, 7) + "-01");
        map.put("endDate", toDate.trim().subSequence(0, 7) + "-01");
        map.put("type", dto.getCharttype());
        // 处理marketplaceid参数
        if (StrUtil.isEmpty(marketplaceid) || "all".equals(marketplaceid)) {
            marketplaceid = null;
            map.put("marketplaceName", null);
        } else {
            Marketplace market = marketplaceService.findMapByMarketplaceId().get(marketplaceid);
            if (market != null) {
                map.put("marketplaceName", market.getPointName());
            } else {
                map.put("marketplaceName", null);
            }
        }
        // 处理其他参数...
        map.put("searchtype", searchType);
        map.put("search", search);
        map.put("shopid", shopid);
        map.put("parentasin", parentasin);
        map.put("currency", currency);
        map.put("groupid", groupid);
        map.put("owner", ownerid);
        map.put("categoryid", dto.getCategoryid());
        map.put("datetype", datetype);
        IPage<Map<String, Object>> list = iAmzSettlementSummarySkuMonthService.findByCondition(dto.getPage(), map);
        return Result.success(list);
    }
    
    @PostMapping("/downloadList")
    public void downDataExcelByRateAction(@RequestBody AmzSettlementDTO dto, HttpServletResponse response) {
        // 创建新的Excel工作薄
        SXSSFWorkbook workbook = new SXSSFWorkbook();
        // 处理参数...
        iAmzSettlementSummarySkuMonthService.getDownloadList(workbook, map);
        try {
            response.setContentType("application/force-download");// 设置强制下载不打开
            response.addHeader("Content-Disposition", "attachment;fileName=CommodityRevenueFinRate" + System.currentTimeMillis() + ".xlsx");// 设置文件名
            ServletOutputStream fOut = response.getOutputStream();
            workbook.write(fOut);
            workbook.close();
            fOut.flush();
            fOut.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
    
    @PostMapping("/getRefreshData")
    @CacheEvict(value = { "findSettlementSKUCache" }, allEntries = true)
    public Result<?> getGroupMarketAction(@RequestBody FinDataMonthDTO dto) {
        UserInfo user = UserInfoContext.get();
        String key = dto.getProgresskey();
        if(StrUtil.isBlank(key)) {
            key = "settlement_refresh_data";
        }
        dto.setShopid(user.getCompanyid());
        ProgressHelper progressHelper = new ProgressHelper(user, key);
        new Thread(new Runnable() {
            @Override
            public void run() {
                amazonSettlementAnalysisAgentService.refreshData(progressHelper, dto);
            }
        }).start();
        return Result.success(key);
    }
}

## 4. 数据流程

### 4.1 利润数据查询流程

1. **前端发起请求**：用户设置筛选条件，选择分析维度，点击搜索按钮
2. **参数处理**：前端处理筛选参数，确保参数完整性
3. **API调用**：前端调用`list` API发送POST请求
4. **后端处理**：
   - 控制器接收请求，获取用户信息
   - 处理各种查询参数，构建查询条件
   - 调用服务层方法`findByCondition`获取利润数据列表
   - 服务层调用Mapper层查询数据库
   - 计算退款率并添加汇总数据
   - 控制器返回分页结果
5. **前端处理**：
   - 接收响应数据
   - 更新表格数据
   - 显示汇总数据

### 4.2 数据导出流程

1. **前端发起请求**：用户设置筛选条件，点击导出按钮
2. **参数处理**：前端处理筛选参数，确保参数完整性
3. **API调用**：前端调用`downloadList` API发送POST请求
4. **后端处理**：
   - 控制器接收请求，获取用户信息
   - 处理各种查询参数，构建查询条件
   - 创建Excel工作簿
   - 调用服务层方法`getDownloadList`生成Excel数据
   - 设置响应头，输出Excel文件
5. **前端处理**：
   - 接收Excel文件
   - 自动下载到本地
   - 显示下载完成提示

### 4.3 数据重算流程

1. **前端发起请求**：用户点击重算按钮，确认操作
2. **参数处理**：前端处理重算参数
3. **API调用**：前端调用`getRefreshData` API发送POST请求
4. **后端处理**：
   - 控制器接收请求，获取用户信息
   - 处理进度键参数
   - 创建ProgressHelper对象
   - 启动异步线程，调用分析服务的`refreshData`方法
   - 控制器返回进度键
5. **前端处理**：
   - 接收响应数据
   - 显示重算进度
   - 重算完成后显示成功提示
   - 刷新利润数据列表

## 5. 关键功能说明

### 5.1 多维度分析

- **SKU维度**：按平台SKU查看利润数据
- **MSKU维度**：按本地SKU查看利润数据
- **ASIN维度**：按平台ASIN查看利润数据
- **父ASIN维度**：按父ASIN查看利润数据，支持展开查看子ASIN数据
- **品类维度**：按产品品类查看利润数据
- **店铺维度**：按店铺查看利润数据

### 5.2 费用分析

- **销售相关费用**：销售佣金、FBA费用、买家运费、促销费、其它收支
- **店铺分摊费用**：仓储费、长期仓储费、折扣券、广告花费、预留金、赔偿金、移除费、其它收支
- **预估费用**：仓储费、市场费用、所得税、关税、汇率损耗、固定费用、其它成本
- **SKU报表费用**：仓储费、长期仓储费、广告花费、赔偿金、移除费、移除数、自定义费用
- **采购相关费用**：采购成本、其它采购成本、退税、货件头程运费

### 5.3 币种切换

- **人民币(CNY)**：将所有费用转换为人民币显示
- **美元(USD)**：将所有费用转换为美元显示
- **站点币种**：按各站点的本地币种显示费用

### 5.4 公式配置

- **自定义毛利润计算公式**：用户可以根据需要自定义毛利润的计算公式
- **公式变量**：支持使用各种费用项目作为公式变量
- **实时计算**：根据配置的公式实时计算毛利润

## 8. 常见问题及解决方案

### 8.1 数据准确性问题

**问题**：利润数据与实际不符
**原因**：
- 数据未及时更新
- 计算公式配置错误
- 费用分摊不正确

**解决方案**：
- 点击"重算"按钮，触发数据重算
- 检查并调整毛利润计算公式
- 检查费用分摊设置

### 8.2 数据加载缓慢

**问题**：查询利润数据时加载缓慢
**原因**：
- 数据量过大
- 查询条件过于复杂
- 服务器性能不足

**解决方案**：
- 缩小查询日期范围
- 增加筛选条件，减少查询数据量
- 优化服务器配置

### 8.3 导出失败

**问题**：导出利润数据时失败
**原因**：
- 数据量过大
- 网络连接中断
- 服务器内存不足

**解决方案**：
- 缩小导出数据范围
- 确保网络连接稳定
- 增加服务器内存

### 8.4 币种转换错误

**问题**：币种转换后的数据不正确
**原因**：
- 汇率数据过期
- 转换逻辑错误

**解决方案**：
- 确保汇率数据及时更新
- 检查币种转换配置

### 8.5 公式计算错误

**问题**：自定义公式计算结果不正确
**原因**：
- 公式语法错误
- 变量引用错误
- 运算逻辑错误

**解决方案**：
- 检查公式语法
- 确保变量引用正确
- 验证运算逻辑
