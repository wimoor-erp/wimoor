# 财务-库存货值模块功能解析文档

## 1. 模块架构

### 1.1 技术栈

- **前端**：Vue 3 + Composition API + Element Plus + Axios
- **后端**：Spring Boot 2.x + MyBatis-Plus + Spring Security
- **数据库**：MySQL
- **API设计**：RESTful API
- **数据传输**：JSON格式
- **Excel导出**：SXSSFWorkbook（支持大数量数据导出）

### 1.2 模块结构

#### 前端结构
```
wimoor-ui/src/views/erp/finance/inventoryValue/
├── index.vue          # 库存货值主组件
└── components/
    ├── self.vue       # 本地仓库存货值组件
    └── fba.vue        # FBA仓库存货值组件
```

#### 后端结构
```
wimoor-erp/erp-boot/src/main/java/com/wimoor/erp/inventory/
├── controller/
│   ├── InventoryController.java          # 库存控制器
│   └── InventoryReportController.java    # 库存报表控制器（包含库存货值相关接口）
├── pojo/
│   └── dto/
│       └── InventoryValueReportDTO.java  # 库存货值报表DTO
└── service/
    └── IInventoryService.java            # 库存服务接口
```

## 2. 前端实现

### 2.1 库存货值主组件（index.vue）

#### 核心功能
- 提供本地仓和FBA仓的切换功能
- 管理子组件的显示和隐藏

#### 关键代码分析

**模板结构**：
- 使用Element Plus的Tabs组件实现本地仓和FBA仓的切换
- 根据activeName的值显示对应的子组件

**数据管理**：
- 使用reactive和ref管理组件状态
- 默认显示本地仓（activeName='1'）

### 2.2 本地仓库存货值组件（components/self.vue）

#### 核心功能
- 本地仓库存货值查询和展示
- 支持按仓库、日期、SKU筛选
- 库存货值数据导出
- 显示合计行

#### 关键代码分析

**模板结构**：
- 使用Warehouse组件选择仓库
- 使用el-date-picker选择库存日期
- 使用el-input输入SKU进行搜索
- 使用GlobalTable组件展示库存货值数据
- 支持表格排序和合计功能

**数据加载流程**：
1. 组件初始化时，设置默认查询参数（默认日期为当天）
2. 选择仓库后，调用`getWarehouse()`方法更新仓库ID并触发查询
3. 选择日期后，调用`dateChange()`方法更新日期并触发查询
4. 输入SKU后，调用`handleQuery()`方法触发查询
5. `loadTableData()`方法通过`inventoryRptApi.inventoryCost()`获取库存货值数据

**导出功能**：
- 点击导出按钮时，调用`download()`方法
- `download()`方法调用`inventoryRptApi.downLoadInvCost()`导出数据

**日期处理**：
- 选择日期时，判断是否为当天
- 如果是当天，设置`queryParams.today=true`，显示待入库库存、在途货值和采购待付款
- 如果不是当天，设置`queryParams.today=false`，只显示现有库存、采购成本单价、总货值和其它成本

### 2.3 FBA仓库存货值组件（components/fba.vue）

#### 核心功能
- FBA仓库存货值查询和展示
- 支持按店铺、日期、SKU筛选
- FBA库存货值数据导出
- 显示合计行

#### 关键代码分析

**模板结构**：
- 使用Group组件选择店铺
- 使用el-date-picker选择库存日期
- 使用el-input输入SKU进行搜索
- 使用GlobalTable组件展示FBA库存货值数据
- 支持表格排序和合计功能

**数据加载流程**：
1. 组件初始化时，设置默认查询参数（默认日期为当天）
2. 选择店铺后，调用`changeGroup()`方法更新店铺ID和市场ID并触发查询
3. 选择日期后，调用`dateChange()`方法更新日期并触发查询
4. 输入SKU后，调用`handleQuery()`方法触发查询
5. `loadTableData()`方法通过`inventoryRptApi.inventoryCostFBA()`获取FBA库存货值数据

**导出功能**：
- 点击导出按钮时，调用`download()`方法
- `download()`方法调用`inventoryRptApi.downloadInvCostFBA()`导出数据

## 3. 后端实现

### 3.1 库存报表控制器（InventoryReportController.java）

#### 核心功能
- 库存货值查询
- 库存货值数据导出

#### 关键API接口

**1. 查询本地仓库存货值**
- **接口**：`POST /api/v1/inventory/report/inventoryCost`
- **功能**：根据条件查询本地仓库存货值数据
- **参数**：`InventoryValueReportDTO`（包含仓库ID、SKU、日期等查询条件）
- **返回**：`IPage<Map<String, Object>>`（分页的库存货值数据）

**2. 导出本地仓库存货值**
- **接口**：`POST /api/v1/inventory/report/downLoadInvCost`
- **功能**：导出本地仓库存货值数据为Excel文件
- **参数**：`InventoryValueReportDTO`（包含导出条件）
- **返回**：Excel文件流

**3. 查询FBA仓库存货值**
- **接口**：`POST /api/v1/inventory/report/inventoryCostFBA`（推测，基于前端调用）
- **功能**：根据条件查询FBA仓库存货值数据
- **参数**：包含店铺ID、市场ID、SKU、日期等查询条件
- **返回**：分页的FBA库存货值数据

**4. 导出FBA仓库存货值**
- **接口**：`POST /api/v1/inventory/report/downloadInvCostFBA`（推测，基于前端调用）
- **功能**：导出FBA仓库存货值数据为Excel文件
- **参数**：包含导出条件
- **返回**：Excel文件流

### 3.2 库存服务（IInventoryService）

#### 核心方法
- `selectInventoryCost()`：查询本地仓库存货值数据
- `selectInventoryCostAll()`：查询所有本地仓库存货值数据（用于导出）
- `selectInventoryCostFBA()`：查询FBA仓库存货值数据（推测）
- `selectInventoryCostFBAAll()`：查询所有FBA仓库存货值数据（用于导出，推测）

## 4. 业务逻辑流程

### 4.1 本地仓库存货值查询流程

1. 前端选择查询条件（仓库、日期、SKU）
2. 前端调用`inventoryRptApi.inventoryCost()`方法
3. 后端接收请求，解析参数
4. 后端调用`inventoryService.selectInventoryCost()`查询库存货值数据
5. 后端计算库存货值的合计数据
6. 后端返回分页的库存货值数据给前端
7. 前端展示库存货值数据，包括合计行

### 4.2 FBA仓库存货值查询流程

1. 前端选择查询条件（店铺、日期、SKU）
2. 前端调用`inventoryRptApi.inventoryCostFBA()`方法
3. 后端接收请求，解析参数
4. 后端调用`inventoryService.selectInventoryCostFBA()`查询FBA库存货值数据
5. 后端计算FBA库存货值的合计数据
6. 后端返回分页的FBA库存货值数据给前端
7. 前端展示FBA库存货值数据，包括合计行

### 4.3 库存货值导出流程

1. 前端点击导出按钮
2. 前端调用导出API接口（如`downLoadInvCost`）
3. 后端接收请求，解析导出条件
4. 后端调用相应的服务方法查询所有符合条件的库存货值数据
5. 后端使用`SXSSFWorkbook`生成Excel文件
6. 后端设置响应头，返回Excel文件流
7. 前端接收文件流，下载Excel文件

## 5. 数据模型

### 5.1 核心数据传输对象（DTO）

**InventoryValueReportDTO**：库存货值报表DTO
- `page`：分页信息
- `warehouseid`：仓库ID
- `sku`：SKU
- `byday`：库存日期
- `today`：是否为当天

### 5.2 数据结构

**本地仓库存货值数据结构**：
- `image`：产品图片
- `name`：产品名称
- `sku`：SKU
- `quantity`：现有库存
- `inbound`：待入库库存
- `price`：采购成本单价
- `totalprice`：总货值
- `inpasscost`：在途货值
- `inpasspay`：采购待付款
- `otherFeer`：其它成本
- `remark`：备注
- `summary`：合计数据

**FBA仓库存货值数据结构**：
- `image`：产品图片
- `name`：产品名称
- `sku`：SKU
- `fulfillable`：可用库存
- `reserved`：预留库存
- `inbound`：待入库库存
- `quantity`：总库存
- `price`：采购单价
- `totalprice`：总货值
- `otherFeer`：其它费用
- `firstLegCharges`：头程运费
- `remark`：备注
- `summary`：合计数据
