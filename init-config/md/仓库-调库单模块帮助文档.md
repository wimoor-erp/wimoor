# 仓库-调库单模块帮助文档

## 1. 模块概述

调库单模块是Wimoor系统中的核心仓库管理模块之一，主要用于实现不同仓库之间的商品调拨功能。该模块支持从一个仓库向另一个仓库转移商品库存，包含完整的调库流程管理，如创建、审核、发货、收货等环节。通过该模块，用户可以实现仓库间的库存平衡、商品转移和库存调整等操作，确保库存管理的灵活性和准确性。

## 2. 功能结构

调库单模块包含以下主要功能：

| 功能模块 | 子功能 | 描述 |
|---------|--------|------|
| 调库单管理 | 调库单列表 | 查看系统中所有调库单 |
| 调库单管理 | 添加调库单 | 创建新的调库单 |
| 调库单管理 | 编辑调库单 | 修改未提交的调库单 |
| 调库单管理 | 导入调库单 | 通过Excel批量导入调库单 |
| 调库单管理 | 调库单详情 | 查看调库单的详细信息 |
| 调库流程管理 | 提交调库单 | 提交调库单等待审核 |
| 调库流程管理 | 审核调库单 | 审核并通过调库单 |
| 调库流程管理 | 发货 | 从调出仓库发货 |
| 调库流程管理 | 收货 | 调入仓库确认收货 |
| 调库流程管理 | 驳回/取消 | 驳回或取消调库单 |
| 调库流程管理 | 直接完成 | 一步完成整个调库流程 |

## 3. 操作指南

### 3.1 调库单管理

#### 3.1.1 查看调库单列表

1. **操作步骤**：
   - 进入仓库管理模块，点击「调库单」菜单
   - 系统显示调库单列表，包含调库单编码、调出/调入仓库、状态、申请人等信息
   - 可以通过标签页切换不同状态的调库单
   - 可以通过搜索框搜索调库编码、SKU或商品名称
   - 可以通过日期选择器筛选特定时间段的调库单

2. **状态说明**：
   - 未提交：调库单已创建但未提交审核
   - 待审核：调库单已提交等待审核
   - 配货中：调库单已通过审核，准备发货
   - 已发货：调库单已发货，等待收货
   - 已完成：调库单已完成，库存已调整
   - 已驳回：调库单被驳回

#### 3.1.2 添加调库单

1. **操作步骤**：
   - 点击「添加调库单」按钮
   - 填写调库单基本信息：
     - 调出仓库：选择商品出库的仓库
     - 调入仓库：选择商品入库的仓库
     - 预计到货日期：设置商品预计到达调入仓库的日期
     - 操作类型：选择调库类型（如正常调库、残次品调拨等）
     - 备注：填写调库相关说明
   - 点击「添加商品」按钮，选择需要调库的商品并设置调库数量
   - 点击「保存」按钮暂存调库单，或点击「提交」按钮提交审核，或点击「直接完成」按钮一步完成整个调库流程

2. **注意事项**：
   - 调出仓库和调入仓库不能相同
   - 调库商品数量不能为负数
   - 调库商品数量不能超过调出仓库的可用库存
   - 当调出和调入仓库地址不同时，建议由调入仓库负责人审核调库单

#### 3.1.3 编辑调库单

1. **操作步骤**：
   - 在调库单列表中，点击状态为「未提交」的调库单
   - 进入调库单详情页面，点击「编辑」按钮
   - 修改调库单信息和商品列表
   - 点击「保存」或「提交」按钮保存修改

2. **注意事项**：
   - 只有状态为「未提交」的调库单可以编辑
   - 已提交或已审核的调库单不能编辑

#### 3.1.4 导入调库单

1. **操作步骤**：
   - 点击「导入」按钮
   - 在弹出的对话框中，点击「下载模板」按钮获取Excel模板
   - 按照模板格式填写调库信息，包括调出仓库、调入仓库、调库商品SKU和数量
   - 点击「点击上传」按钮，选择填写好的Excel文件
   - 点击「上传文件」按钮完成导入

2. **注意事项**：
   - 导入文件大小不能超过5MB
   - 模板中的SKU必须在系统中已存在
   - 调库数量不能超过调出仓库的可用库存

### 3.2 调库流程管理

#### 3.2.1 提交调库单

1. **操作步骤**：
   - 创建调库单后，点击「提交」按钮
   - 调库单状态变为「待审核」
   - 等待审核人员审核

#### 3.2.2 审核调库单

1. **操作步骤**：
   - 在调库单列表中，点击状态为「待审核」的调库单
   - 进入调库单详情页面，点击「通过」按钮
   - 调库单状态变为「配货中」

#### 3.2.3 发货

1. **操作步骤**：
   - 在调库单列表中，点击状态为「配货中」的调库单
   - 进入调库单详情页面，点击「发货」按钮
   - 调库单状态变为「已发货」
   - 系统自动减少调出仓库的库存

#### 3.2.4 收货

1. **操作步骤**：
   - 在调库单列表中，点击状态为「已发货」的调库单
   - 进入调库单详情页面，点击「收货」按钮
   - 调库单状态变为「已完成」
   - 系统自动增加调入仓库的库存

#### 3.2.5 驳回/取消调库单

1. **操作步骤**：
   - 在调库单列表中，点击需要驳回/取消的调库单
   - 进入调库单详情页面，点击「驳回」（待审核状态）或「取消」（配货中/已发货状态）按钮
   - 调库单状态变为「已驳回」
   - 系统自动回滚相关库存操作

#### 3.2.6 直接完成

1. **操作步骤**：
   - 创建调库单后，点击「直接完成」按钮
   - 系统一步完成提交、审核、发货、收货整个流程
   - 调库单状态直接变为「已完成」
   - 系统自动调整相关仓库的库存

2. **适用场景**：
   - 同地址仓库之间的调库
   - 紧急调库需求
   - 无需多级审核的内部调库

## 4. 调库单状态流转

调库单的状态流转如下：

1. **创建** → **未提交**：用户创建调库单但未提交
2. **未提交** → **待审核**：用户提交调库单
3. **待审核** → **配货中**：审核人员通过调库单
4. **配货中** → **已发货**：调出仓库发货
5. **已发货** → **已完成**：调入仓库收货
6. **任何状态** → **已驳回**：审核人员驳回或用户取消调库单

## 5. 常见问题

### 5.1 调库单无法提交

**问题**：点击提交按钮后提示提交失败
**解决方案**：
- 检查调出仓库和调入仓库是否选择正确
- 检查调库商品数量是否为正数
- 检查调库商品数量是否超过调出仓库的可用库存
- 检查必填字段是否已填写完整

### 5.2 调库单无法审核通过

**问题**：点击通过按钮后提示审核失败
**解决方案**：
- 检查调库商品在调出仓库的库存是否充足
- 检查调库单信息是否完整
- 检查用户是否有审核权限

### 5.3 调库后库存未更新

**问题**：完成调库流程后，库存数量未更新
**解决方案**：
- 检查调库单是否已完成所有流程步骤
- 检查调库单状态是否为「已完成」
- 刷新页面或重新登录系统后查看库存

### 5.4 导入调库单失败

**问题**：上传Excel文件后提示导入失败
**解决方案**：
- 检查Excel文件格式是否符合模板要求
- 检查文件大小是否超过5MB
- 检查SKU是否在系统中已存在
- 检查调库数量是否超过调出仓库的可用库存
- 检查仓库名称是否与系统中的仓库名称完全匹配

### 5.5 调库单无法删除

**问题**：无法删除调库单
**解决方案**：
- 只有状态为「未提交」的调库单可以删除
- 已提交或已审核的调库单只能通过驳回/取消操作终止

## 7. 最佳实践

1. **调库前检查**：在创建调库单前，建议先检查调出仓库的库存是否充足，确保调库操作能够顺利完成。

2. **合理设置操作类型**：根据实际调库需求选择正确的操作类型，如正常调库、残次品调拨等，便于后续的库存分析和统计。

3. **跨地址调库注意事项**：当调出和调入仓库地址不同时，建议由调入仓库负责人审核调库单，确保能够正常收货。

4. **批量调库**：对于大量商品的调库需求，建议使用Excel导入功能，提高操作效率。

5. **调库流程跟踪**：及时关注调库单的状态变化，确保调库流程能够及时完成，避免库存长时间处于中间状态。

6. **调库记录查询**：调库完成后，系统会记录完整的调库历史，便于后续的库存追溯和审计。
