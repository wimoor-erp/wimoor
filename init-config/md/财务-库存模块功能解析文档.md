# 财务-库存模块功能解析文档

## 1. 模块架构

### 1.1 技术栈

- **前端**：Vue 3 + Composition API + Element Plus + Axios
- **后端**：Spring Boot 2.x + MyBatis-Plus + Spring Security
- **数据库**：MySQL
- **API设计**：RESTful API
- **数据传输**：JSON格式
- **Excel导出**：SXSSFWorkbook（支持大数量数据导出）

## 2. 前端实现

### 2.1 库存日报表组件（daily/index.vue）

#### 核心功能
- 库存每日变化查询和展示
- 支持按仓库、日期、SKU筛选
- 库存日报表数据导出

#### 关键代码分析

**模板结构**：
- 使用Element Plus的Cascader组件实现仓库选择
- 使用自定义Datepicker组件实现日期选择
- 使用GlobalTable组件展示库存数据
- 支持表格排序和合计功能

**数据加载流程**：
1. 组件挂载时，通过`warehouseApi.getWarehouseList()`获取仓库列表
2. 选择仓库后，调用`handleQuery()`方法加载数据
3. `handleQuery()`方法首先调用`inventoryRptApi.getInvDayField()`获取日期字段
4. 然后调用`globalTable.value.loadTable()`加载表格数据
5. `loadTableData()`方法通过`inventoryRptApi.getInvDayDetail()`获取库存日报表数据

**导出功能**：
- 点击导出按钮时，调用`downloadExcel()`方法
- `downloadExcel()`方法调用`inventoryRptApi.getInvDayDetailExport()`导出数据

### 2.2 基础库存报表组件（report/base/index.vue）

#### 核心功能
- 库存数据查询和展示
- 支持按供应商、仓库、产品分类等条件筛选
- 库存状态管理（待入库、可用库存、待出库）
- 库存数据导出
- 库存数据修复功能

#### 关键代码分析

**模板结构**：
- 使用Supplier组件选择供应商
- 使用Cascader组件选择仓库
- 使用GlobalTable组件展示库存数据
- 支持表格排序和合计功能
- 待入库和待出库数据支持悬停查看详情

**数据加载流程**：
1. 组件挂载时，通过`warehouseApi.getWarehouseList()`获取仓库列表
2. 选择仓库后，调用`handleQuery()`方法加载数据
3. `handleQuery()`方法调用`globalTable.value.loadTable()`加载表格数据
4. `loadTableData()`方法通过`inventoryApi.getWarehouse()`获取库存数据

**库存详情查询**：
- 鼠标悬停在待入库数据上时，调用`findInboundDetail()`方法获取入库详情
- 鼠标悬停在待出库数据上时，调用`findOutboundDetail()`方法获取出库详情

**库存修复功能**：
- 在详情弹窗中，点击刷新图标调用`handleRefreshInbound()`方法修复库存数据
- `handleRefreshInbound()`方法调用`inventoryApi.refreshInventory()`修复库存

**导出功能**：
- 点击导出按钮时，调用`downloadExcel()`方法
- `downloadExcel()`方法调用`inventoryApi.getWarehouseExport()`导出数据

## 3. 后端实现

### 3.1 库存控制器（InventoryController.java）

#### 核心功能
- 库存数据查询
- 库存数据导出
- 库存详情查询
- 库存成本查询

#### 关键API接口

**1. 获取仓库库存列表**
- **接口**：`POST /api/v1/inventory/getWarehouse`
- **功能**：根据条件查询库存数据
- **参数**：`InventoryQueryDTO`（包含仓库ID、SKU、分类等查询条件）
- **返回**：`IPage<Map<String, Object>>`（分页的库存数据）

**2. 导出仓库库存**
- **接口**：`POST /api/v1/inventory/getWarehouseExport`
- **功能**：导出库存数据为Excel文件
- **参数**：`InventoryQueryDTO`（包含导出条件）
- **返回**：Excel文件流

**3. 查询入库详情**
- **接口**：`GET /api/v1/inventory/findInboundDetail`
- **功能**：查询指定产品在指定仓库的入库详情
- **参数**：`mid`（产品ID）、`warehouseid`（仓库ID）
- **返回**：`List<Map<String, Object>>`（入库详情列表）

**4. 查询出库详情**
- **接口**：`GET /api/v1/inventory/findOutboundDetail`
- **功能**：查询指定产品在指定仓库的出库详情
- **参数**：`mid`（产品ID）、`warehouseid`（仓库ID）
- **返回**：`List<Map<String, Object>>`（出库详情列表）

**5. 查询库存成本**
- **接口**：`GET /api/v1/inventory/findInventoryNowCostByShopId`
- **功能**：查询指定店铺的库存成本
- **参数**：`shopid`（店铺ID）
- **返回**：`List<Map<String, Object>>`（库存成本列表）

### 3.2 库存服务（IInventoryService）

#### 核心方法
- `findLocalInventory()`：查询本地库存数据
- `findInboundDetail()`：查询入库详情
- `findOutboundDetail()`：查询出库详情
- `findSum()`：计算库存汇总数据
- `getExcelBookInventoryReport()`：生成库存报表Excel

## 4. 业务逻辑流程

### 4.1 库存查询流程

1. 前端选择查询条件（仓库、SKU、日期等）
2. 前端调用后端API接口（如`getWarehouse`）
3. 后端接收请求，解析参数
4. 后端调用`inventoryService.findLocalInventory()`查询库存数据
5. 后端计算库存汇总数据（如总待入库、总可用库存、总待出库）
6. 后端返回分页的库存数据给前端
7. 前端展示库存数据，包括合计行

### 4.2 库存日报表查询流程

1. 前端选择查询条件（仓库、日期、SKU）
2. 前端调用`inventoryRptApi.getInvDayField()`获取日期字段
3. 前端调用`inventoryRptApi.getInvDayDetail()`获取库存日报表数据
4. 后端查询指定日期范围内的库存变化数据
5. 后端返回库存日报表数据给前端
6. 前端展示库存日报表数据，包括合计行

### 4.3 库存数据导出流程

1. 前端点击导出按钮
2. 前端调用后端导出API接口（如`getWarehouseExport`）
3. 后端接收请求，解析导出条件
4. 后端查询符合条件的库存数据
5. 后端使用`SXSSFWorkbook`生成Excel文件
6. 后端设置响应头，返回Excel文件流
7. 前端接收文件流，下载Excel文件

### 4.4 库存状态修复流程

1. 前端鼠标悬停在待入库或待出库数据上
2. 前端调用`findInboundDetail()`或`findOutboundDetail()`查询详情
3. 前端点击详情弹窗中的刷新图标
4. 前端调用`inventoryApi.refreshInventory()`修复库存
5. 后端接收请求，修复库存数据
6. 后端返回修复结果
7. 前端显示修复成功提示，并重新加载数据

## 5. 数据模型

### 5.1 核心实体类

**Inventory**：库存实体
- `id`：库存ID
- `materialid`：产品ID
- `warehouseid`：仓库ID
- `fulfillable`：可用库存
- `inbound`：待入库
- `outbound`：待出库
- `shopid`：店铺ID
- `creator`：创建人
- `updator`：更新人

**Warehouse**：仓库实体
- `id`：仓库ID
- `name`：仓库名称
- `type`：仓库类型
- `shopid`：店铺ID
- `address`：仓库地址

**Material**：产品实体
- `id`：产品ID
- `sku`：SKU
- `name`：产品名称
- `issfg`：是否组合产品
- `shopid`：店铺ID

### 5.2 数据传输对象（DTO）

**InventoryQueryDTO**：库存查询DTO
- `page`：分页信息
- `warehouseid`：仓库ID
- `sku`：SKU
- `category`：产品分类
- `ftypes`：产品类型
- `supplierid`：供应商ID
- `search`：搜索关键词
- `hasinv`：是否仅查看有库存产品
- `notlike`：是否精确查询
