# 店铺利润模块（按店铺统计）帮助文档

## 1. 模块概述

店铺利润模块是Wimoor系统中用于按店铺和站点分别统计各项店铺费用和利润的核心财务管理功能。该模块支持按店铺和站点分类汇总销售额、销售佣金、FBA费用、退款金额、仓储费、广告费、采购成本等费用，并计算出店铺利润，为企业的财务分析和决策提供有力支持。

### 1.1 核心功能

- **店铺和站点分类统计**：按店铺和站点分别统计各项费用和利润
- **多币种支持**：支持人民币(CNY)、美元(USD)、站点币种三种币种切换
- **日期范围选择**：支持自定义日期范围查询
- **数据导出**：支持导出店铺利润报表
- **结算日期和转账日期切换**：支持按结算日期或转账日期统计
- **数据汇总**：自动汇总店铺和站点的各项费用和利润

### 1.2 技术架构

- **前端**：Vue 3、Composition API、Element Plus、Axios
- **后端**：Spring Boot 2.x、MyBatis-Plus、Spring Security
- **数据库**：MySQL
- **API风格**：RESTful API
- **数据传输**：JSON格式
- **汇率转换**：内置汇率转换服务

```vue
<script setup>
import { ref, reactive, onMounted } from 'vue';
import Group from '@/components/header/group.vue';
import Market from '@/components/header/market.vue';
import Datepicker from '@/components/header/datepicker.vue';
import settlementAccRptApi from '@/api/amazon/finances/settlementAccRptApi.js';
import { ElMessage } from 'element-plus';

// 状态管理
const queryParams = reactive({
  groupid: '',
  marketplaceid: '',
  startDate: '',
  endDate: '',
  currencyType: 'CNY',
  dateType: 'settlement'
});

const tableData = ref([]);
const pagination = reactive({
  currentPage: 1,
  pageSize: 20,
  total: 0
});

// 查询数据
async function handleQuery() {
  try {
    const res = await settlementAccRptApi.getSettlementOverAll({
      ...queryParams,
      page: pagination.currentPage,
      limit: pagination.pageSize
    });
    if (res.code === 200) {
      tableData.value = res.data.records;
      pagination.total = res.data.total;
    } else {
      ElMessage.error(res.msg);
    }
  } catch (error) {
    ElMessage.error('查询失败');
  }
}

// 导出数据
async function handleDownload() {
  try {
    const res = await settlementAccRptApi.downloadSettlementOverAll(queryParams);
    if (res.code === 200) {
      // 处理文件下载
      const blob = new Blob([res.data], { type: 'application/vnd.ms-excel' });
      const url = URL.createObjectURL(blob);
      const link = document.createElement('a');
      link.href = url;
      link.download = `店铺利润报表_${new Date().toISOString().slice(0, 10)}.xlsx`;
      link.click();
      URL.revokeObjectURL(url);
    } else {
      ElMessage.error(res.msg);
    }
  } catch (error) {
    ElMessage.error('导出失败');
  }
}

// 分页处理
function handleSizeChange(size) {
  pagination.pageSize = size;
  handleQuery();
}

function handleCurrentChange(current) {
  pagination.currentPage = current;
  handleQuery();
}

// 初始化
onMounted(() => {
  // 设置默认日期范围为最近7天
  const endDate = new Date();
  const startDate = new Date();
  startDate.setDate(startDate.getDate() - 7);
  queryParams.startDate = startDate.toISOString().slice(0, 10);
  queryParams.endDate = endDate.toISOString().slice(0, 10);
});
</script>
```

```java
@RestController
@RequestMapping("/api/v1/settlementReport")
public class AmzSettlementReportController {
    
    @Autowired
    private IAmzSettlementReportService amzSettlementReportService;
    
    @PostMapping("/getSettlementOverAll")
    public Result<?> getSettlementOverAll(@RequestBody AmzSettlementDTO dto, HttpServletRequest request) {
        // 获取用户信息
        SysUser user = LoginUser.getUser();
        // 处理查询参数
        if (dto.getGroupid() == null || dto.getGroupid().isEmpty()) {
            return Result.failed("店铺不能为空");
        }
        // 调用服务层方法获取数据
        Page<Map<String, Object>> page = amzSettlementReportService.getSettlementGroupOverAll(dto, user);
        return Result.success(page);
    }
    
    @PostMapping("/downloadSettlementOverAll")
    public void downloadSettlementOverAll(@RequestBody AmzSettlementDTO dto, HttpServletRequest request, HttpServletResponse response) {
        // 获取用户信息
        SysUser user = LoginUser.getUser();
        // 调用服务层方法导出数据
        amzSettlementReportService.downloadSettlementGroupOverAll(dto, user, response);
    }
}
```

## 4. 数据流程

### 4.1 店铺利润数据查询流程

1. **前端发起请求**：用户选择店铺、站点、日期范围、币种等参数，点击查询按钮
2. **参数处理**：前端处理参数，确保参数完整性
3. **API调用**：前端调用`getSettlementOverAll` API发送POST请求
4. **后端处理**：
   - 控制器接收请求，获取用户信息
   - 处理各种查询参数，构建查询条件
   - 调用服务层方法`getSettlementGroupOverAll`获取店铺利润数据
   - 服务层调用Mapper层查询数据库
   - 计算各项费用和利润
   - 控制器返回数据
5. **前端处理**：
   - 接收响应数据
   - 格式化金额数据
   - 更新表格数据
   - 显示店铺和站点的利润数据

### 4.2 店铺利润数据导出流程

1. **前端发起请求**：用户设置查询参数，点击导出按钮
2. **参数处理**：前端处理参数，确保参数完整性
3. **API调用**：前端调用`downloadSettlementOverAll` API发送POST请求
4. **后端处理**：
   - 控制器接收请求，获取用户信息
   - 调用服务层方法`downloadSettlementGroupOverAll`导出数据
   - 服务层查询数据，生成Excel文件
   - 通过HttpServletResponse输出Excel文件流
5. **前端处理**：
   - 接收文件流
   - 创建Blob对象
   - 生成下载链接
   - 触发文件下载

## 5. 操作步骤

### 5.1 进入店铺利润模块

1. 登录Wimoor系统
2. 在左侧导航栏中，依次点击 **财务** → **平台账目** → **店铺利润**
3. 进入店铺利润模块页面

### 5.2 选择店铺和站点

1. 在页面顶部的店铺选择器中，点击下拉菜单
2. 选择您要查看的店铺
3. 选择对应的站点（如美国、英国、德国等）
4. 系统会自动加载该店铺和站点的利润数据

### 5.3 设置查询条件

1. **日期范围**：
   - 点击日期选择器，选择您要查询的开始日期和结束日期
   - 点击确定按钮，系统会根据选择的日期范围更新数据
   - 默认显示最近7天的数据

2. **币种选择**：
   - 点击币种下拉菜单，选择您要查看的币种（人民币、美元、站点币种）
   - 系统会根据选择的币种显示相应的金额

3. **日期类型**：
   - 点击日期类型下拉菜单，选择结算日期或转账日期
   - 系统会根据选择的日期类型统计数据

### 5.4 查看利润数据

1. 点击查询按钮，系统会根据设置的条件显示店铺利润数据
2. 表格会显示店铺、站点、销售额、退款金额、FBA费用、销售佣金、仓储费、广告费、采购成本、利润、利润率等数据
3. 您可以点击表格列标题进行排序
4. 您可以通过分页控件查看更多数据

### 5.5 导出利润报表

1. 设置好查询条件
2. 点击导出按钮
3. 系统会生成Excel格式的利润报表并自动下载
4. 您可以在下载文件夹中找到该报表

## 6. 常见问题及解决方案

### 6.1 数据不显示

**问题**：选择店铺和站点后，表格中没有显示数据

**解决方案**：
- 检查日期范围是否正确，确保选择的日期范围内有数据
- 检查店铺和站点是否正确选择
- 检查网络连接是否正常
- 尝试刷新页面后重新查询

### 6.2 导出失败

**问题**：点击导出按钮后，报表没有下载

**解决方案**：
- 检查浏览器是否阻止了下载
- 检查网络连接是否正常
- 尝试缩小日期范围，减少数据量
- 联系系统管理员

### 6.3 数据不准确

**问题**：显示的利润数据与预期不符

**解决方案**：
- 检查日期范围是否正确
- 检查币种选择是否正确
- 检查日期类型（结算日期/转账日期）选择是否正确
- 联系系统管理员检查数据同步情况

### 6.4 页面加载缓慢

**问题**：页面加载或查询数据时速度较慢

**解决方案**：
- 尝试缩小日期范围，减少数据量
- 检查网络连接是否正常
- 避免在系统高峰期使用
- 联系系统管理员优化系统性能
