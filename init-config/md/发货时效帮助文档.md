# 发货时效模块使用指南

## 1. 功能概述

发货时效模块是Wimoor系统中用于监控和分析商品发货各环节时间节点的重要功能。该模块能够追踪从计划创建到最终完成的整个发货流程，计算各个环节的时效，帮助用户识别发货流程中的瓶颈，优化发货效率。

### 1.1 模块价值

- **流程监控**：实时监控发货流程的各个时间节点，确保流程顺畅
- **瓶颈识别**：通过时效数据识别发货流程中的瓶颈环节，有针对性地进行优化
- **效率提升**：通过数据分析，找出影响发货效率的因素，提升整体发货速度
- **决策支持**：为物流渠道选择、仓库布局优化等决策提供数据支持
- **成本控制**：通过优化发货时效，减少库存积压，降低仓储成本

### 1.2 适用场景

- **日常发货管理**：监控日常发货流程的时间节点和时效
- **物流渠道评估**：评估不同物流渠道的时效表现，选择最优渠道
- **仓库效率分析**：分析不同仓库的发货效率，优化仓库运营
- **季度/年度总结**：对一定时期内的发货时效进行总结分析，制定改进计划

## 3. 界面介绍

### 3.1 主界面布局

发货时效模块主界面包含以下主要部分：

1. **筛选条件区域**：位于页面顶部，包含以下筛选器：
   - **店铺分组选择**：下拉菜单，选择特定的店铺分组
   - **日期范围选择**：日期选择器，选择起始日期和结束日期
   - **SKU/MSKU搜索框**：可切换搜索类型（SKU或MSKU），输入编码进行搜索

2. **操作按钮区域**：位于筛选条件区域右侧，包含以下按钮：
   - **导出**：将当前筛选条件下的数据导出为Excel文件

3. **数据表格区域**：位于页面中部，占页面主要区域，展示发货时效相关的详细数据

### 3.2 数据表格字段详细说明

| 字段名称 | 详细说明 | 数据来源 |
|---------|---------|----------|
| **图片** | 商品的主图，帮助快速识别商品 | 商品信息表 |
| **商品信息** | 包含商品名称、平台SKU和本地SKU | 商品信息表和本地物料表 |
| **店铺站点** | 店铺名称、站点名称和货件号，点击货件号可跳转到货件详情页 | 店铺表、站点表和货件表 |
| **发货仓库** | 仓库名称和SKU发货数量 | 仓库表和货件商品表 |
| **计划时间** | 计划创建时间和审核时间，反映计划制定和审批的及时性 | 入库计划表 |
| **配货时间** | 配货时间、装箱时间和打标时间，反映仓库操作效率 | 入库流程记录表 |
| **出库时间** | 出库时间和收货时间，反映物流开始和亚马逊仓库接收的时间 | 货件表和货件商品表 |
| **物流信息** | 物流区域、运输方式、渠道名称和渠道别名，反映物流服务提供商和服务类型 | 运输信息表和渠道表 |

| **收货时效** | 从配货完成到货件关闭的时间（天数），反映物流运输和亚马逊处理效率 | 系统计算（货件关闭时间 - 配货完成时间） |
| **总时效** | 从计划创建到货件关闭的总时间（天数），反映整个发货流程的总效率 | 系统计算（货件关闭时间 - 计划创建时间） |

## 4. 操作指南

### 4.1 数据筛选

#### 4.1.1 店铺分组筛选

1. **点击店铺分组下拉框**：位于页面顶部左侧
2. **选择店铺分组**：从下拉列表中选择一个或多个店铺分组
3. **自动加载数据**：选择完成后，系统会自动加载该分组下的所有发货时效数据

#### 4.1.2 日期范围筛选

1. **点击日期选择器**：位于店铺分组下拉框右侧
2. **选择日期范围**：
   - 点击起始日期，选择开始日期
   - 点击结束日期，选择结束日期
   - 或选择预设的日期范围（如最近7天、最近30天、本月、上月等）
3. **确认选择**：点击确定按钮，系统会显示该日期范围内的发货数据

#### 4.1.3 SKU/MSKU搜索

1. **切换搜索类型**：点击搜索框左侧的下拉菜单，选择搜索类型（SKU或MSKU）
2. **输入搜索关键词**：在搜索框中输入完整或部分SKU/MSKU编码
3. **执行搜索**：
   - 点击搜索框右侧的搜索按钮，或
   - 按键盘上的回车键
4. **查看结果**：系统会显示匹配的SKU/MSKU的发货时效数据

### 4.2 数据导出

1. **设置筛选条件**：根据需要设置店铺分组、日期范围和搜索条件
2. **点击导出按钮**：位于页面顶部右侧，蓝色"导出"按钮
3. **等待导出完成**：
   - 系统会显示导出进度提示
   - 导出时间取决于数据量大小，一般为3-30秒
4. **下载文件**：导出完成后，系统会自动下载Excel文件

### 4.3 数据查看技巧

#### 4.3.1 多维度分析

1. **按店铺分析**：选择不同的店铺分组，比较不同店铺的发货时效
2. **按时间段分析**：选择不同的日期范围，分析发货时效的变化趋势
3. **按SKU分析**：搜索特定SKU，分析该SKU的发货时效表现
4. **按物流渠道分析**：通过物流信息字段，分析不同物流渠道的时效表现

#### 4.3.2 异常数据识别

- **发货时效异常**：发货时效明显高于平均值，可能是仓库操作效率低
- **收货时效异常**：收货时效明显高于平均值，可能是物流渠道选择不当或海关查验
- **总时效异常**：总时效明显高于平均值，需要分析是发货环节还是收货环节出现问题
- **时间节点缺失**：某个时间节点为空，可能是流程未完成或数据未同步

## 5. 数据说明

### 5.1 时间节点说明

| 时间节点 | 详细说明 | 数据录入方式 |
|---------|---------|------------|
| **计划时间** | 发货计划的创建时间，标志着发货流程的开始 | 系统自动记录，计划创建时生成 |
| **审核时间** | 发货计划的审核通过时间，标志着计划正式生效 | 系统自动记录，计划审核通过时生成 |
| **配货时间** | 开始配货的时间，标志着仓库操作的开始 | 系统自动记录，配货操作开始时生成 |
| **装箱时间** | 完成装箱的时间，标志着商品包装完成 | 系统自动记录，装箱操作完成时生成 |
| **打标时间** | 完成打标的时间，标志着商品标识完成 | 系统自动记录，打标操作完成时生成 |
| **出库时间** | 货件出库的时间，标志着商品离开仓库 | 系统自动记录，出库操作完成时生成 |
| **收货时间** | 亚马逊仓库收货的时间，标志着商品到达目的地仓库 | 系统自动记录，亚马逊确认收货时生成 |
| **完成时间** | 货件状态变为完成的时间，标志着整个发货流程的结束 | 系统自动记录，货件状态变为完成时生成 |

### 5.2 时效计算规则

| 时效类型 | 计算方法 | 单位 | 影响因素 |
|---------|---------|------|---------|
| **发货时效** | 配货完成时间 - 审核时间 | 天数 | 仓库操作效率、商品类型、库存状态 |
| **收货时效** | 完成时间 - 配货完成时间 | 天数 | 物流渠道、运输距离、海关查验、天气因素 |
| **总时效** | 完成时间 - 计划时间 | 天数 | 上述所有因素的综合影响 |

### 5.3 物流信息说明

| 字段名称 | 详细说明 | 用途 |
|---------|---------|------|
| **物流区域** | 物流服务的区域范围，如北美、欧洲、亚太等 | 分析不同区域的物流时效 |
| **运输方式** | 采用的运输方式，如海运、空运、快递等 | 分析不同运输方式的时效表现 |
| **渠道名称** | 物流渠道的标准名称，如UPS、FedEx、DHL等 | 评估不同物流服务商的表现 |
| **渠道别名** | 物流渠道的自定义别名，方便内部识别 | 快速识别常用物流渠道 |

### 5.4 数据更新频率

- **实时数据**：时间节点数据实时更新
- **时效计算**：时间节点更新后，时效数据自动重新计算
- **数据同步**：与亚马逊系统的数据同步频率为每4小时一次

## 6. 常见问题

### 6.1 数据问题

#### 6.1.1 为什么某些时间节点为空？

可能的原因：
- **流程未完成**：该环节尚未执行完成，因此没有时间记录
- **数据未同步**：系统与亚马逊之间的数据同步存在延迟
- **操作未记录**：相关操作未在系统中记录，导致时间节点缺失
- **权限问题**：当前用户没有查看某些时间节点的权限

解决方法：
- 等待流程完成后再查看
- 手动触发数据同步
- 检查相关操作是否已正确记录
- 联系系统管理员检查权限设置

#### 6.1.2 为什么时效数据与实际不符？

可能的原因：
- **时间节点记录不准确**：某些时间节点的记录时间与实际操作时间存在偏差
- **数据同步延迟**：亚马逊系统的数据尚未同步到Wimoor系统
- **计算逻辑理解差异**：对时效计算逻辑的理解与系统实际计算方式存在差异

解决方法：
- 检查时间节点记录是否准确
- 手动触发数据同步
- 参考本文档中的时效计算规则，理解系统计算逻辑

### 6.2 操作问题

#### 6.2.1 如何快速找到特定SKU的时效数据？

解决方法：
1. 在搜索框左侧选择"SKU"搜索类型
2. 在搜索框中输入完整的SKU编码
3. 点击搜索按钮或按回车键
4. 系统会显示该SKU的所有发货时效记录

#### 6.2.2 如何导出特定时间段的数据？

解决方法：
1. 在日期选择器中选择起始日期和结束日期
2. 点击确定按钮，等待数据加载完成
3. 点击导出按钮，系统会导出该时间段内的数据

### 6.3 分析问题

#### 6.3.1 如何分析不同物流渠道的时效表现？

解决方法：
1. 设置合适的日期范围（建议至少30天）
2. 导出数据到Excel文件
3. 在Excel中按"物流渠道"字段进行分组
4. 计算每个物流渠道的平均发货时效、平均收货时效和平均总时效
5. 比较不同物流渠道的时效表现，选择最优渠道

#### 6.3.2 如何识别发货流程中的瓶颈环节？

解决方法：
1. 导出一段时间内的发货时效数据
2. 在Excel中计算每个环节的平均时间
3. 识别平均时间较长的环节
4. 分析该环节耗时较长的原因
5. 制定针对性的优化措施

## 7. 注意事项

### 7.1 数据准确性

- **及时记录操作**：确保所有发货相关操作都在系统中及时记录
- **定期同步数据**：定期手动触发与亚马逊系统的数据同步，确保数据准确性
- **核对时间节点**：定期核对系统记录的时间节点与实际操作时间是否一致

### 7.2 系统使用

- **合理设置筛选条件**：避免设置过大的日期范围，导致数据加载缓慢
- **优化搜索方式**：使用精确的SKU/MSKU编码进行搜索，提高搜索效率
- **及时导出数据**：定期导出重要数据，备份到本地，防止数据丢失

### 7.3 最佳实践

- **建立时效基准**：为不同物流渠道、不同仓库建立合理的时效基准
- **设置时效预警**：当发货时效超过基准值时，及时预警
- **定期分析**：每周/每月对发货时效数据进行分析，找出改进空间
- **持续优化**：根据分析结果，持续优化发货流程和物流选择
- **培训员工**：对仓库员工进行培训，提高操作效率，减少人为延误

### 7.4 系统限制

- **数据量限制**：单次导出的数据量最大为10,000条
- **历史数据**：系统默认保存最近24个月的发货时效数据
- **搜索限制**：SKU/MSKU搜索支持部分匹配，但建议使用完整编码以获得准确结果

| **装箱** | 将商品装入包装箱的过程 |
| **打标** | 在商品或包装上粘贴亚马逊物流标签的过程 |
| **出库** | 商品离开仓库的过程 |
| **收货** | 亚马逊仓库接收商品的过程 |
