# 店铺利润模块操作指南

## 1. 模块概述

店铺利润模块是Wimoor系统中用于查看和分析店铺及站点利润情况的财务管理工具。通过该模块，您可以按店铺和站点分别查看销售额、销售佣金、FBA费用、退款金额、仓储费、广告费、采购成本等费用，并计算出店铺利润，帮助您全面了解店铺的经营状况。

## 2. 功能介绍

- **店铺和站点分类统计**：按店铺和站点分别统计各项费用和利润，清晰展示数据层级关系
- **多币种支持**：支持人民币(CNY)、美元(USD)、站点币种三种币种切换
- **日期范围选择**：支持自定义日期范围查询，默认显示最近7天的数据
- **结算日期切换**：支持按结算日期或转账日期统计数据
- **数据导出**：支持导出店铺利润报表为Excel文件

## 3. 操作步骤

### 3.1 进入店铺利润模块

1. 登录Wimoor系统
2. 在左侧导航栏中，依次点击 **财务** → **平台账目** → **店铺利润**
3. 进入店铺利润模块页面

### 3.2 选择店铺和站点

1. 在页面顶部的店铺选择器中，点击下拉菜单
2. 选择您要查看的店铺
3. 选择对应的站点（如美国、英国、德国等）
4. 系统会自动加载该店铺和站点的利润数据

### 3.3 设置查询条件

1. **日期范围**：
   - 点击日期选择器，选择您要查询的开始日期和结束日期
   - 点击确定按钮，系统会根据选择的日期范围更新数据

2. **结算日期/转账日期**：
   - 在日期选择器左侧的下拉菜单中，选择"结算日期"或"转账日期"
   - 系统会根据选择的日期类型更新数据

3. **币种选择**：
   - 在日期选择器右侧的下拉菜单中，选择您要查看的币种（人民币、美元或站点币种）
   - 系统会根据选择的币种转换并显示数据

### 3.4 查看利润数据

1. 数据加载完成后，您会看到店铺和站点的利润数据表格
2. 表格中包含以下字段：
   - **店铺**：显示店铺名称和站点名称（店铺为父级，站点为子级）
   - **币种**：显示当前数据的币种
   - **销售额**：店铺的销售总额
   - **销售佣金**：平台收取的销售佣金
   - **FBA费用**：亚马逊FBA物流费用
   - **退款金额**：退款的总金额
   - **仓储费**：店铺的仓储费用
   - **广告费**：店铺的广告费用
   - **其它**：其他费用
   - **结算收入**：店铺的结算收入
   - **采购成本**：产品的采购成本
   - **利润**：销售额减去各项成本和费用后的利润

3. 店铺数据行以灰色背景显示，站点数据行以白色背景显示，方便您区分层级

### 3.5 导出利润报表

1. 确认您已经设置好查询条件
2. 点击页面右上角的"导出"按钮
3. 系统会生成Excel格式的利润报表并自动下载到您的电脑
4. 打开下载的Excel文件，查看详细的利润数据

## 4. 数据说明

### 4.1 费用项目说明

| 项目 | 说明 |
|------|------|
| 销售额 | 店铺在所选时间范围内的销售总额 |
| 销售佣金 | 亚马逊平台收取的销售佣金费用 |
| FBA费用 | 亚马逊FBA物流服务产生的费用 |
| 退款金额 | 客户退款的总金额 |
| 仓储费 | 亚马逊仓库存储费用 |
| 广告费 | 亚马逊广告投放费用 |
| 其它 | 其他未分类的费用 |
| 结算收入 | 店铺在所选时间范围内的结算收入 |
| 采购成本 | 产品的采购成本总和 |
| 利润 | 销售额减去各项成本和费用后的利润 |

### 4.2 币种说明

| 币种 | 说明 |
|------|------|
| 人民币(CNY) | 将所有费用转换为人民币显示 |
| 美元(USD) | 将所有费用转换为美元显示 |
| 站点币种 | 按各站点的本地币种显示费用（如美国站点显示美元，英国站点显示英镑等） |

### 4.3 日期类型说明

| 日期类型 | 说明 |
|---------|------|
| 结算日期 | 按亚马逊平台的结算日期统计数据 |
| 转账日期 | 按实际资金转账日期统计数据 |

## 5. 常见问题

### 5.1 数据加载缓慢

**问题**：点击查询后，数据加载时间较长

**解决方法**：
- 缩小查询日期范围，减少查询数据量
- 选择特定的店铺和站点，避免查询所有数据
- 确保网络连接稳定

### 5.2 数据与实际不符

**问题**：系统显示的利润数据与实际情况不符

**解决方法**：
- 检查日期范围是否正确
- 确认币种设置是否正确
- 检查是否选择了正确的结算日期类型
- 确保数据已经同步到系统（可能需要等待数据同步完成）

### 5.3 导出失败

**问题**：点击导出按钮后，报表下载失败

**解决方法**：
- 缩小查询日期范围，减少导出数据量
- 确保网络连接稳定
- 检查浏览器是否阻止了文件下载

### 5.4 币种转换错误

**问题**：币种转换后的数据不正确

**解决方法**：
- 确认选择了正确的币种
- 系统会自动使用最新的汇率进行转换，如果汇率数据过期，可能会导致转换结果不准确

## 6. 注意事项

1. **数据更新时间**：店铺利润数据可能需要一定时间同步到系统，建议在数据同步完成后再进行查询

2. **日期范围限制**：查询的日期范围不宜过大，否则可能会导致数据加载缓慢或导出失败

3. **权限限制**：您只能查看您有权限访问的店铺数据

4. **数据准确性**：系统提供的数据仅供参考，实际财务数据请以官方报表为准

5. **浏览器兼容性**：建议使用Chrome、Firefox、Edge等现代浏览器访问本模块，以获得最佳体验
