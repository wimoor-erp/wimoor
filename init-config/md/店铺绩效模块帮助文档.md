# 店铺绩效模块帮助文档

## 1. 模块概述

店铺绩效模块是 Wimoor ERP 系统中用于展示和监控亚马逊店铺绩效指标的功能模块。该模块提供了客户服务绩效、商品政策合规性和配送绩效三个核心维度的绩效数据，帮助卖家实时了解店铺运营状况，及时发现并解决问题，确保店铺健康运营。

## 2. 功能说明

### 2.1 核心功能

- **客户服务绩效**：展示订单缺陷率和发票缺陷率等客户服务相关指标
- **商品政策合规性**：展示账号健康评级和各类商品政策违规情况
- **配送绩效**：展示迟发率、准时交货率、有效追踪率和预配送取消率等配送相关指标
- **数据筛选**：支持按店铺分组和站点进行数据筛选
- **实时数据**：展示最新的绩效数据和目标值对比

### 2.2 界面元素

| 元素名称 | 功能描述 | 操作说明 |
|---------|---------|--------|

| 客户服务绩效卡片 | 展示订单缺陷率和发票缺陷率 | 自动加载并显示最新数据 |

| 数据更新按钮 | 手动刷新绩效数据 | 点击按钮刷新数据 |

## 3. 使用指南

### 3.1 基本操作流程

1. **进入模块**：从左侧菜单导航至 "亚马逊 > 报表 > 绩效 > 店铺绩效" 模块
2. **选择店铺**：在顶部的店铺选择器中选择要查看的店铺分组和站点
3. **查看绩效数据**：系统会自动加载并显示所选店铺的绩效数据
4. **分析绩效指标**：查看各项绩效指标的数值和目标值对比
5. **刷新数据**：点击刷新按钮获取最新的绩效数据

### 3.2 常见操作场景

#### 场景一：查看店铺整体绩效状况

1. 进入店铺绩效模块
2. 选择要查看的店铺分组和站点
3. 查看三个绩效卡片中的各项指标
4. 分析哪些指标需要改进

#### 场景二：监控订单缺陷率

1. 进入店铺绩效模块
2. 选择要查看的店铺分组和站点
3. 在客户服务绩效卡片中查看订单缺陷率
4. 分析卖家自配送和FBA配送的订单缺陷率
5. 查看具体的缺陷类型分布（差评、A-to-Z索赔、信用卡拒付）

#### 场景三：检查账号健康状况

1. 进入店铺绩效模块
2. 选择要查看的店铺分组和站点
3. 在商品政策合规性卡片中查看账号健康评级
4. 查看具体的政策违规情况
5. 针对违规情况采取相应的整改措施

#### 场景四：优化配送绩效

1. 进入店铺绩效模块
2. 选择要查看的店铺分组和站点
3. 在配送绩效卡片中查看各项配送指标
4. 分析哪些指标未达到目标值
5. 制定相应的优化措施

## 4. 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| 数据加载失败 | 网络连接问题或API限制 | 检查网络连接，稍后重试 |
| 数据显示不完整 | 亚马逊API数据同步延迟 | 等待一段时间后刷新数据 |
| 绩效指标异常 | 数据统计周期问题 | 查看数据的统计周期，确认是否在合理范围内 |
| 无法选择特定站点 | 该站点未授权或数据未同步 | 检查站点授权状态，确保数据同步正常 |
| 绩效指标未达到目标 | 运营问题 | 根据具体指标分析原因，采取相应的优化措施 |

## 5. 注意事项

- 店铺绩效数据来源于亚马逊平台，系统会定期同步最新数据
- 各项绩效指标都有亚马逊平台设定的目标值，卖家应努力达到或超过这些目标
- 订单缺陷率、迟发率等关键指标如果长期不达标，可能会影响店铺的销售权限
- 商品政策合规性是店铺健康运营的重要保障，应定期检查并及时处理违规情况
- 配送绩效直接影响买家体验，应优化物流流程，提高配送效率
