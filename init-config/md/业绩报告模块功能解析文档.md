# 业绩报告模块功能解析文档

## 1. 模块概述

业绩报告模块是亚马逊店铺管理系统中用于展示和分析店铺绩效数据的核心功能模块。该模块采用前后端分离架构，前端使用Vue 3 + Element Plus实现，后端基于Spring Boot + MyBatis-Plus构建，提供了全面的店铺绩效指标展示，包括客户服务绩效、商品政策合规性、配送绩效等多个维度的绩效数据。

### 1.1 核心功能

- **多维度绩效展示**：展示客户服务绩效、商品政策合规性、配送绩效等多个维度的绩效数据
- **实时数据更新**：实时获取和展示店铺绩效数据
- **店铺和站点选择**：支持选择不同店铺和站点查看绩效数据
- **详细指标分析**：提供详细的绩效指标分析，包括订单缺陷率、发票缺陷率、账号健康评级、迟发率、准时交货率等
- **视觉化展示**：使用图表和颜色编码直观展示绩效状态

### 1.2 技术架构

- **前端**：Vue 3、Composition API、Element Plus、Axios
- **后端**：Spring Boot 2.x、MyBatis-Plus、Spring Security
- **数据库**：MySQL
- **API风格**：RESTful API
- **数据传输**：JSON格式
- **亚马逊API集成**：通过SP-API获取店铺绩效数据

## 3. 后端实现分析

### 3.1 控制器实现

业绩报告模块的后端控制器位于`wimoor-amazon/amazon-boot/src/main/java/com/wimoor/amazon/auth/controller/AmazonAuthorityController.java`，主要提供以下接口：

#### 获取店铺绩效数据

```java
@ApiOperation(value = "获取站点")
@GetMapping("/getPerformance")
public Result<AmazonAuthMarketPerformance> getPerformanceAction(String groupid, String marketplaceid) {
    AmazonAuthority auth = iAmazonAuthorityService.selectByGroupAndMarket(groupid, marketplaceid);
    AmazonAuthMarketPerformance result = iAmazonAuthorityService.getPerformance(auth,marketplaceid);
    if(result!=null&&result.getPerformance()!=null) {
        result.setPerformanceJson(GeneralUtil.getJsonObject(result.getPerformance()));
    }
    return Result.success(result);
}
```

**关键实现**：
- 根据店铺ID和站点ID获取授权信息
- 调用服务层方法获取绩效数据
- 将绩效数据转换为JSON对象，方便前端处理
- 返回绩效数据

### 3.2 服务实现

业绩报告模块的后端服务实现位于`wimoor-amazon/amazon-boot/src/main/java/com/wimoor/amazon/auth/service/impl/AmazonAuthorityServiceImpl.java`，主要提供以下方法：

#### 获取店铺绩效数据

```java
@Override
public AmazonAuthMarketPerformance getPerformance(AmazonAuthority auth, String marketplaceid) {
    QueryWrapper<AmazonAuthMarketPerformance> query= new QueryWrapper<AmazonAuthMarketPerformance>();
    query.eq("amazonauthid", auth.getId());
    query.eq("marketplaceid", marketplaceid);
    AmazonAuthMarketPerformance perf = amazonAuthMarketPerformanceMapper.selectOne(query);
    return perf;
}
```

**关键实现**：
- 构建查询条件，根据授权ID和站点ID查询绩效数据
- 调用Mapper方法从数据库获取绩效数据
- 返回绩效数据

## 5. 业务逻辑流程

### 5.1 绩效数据查询流程

1. **前端发起请求**：用户选择店铺和站点，点击查询按钮
2. **参数处理**：前端处理查询参数，确保参数完整性
3. **API调用**：前端调用`authApi.getPerformance` API发送GET请求
4. **后端处理**：
   - 控制器接收请求，获取店铺ID和站点ID
   - 调用服务层方法根据店铺ID和站点ID获取授权信息
   - 调用服务层方法根据授权信息和站点ID获取绩效数据
   - 将绩效数据转换为JSON对象
   - 控制器返回绩效数据
5. **前端处理**：
   - 接收响应数据
   - 更新`state.dataList`
   - 页面自动更新展示绩效数据
   - 根据绩效数据计算并展示账号健康状态

### 5.2 绩效数据更新流程

1. **定时任务触发**：系统定时触发绩效数据更新任务
2. **数据获取**：系统通过亚马逊SP-API获取最新的绩效数据
3. **数据处理**：系统处理获取到的绩效数据，转换为适合存储的格式
4. **数据存储**：系统将处理后的绩效数据存储到数据库
5. **缓存更新**：系统更新相关缓存，确保数据一致性
