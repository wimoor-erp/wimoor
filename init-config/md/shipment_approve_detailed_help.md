# FBA 发货单审核功能详细帮助文档

## 1. 功能简介

FBA 发货单审核功能是 Wimoor 系统中 FBA 发货流程的重要环节，用于审核和管理发货计划。该功能位于系统的 ERP 模块中，主要负责处理发货计划的审核、驳回以及货件的管理操作。

### 1.1 主要功能

- **计划信息查看**：查看发货计划的详细信息，包括计划编码、商品列表、预估费用等
- **计划审核**：审核通过或驳回发货计划
- **货件管理**：对计划中的货件进行审核操作
- **计划拆分**：将发货计划拆分为多个子计划
- **计划复制**：基于现有计划创建新的发货计划

## 2. 界面介绍

### 2.1 整体布局

FBA 发货单审核页面采用清晰的三区域布局：

1. **头部信息区域**：显示计划基本信息和操作按钮
2. **主体内容区域**：左侧显示商品列表，右侧显示计划详细信息
3. **对话框区域**：包含审核对话框和拆分对话框

### 2.2 头部信息区域

头部信息区域位于页面顶部，包含以下内容：

- **计划编码**：显示当前发货计划的唯一编码
- **复制新增**：点击可基于当前计划创建新的发货计划
- **预估配置费**：显示该计划的预估配置费用
- **操作按钮组**：
  - **拆分**：将当前计划拆分为多个子计划
  - **关闭**：关闭当前页面，返回发货单列表
  - **审核**：当计划状态为"待审核"时显示，用于审核计划
  - **已审核**：当计划状态为"已审核"时显示，状态为禁用
  - **已驳回**：当计划状态为"已驳回"时显示，状态为禁用
  - **审核失败**：当计划审核失败时显示，状态为禁用
  - **帮助文档**：点击可查看相关帮助文档

### 2.3 主体内容区域

主体内容区域分为左右两部分：

- **左侧商品列表**：显示计划中的所有商品信息，包括 SKU、名称、数量、重量等
- **右侧信息列表**：显示计划的详细信息，包括发货店铺、仓库、地址、备注等

### 2.4 审核对话框

审核对话框用于审核计划中的货件，包含以下内容：

- **货件列表**：显示计划中的所有货件信息
- **审核选项**：为每个货件提供"通过"或"驳回"选项
- **全选驳回**：点击可将所有货件设置为"驳回"
- **警告信息**：提醒用户亚马逊规定同一订单中的货件需要同时通过或者驳回
- **操作按钮**：
  - **取消**：取消审核操作
  - **确认**：提交审核结果

## 3. 操作指南

### 3.1 查看计划信息

1. **进入页面**：通过系统导航进入 FBA 发货单审核页面
2. **系统自动加载**：页面加载时会自动获取并显示当前计划的详细信息
3. **查看商品列表**：在左侧商品列表中查看计划包含的所有商品
4. **查看详细信息**：在右侧信息列表中查看计划的详细信息

### 3.2 审核计划

1. **点击审核按钮**：在头部操作按钮组中点击"审核"按钮
2. **确认审核操作**：系统会弹出审核确认对话框
3. **选择审核结果**：
   - 点击"通过"：审核通过当前计划
   - 点击"驳回"：驳回当前计划
4. **系统处理**：系统会根据您的选择处理审核操作
5. **查看结果**：审核完成后，页面会自动更新计划状态

### 3.3 审核货件

1. **打开审核对话框**：在审核确认对话框中，系统会显示货件审核界面
2. **选择货件状态**：为每个货件选择"通过"或"驳回"
3. **使用全选驳回**：如果需要驳回所有货件，可点击"全选驳回"复选框
4. **提交审核**：点击"确认"按钮提交审核结果
5. **查看结果**：系统会处理审核操作并更新页面显示

### 3.4 拆分计划

1. **点击拆分按钮**：在头部操作按钮组中点击"拆分"按钮
2. **操作拆分对话框**：在拆分对话框中进行拆分操作
3. **完成拆分**：按照对话框提示完成计划拆分

### 3.5 复制计划

1. **点击复制按钮**：在头部信息区域点击"复制"按钮
2. **系统跳转**：系统会自动跳转到添加新版货件页面
3. **编辑新计划**：在新页面中编辑并保存新的发货计划

### 3.6 关闭页面

1. **点击关闭按钮**：在头部操作按钮组中点击"关闭"按钮
2. **系统跳转**：系统会自动跳转到新发货单页面

## 4. 技术实现

### 4.1 前端实现

FBA 发货单审核功能的前端实现基于 Vue 3 框架，使用了 Composition API 进行状态管理和逻辑组织。

#### 4.1.1 核心组件

- **index.vue**：核心审核组件，包含完整的审核功能
- **table.vue**：商品列表组件，显示计划中的商品信息
- **list.vue**：信息列表组件，显示计划的详细信息
- **split_dialog.vue**：拆分对话框组件，用于拆分计划

#### 4.1.2 主要 API 调用

| API 方法 | 用途 | 参数 |
|---------|------|------|
| `getPlanInfo` | 获取发货计划信息 | `{formid: 计划ID}` |
| `approveInboundPlan` | 审核通过发货计划 | `{id: 计划ID}` |
| `rejectInboundPlan` | 驳回发货计划 | `{id: 计划ID}` |
| `createShipment` | 创建货件 | `{shipmentid: 货件ID}` |
| `cancelShipment` | 取消货件 | `{shipmentid: 货件ID}` |
| `inplacefee` | 计算入库费用 | `{sku, inplace, length, width, height, weight}` |

### 4.2 后端实现

FBA 发货单审核功能的后端实现基于 Spring Boot 框架，使用了 MyBatis Plus 进行数据库操作。

#### 4.2.1 核心控制器

- **ShipInboundPlanV2Controller**：处理发货计划相关的操作，如获取计划信息、审核计划等
- **ShipFormController**：处理货件相关的操作，如创建货件、取消货件等

#### 4.2.2 主要数据模型

- **ShipInboundPlan**：发货计划实体，存储计划的基本信息
- **ShipInboundItem**：发货计划商品实体，存储计划中的商品信息
- **ShipInboundShipment**：货件实体，存储货件的信息
- **ShipInboundOperation**：操作记录实体，存储操作的记录

## 5. 业务流程

### 5.1 页面加载流程

1. **用户进入页面**：通过系统导航进入 FBA 发货单审核页面
2. **系统获取计划ID**：从 URL 参数中获取当前要审核的计划 ID
3. **加载计划信息**：系统调用 `getPlanInfo` API 获取计划的详细信息
4. **初始化页面**：系统使用获取到的计划信息初始化页面各个组件
5. **计算费用**：系统计算计划的预估入库费用

### 5.2 审核流程

1. **用户点击审核按钮**：在页面头部点击"审核"按钮
2. **系统弹出确认对话框**：显示审核确认信息
3. **用户选择审核结果**：点击"通过"或"驳回"按钮
4. **系统处理审核操作**：
   - 如果选择"通过"，系统调用 `approveInboundPlan` API 审核通过计划
   - 如果选择"驳回"，系统调用 `rejectInboundPlan` API 驳回计划
5. **系统更新页面**：审核完成后，系统重新加载计划信息并更新页面显示

### 5.3 货件审核流程

1. **用户进入货件审核**：在审核确认对话框中，系统显示货件审核界面
2. **用户选择货件状态**：为每个货件选择"通过"或"驳回"
3. **用户提交审核**：点击"确认"按钮提交审核结果
4. **系统处理货件操作**：
   - 对于选择"通过"的货件，系统调用 `createShipment` API 创建货件
   - 对于选择"驳回"的货件，系统调用 `cancelShipment` API 取消货件
5. **系统更新页面**：审核完成后，系统重新加载计划信息并更新页面显示

## 6. 常见问题与解决方案

### 6.1 审核失败

**问题**：点击审核按钮后，系统提示审核失败

**可能原因**：
- 网络连接问题
- 系统服务暂时不可用
- 计划信息不完整或存在错误

**解决方案**：
- 检查网络连接是否正常
- 稍后重试审核操作
- 检查计划信息是否完整，如有错误请修正后重试

### 6.2 货件创建失败

**问题**：审核货件时，系统提示货件创建失败

**可能原因**：
- 货件信息不完整
- 亚马逊 API 暂时不可用
- 账户权限不足

**解决方案**：
- 检查货件信息是否完整
- 稍后重试操作
- 确认账户是否有足够的权限

### 6.3 页面加载缓慢

**问题**：进入审核页面后，页面加载缓慢

**可能原因**：
- 计划包含大量商品
- 网络连接速度较慢
- 系统负载较高

**解决方案**：
- 耐心等待页面加载完成
- 检查网络连接速度
- 避开系统高峰期使用该功能

### 6.4 拆分功能失败

**问题**：点击拆分按钮后，拆分操作失败

**可能原因**：
- 计划信息不完整
- 系统服务暂时不可用
- 拆分参数设置错误

**解决方案**：
- 检查计划信息是否完整
- 稍后重试拆分操作
- 确保拆分参数设置正确
