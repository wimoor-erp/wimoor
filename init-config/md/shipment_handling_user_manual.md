## 模块介绍

货件处理模块是 Wimoor 系统中 FBA 发货流程的核心环节，用于处理和管理发货计划中的货件。该模块位于系统的 ERP 模块中，主要负责货件的信息展示、操作管理、状态更新等功能。

通过该模块，用户可以：
- 查看货件的详细信息，包括基本信息、运输信息、地址信息等
- 对货件进行删除、复制、本地完成等操作
- 管理货件的箱标收货地址
- 与亚马逊后台同步货件状态
- 查看货件的物流费用和货值

## 功能概览

### 1. 货件信息展示

货件信息展示功能为用户提供了货件的全面详细信息，包括：
- **基本信息**：货件编码、货件名称、货件状态等
- **运输信息**：总货值、物流总费用、SKU个数、重量、体积等
- **地址信息**：发货地址、收货地址、箱标收货地址等

### 2. 货件操作管理

货件操作管理功能为用户提供了对货件的各种操作，包括：
- **删除货件**：删除本地货件或同步删除亚马逊货件
- **复制货件**：基于当前货件创建新的货件
- **本地完成**：将货件标记为本地已完成状态

### 3. 箱标地址管理

箱标地址管理功能允许用户为货件添加和管理箱标收货地址，确保货件能够准确送达指定地点。

### 4. 货件状态同步

货件状态同步功能能够与亚马逊后台同步货件状态，确保用户看到的货件状态与亚马逊后台保持一致。

### 5. 费用计算

费用计算功能能够自动计算货件的物流费用和货值，帮助用户了解货件的成本情况。

## 操作指南

### 3.1 货件信息查看

**操作步骤**：
1. **进入货件处理页面**：通过系统导航菜单进入货件处理页面
2. **选择货件**：在货件列表中选择要查看的货件
3. **查看货件信息**：系统会自动加载并显示货件的详细信息，包括基本信息、运输信息、地址信息等
4. **查看详细信息**：
   - 在"单据信息"区域查看货件的基本信息
   - 在"运输信息"区域查看货件的运输相关信息
   - 在"地址信息"区域查看货件的发货地址和收货地址

**注意事项**：
- 货件信息加载可能需要几秒钟时间，请耐心等待
- 如果货件信息不完整，可能是因为货件尚未完全创建或同步

### 3.2 货件删除操作

**操作步骤**：
1. **进入货件处理页面**：通过系统导航菜单进入货件处理页面
2. **找到删除按钮**：在操作按钮区域找到删除按钮
3. **点击删除按钮**：系统会弹出确认删除对话框
4. **查看货件状态**：系统会自动获取亚马逊后台的货件状态
5. **选择删除方式**：
   - 如果货件状态不是"CANCELLED"，系统会询问是否同步删除亚马逊货件
   - 如果货件状态是"CANCELLED"，系统会询问是否删除本地货件
   - 如果无法判断货件状态，系统会建议仅删除本地货件
6. **确认删除**：点击"确认"按钮执行删除操作
7. **查看操作结果**：系统会显示删除操作的结果

**注意事项**：
- 删除操作不可逆，请谨慎操作
- 同步删除亚马逊货件会影响亚马逊后台的数据，请确保操作的必要性
- 仅删除本地货件不会影响亚马逊后台的数据

### 3.3 货件复制操作

**操作步骤**：
1. **进入货件处理页面**：通过系统导航菜单进入货件处理页面
2. **找到复制按钮**：在操作按钮区域找到复制按钮
3. **点击复制按钮**：系统会跳转到添加货件页面
4. **编辑新货件**：在添加货件页面中编辑新货件的信息
5. **保存新货件**：编辑完成后，点击"保存"按钮保存新货件

**注意事项**：
- 复制货件会创建一个新的货件，与原货件相互独立
- 新货件会继承原货件的大部分信息，但用户可以根据需要进行修改

### 3.4 本地完成操作

**操作步骤**：
1. **进入货件处理页面**：通过系统导航菜单进入货件处理页面
2. **找到本地完成按钮**：在操作按钮区域找到本地完成按钮
3. **点击本地完成按钮**：系统会弹出确认对话框
4. **确认操作**：点击"确认"按钮执行本地完成操作
5. **查看操作结果**：系统会显示操作结果

**注意事项**：
- 本地完成操作仅在系统内部标记货件为已完成状态，不会影响亚马逊后台的数据
- 本地完成操作通常用于已经在亚马逊后台完成但系统尚未同步的货件

### 3.5 箱标地址管理

**操作步骤**：
1. **进入货件处理页面**：通过系统导航菜单进入货件处理页面
2. **找到添加箱标地址按钮**：在地址信息区域找到"添加箱标地址"按钮
3. **点击添加箱标地址按钮**：系统会弹出地址选择对话框
4. **选择箱标收货地址**：在地址列表中选择合适的箱标收货地址
5. **确认选择**：点击"确认"按钮保存箱标收货地址
6. **查看操作结果**：系统会显示操作结果，并在地址信息区域显示箱标收货地址

**注意事项**：
- 箱标收货地址是货件箱标上显示的收货地址，用于物流配送
- 箱标收货地址可以与货件的收货地址不同
- 可以随时修改或删除箱标收货地址

## 界面说明

### 4.1 头部信息区域

头部信息区域位于货件处理页面的顶部，包含以下内容：
- **货件编码**：显示当前货件的唯一编码
- **货件名称**：显示当前货件的名称
- **操作按钮**：包括删除、复制、本地完成等操作按钮

### 4.2 货件信息区域

货件信息区域是页面的主要内容区域，分为三个部分：
1. **单据信息**：
   - 订单编码：显示关联的订单编码
   - 货件名称：显示货件的名称
   - 货件编码：显示货件的唯一编码
   - 货件状态：显示货件的当前状态
   - 店铺：显示货件所属的店铺
   - 收货站点：显示货件的收货国家/地区
   - 发货仓库：显示货件的发货仓库
   - 备注：显示货件的备注信息

2. **运输信息**：
   - 总货值：显示货件的总价值
   - 物流总费用：显示货件的物流费用
   - 实际发货总数：显示货件的实际发货数量
   - SKU个数：显示货件包含的SKU数量
   - 实际结算重量：显示货件的实际结算重量
   - 产品净重：显示货件中产品的净重
   - 箱子重：显示货件中箱子的重量
   - 体积：显示货件的体积
   - 亚马逊运输方式：显示货件的运输方式
   - 装箱箱数：显示货件的装箱数量

3. **地址信息**：
   - 发货地址：显示货件的发货地址
   - 收货地址：显示货件的收货地址
   - 箱标收货地址：显示货件的箱标收货地址（如果已设置）

### 4.3 操作按钮区域

操作按钮区域位于页面的顶部，包含以下按钮：
- **删除**：删除当前货件
- **复制**：复制当前货件创建新货件
- **本地完成**：将当前货件标记为本地已完成
- **添加箱标地址**：为当前货件添加箱标收货地址

## 常见问题

### 1. 货件状态同步失败怎么办？

**原因**：
- 网络连接问题
- 亚马逊 API 限制
- 亚马逊账号权限问题

**解决方案**：
- 检查网络连接是否正常
- 稍后重试操作
- 检查亚马逊账号权限是否正确

### 2. 箱标地址保存失败怎么办？

**原因**：
- 地址信息不完整
- 地址格式错误
- 系统内部错误

**解决方案**：
- 检查地址信息是否完整，确保所有必填字段都已填写
- 检查地址格式是否正确，确保地址符合系统要求
- 稍后重试操作，如仍失败请联系系统管理员

### 3. 货件删除失败怎么办？

**原因**：
- 亚马逊后台货件状态不允许删除
- 系统内部错误
- 权限不足

**解决方案**：
- 先确认亚马逊后台货件状态，再执行删除操作
- 尝试仅删除本地货件
- 检查用户权限是否足够

### 4. 页面加载缓慢怎么办？

**原因**：
- 货件信息过多
- 网络延迟
- 系统负载过高

**解决方案**：
- 优化网络连接
- 稍后重试操作
- 如问题持续，请联系系统管理员

### 5. 费用计算错误怎么办？

**原因**：
- 物流信息不完整
- 汇率更新不及时
- 系统内部错误

**解决方案**：
- 检查物流信息是否完整
- 确保汇率数据已更新
- 如问题持续，请联系系统管理员

## 故障排除

### 1. 无法进入货件处理页面

**症状**：点击货件处理页面链接后，页面无法加载或显示错误信息

**解决方案**：
- 检查网络连接是否正常
- 清除浏览器缓存后重试
- 检查用户权限是否足够
- 如问题持续，请联系系统管理员

### 2. 货件信息显示不完整

**症状**：货件处理页面显示的货件信息不完整或缺失

**解决方案**：
- 刷新页面后重试
- 检查货件是否已完全创建
- 检查网络连接是否正常
- 如问题持续，请联系系统管理员

### 3. 无法执行货件操作

**症状**：点击货件操作按钮后，操作无法执行或显示错误信息

**解决方案**：
- 检查网络连接是否正常
- 检查用户权限是否足够
- 检查货件状态是否允许执行该操作
- 如问题持续，请联系系统管理员

### 4. 箱标地址无法保存

**症状**：选择箱标收货地址后，无法保存或保存后不显示

**解决方案**：
- 检查地址信息是否完整
- 检查网络连接是否正常
- 刷新页面后重试
- 如问题持续，请联系系统管理员

### 5. 费用计算不正确

**症状**：货件的费用计算结果与实际不符

**解决方案**：
- 检查物流信息是否完整
- 检查汇率数据是否已更新
- 如问题持续，请联系系统管理员

## 最佳实践

### 1. 货件管理最佳实践

- **及时更新货件状态**：定期检查和更新货件状态，确保货件状态与实际情况一致
- **合理使用箱标地址**：为货件设置合适的箱标收货地址，确保货件能够准确送达
- **定期清理无用货件**：定期清理不再需要的货件，保持系统整洁
- **备份重要货件信息**：对重要的货件信息进行备份，以防数据丢失

### 2. 操作安全最佳实践

- **谨慎执行删除操作**：删除操作不可逆，请在执行前仔细确认
- **合理选择删除方式**：根据实际情况选择合适的删除方式（仅删除本地或同步删除亚马逊货件）
- **定期检查权限设置**：确保只有授权用户能够执行敏感操作
- **记录重要操作**：对重要操作进行记录，便于后续追溯

### 3. 系统使用最佳实践

- **保持浏览器更新**：使用最新版本的浏览器，确保系统能够正常运行
- **优化网络连接**：确保网络连接稳定，避免因网络问题导致操作失败
- **合理安排操作时间**：避开系统高峰期执行大量操作
- **及时反馈问题**：发现问题及时反馈给系统管理员，帮助系统不断改进
