# 采购时效模块帮助文档

## 1. 模块概述

采购时效模块是Wimoor系统中用于监控和分析采购流程时效的重要功能模块。该模块提供了详细的采购时效数据，帮助用户追踪从采购下单到入库的全过程时间消耗，分析采购效率，优化采购流程，提高供应链管理水平。

### 1.1 主要功能

- **采购时效分析**：分析从下单到入库的全过程时效
- **数据筛选**：按日期、仓库、SKU等维度筛选数据
- **详细时效展示**：展示下单时间、审核时间、入库时间等关键节点
- **库存状态分析**：分析采购物品的库存状态，包括可用库存、待入库、待出库等
- **报表导出**：导出采购时效报表，方便离线分析
- **多维数据展示**：展示收货、换货、出库等相关数据

### 1.2 应用场景

- **采购效率分析**：分析采购流程的时间消耗，识别瓶颈环节
- **供应商评估**：基于时效数据评估供应商的交货及时性
- **库存管理优化**：根据时效数据优化库存管理策略
- **采购计划调整**：基于历史时效数据调整采购计划
- **流程改进**：识别采购流程中的问题，提出改进方案
- **成本分析**：分析时效对采购成本的影响

## 2. 界面介绍

### 2.1 主界面布局

采购时效模块采用清晰的响应式布局，主要分为以下几个区域：

1. **筛选区域**（顶部）：提供日期选择器、仓库选择器和SKU搜索框
2. **数据展示区域**（中部）：以表格形式展示采购时效数据
3. **报表导出区域**（顶部右侧）：提供报表导出按钮

| 组件名称 | 位置 | 功能描述 |

## 3. 使用指南

### 3.1 基本操作流程

1. **进入模块**：从系统左侧菜单导航至「采购」→「采购报表」→「采购时效」模块
2. **设置筛选条件**：在顶部筛选区域选择日期范围、仓库，输入SKU关键词
3. **查询数据**：点击搜索按钮或按回车键执行查询
4. **查看数据**：在数据展示区域查看采购时效数据
5. **查看详细信息**：鼠标悬停在相应列上，查看详细信息
6. **导出报表**：点击顶部右侧的「导出」按钮，导出采购时效报表

### 3.2 功能使用详解

#### 3.2.1 数据筛选

1. **日期筛选**：
   - 点击日期选择器，选择需要查询的开始日期和结束日期
   - 系统默认显示最近30天的数据

2. **仓库筛选**：
   - 点击仓库选择器，从下拉列表中选择需要查询的仓库
   - 可选择多个仓库进行查询

3. **SKU筛选**：
   - 在SKU搜索框中输入SKU关键词
   - 点击搜索按钮或按回车键执行搜索

#### 3.2.2 时效分析

1. **查看总时效**：
   - 在「时效」列查看从下单到入库的总时效天数
   - 总时效为0天时，显示为1天

2. **查看详细时效节点**：
   - 鼠标悬停在「时效」列的数值上
   - 在弹出的提示框中查看详细的时效节点，包括：
     - 下单时间：采购订单创建时间
     - 审核时间：采购订单审核通过时间
     - 最近入库：最近一次入库时间
     - 最近组装：最近一次组装时间
     - 最近发货：最近一次发货时间

#### 3.2.3 库存状态分析

1. **查看可用库存**：
   - 在「可用库存」列查看当前可用库存数量

2. **查看详细库存状态**：
   - 鼠标悬停在「可用库存」列的数值上
   - 在弹出的提示框中查看详细的库存状态，包括：
     - 待入库：等待入库的数量
     - 可用库存：当前可使用的库存数量
     - 待出库：等待出库的数量
     - 主SKU相关库存：如果该商品是组装件的子件，还会显示主SKU的库存状态

#### 3.2.4 报表导出

1. **导出报表**：
   - 点击顶部右侧的「导出」按钮
   - 系统会生成采购时效报表
   - 生成完成后，在弹出的下载对话框中点击下载

2. **报表内容**：
   - 报表包含完整的采购时效数据
   - 包括商品信息、订单信息、时效数据、库存状态等
   - 支持Excel格式导出

## 4. 操作流程

### 4.1 采购时效查询流程

1. **进入模块**：从左侧菜单导航至「采购」→「采购报表」→「采购时效」
2. **设置日期范围**：点击日期选择器，选择查询的开始和结束日期
3. **选择仓库**：点击仓库选择器，选择需要查询的仓库
4. **输入SKU**：在SKU搜索框中输入SKU关键词
5. **执行查询**：点击搜索按钮或按回车键执行查询
6. **查看结果**：在数据展示区域查看查询结果
7. **查看详细信息**：鼠标悬停在相应列上，查看详细信息

### 4.2 报表导出流程

1. **设置筛选条件**：按照需要设置日期范围、仓库、SKU等筛选条件
2. **点击导出按钮**：点击顶部右侧的「导出」按钮
3. **等待报表生成**：系统会提示「已提交报表，等待生成」
4. **下载报表**：在弹出的下载对话框中，点击下载按钮下载报表

## 5. 数据指标说明

### 5.1 时效指标

| 指标名称 | 指标说明 | 计算方法 |
|---------|---------|----------|
| 总时效 | 从下单到入库的总天数 | 最近入库时间 - 下单时间 |
| 下单时间 | 采购订单创建时间 | 系统记录的订单创建时间 |
| 审核时间 | 采购订单审核通过时间 | 系统记录的订单审核时间 |
| 入库时间 | 商品入库时间 | 系统记录的入库时间 |
| 组装时间 | 商品组装时间 | 系统记录的组装时间 |
| 发货时间 | 商品发货时间 | 系统记录的发货时间 |

### 5.2 库存指标

| 指标名称 | 指标说明 | 计算方法 |
|---------|---------|----------|
| 采购数量 | 采购订单中的商品数量 | 订单中记录的数量 |
| 收货数量 | 实际收到的商品数量 | 入库单中记录的数量 |
| 可用库存 | 当前可使用的库存数量 | 总库存 - 预留库存 |
| 待入库 | 等待入库的商品数量 | 已下单但未入库的数量 |
| 待出库 | 等待出库的商品数量 | 已分配但未出库的数量 |
| 换货数量 | 因质量问题等原因换货的数量 | 换货单中记录的数量 |
| 出库数量 | 已经出库的商品数量 | 出库单中记录的数量 |
| 盘点出库 | 因盘点差异出库的数量 | 盘点单中记录的数量 |

### 5.3 其他指标

| 指标名称 | 指标说明 | 计算方法 |
|---------|---------|----------|
| 主SKU | 由当前商品组装而成的主SKU | 系统记录的组装关系 |
| 主SKU库存 | 主SKU的库存状态 | 系统记录的主SKU库存数据 |
| 组装消耗 | 组装过程中消耗的当前商品数量 | 组装单中记录的消耗数量 |
| 发货消耗 | 发货过程中消耗的当前商品数量 | 出库单中记录的消耗数量 |

## 6. 常见问题与解决方案

### 6.1 数据显示问题

| 问题描述 | 可能原因 | 解决方案 |
|---------|---------|----------|
| 时效显示为「暂无」 | 商品尚未入库或系统中没有入库记录 | 检查商品是否已入库，或确认系统中是否有入库记录 |
| 库存数据与实际不符 | 库存数据未及时更新 | 等待系统自动更新或手动触发库存同步 |
| 筛选后无数据 | 筛选条件设置不当或该条件下确实无数据 | 调整筛选条件，尝试扩大日期范围或选择其他仓库 |
| 报表导出失败 | 数据量过大或系统繁忙 | 尝试减少筛选范围，或在系统空闲时导出 |

### 6.2 功能操作问题

| 问题描述 | 可能原因 | 解决方案 |
|---------|---------|----------|
| 日期选择器无法选择日期 | 浏览器兼容性问题或系统时间设置错误 | 尝试使用其他浏览器，或检查系统时间设置 |
| 仓库选择器无选项 | 没有仓库权限或系统中未设置仓库 | 联系管理员分配仓库权限，或检查仓库设置 |
| SKU搜索无结果 | SKU输入错误或该SKU在选定条件下无数据 | 检查SKU输入是否正确，或调整筛选条件 |
| 详细信息提示框不显示 | 浏览器设置问题或鼠标悬停时间过短 | 检查浏览器设置，确保允许弹出提示框，或延长鼠标悬停时间 |

### 6.3 数据准确性问题

| 问题描述 | 可能原因 | 解决方案 |
|---------|---------|----------|
| 时效计算不准确 | 系统中缺少关键时间节点的记录 | 检查系统中是否有完整的时间记录，确保所有操作都及时录入系统 |
| 库存数据不准确 | 库存操作未及时录入系统 | 确保所有库存操作都及时录入系统，定期进行库存盘点 |
| 组装消耗数据不准确 | 组装单未正确录入 | 确保组装单的准确性，定期核对组装消耗数据 |
| 发货消耗数据不准确 | 出库单未正确录入 | 确保出库单的准确性，定期核对发货消耗数据 |

## 7. 最佳实践

### 7.1 日常使用

1. **定期查看**：每周查看采购时效数据，及时发现问题
2. **对比分析**：与历史数据进行对比，分析时效变化趋势
3. **分类分析**：按供应商、商品类别等维度分类分析时效数据
4. **重点关注**：重点关注时效异常的采购订单，分析原因
5. **数据导出**：每月导出一次完整报表，进行深度分析

### 7.2 供应商管理

1. **时效评估**：基于时效数据评估供应商的表现
2. **分级管理**：根据时效表现对供应商进行分级管理
3. **沟通改进**：与时效表现差的供应商沟通，提出改进要求
4. **激励机制**：对时效表现好的供应商给予更多订单或其他激励

### 7.3 流程优化

1. **瓶颈识别**：通过时效数据识别采购流程中的瓶颈环节
2. **流程改进**：针对瓶颈环节，提出流程改进方案
3. **效果验证**：实施改进方案后，通过时效数据验证改进效果
4. **持续优化**：建立持续优化机制，不断改进采购流程

### 7.4 库存管理

1. **采购计划**：基于时效数据制定更准确的采购计划
2. **安全库存**：根据时效数据调整安全库存水平
3. **库存预警**：设置基于时效的库存预警机制
4. **库存周转率**：结合时效数据分析库存周转率，优化库存管理

### 7.5 成本控制

1. **时效成本**：分析时效对采购成本的影响
2. **优化策略**：制定成本与时效平衡的采购策略
3. **供应商选择**：基于成本和时效综合评估选择供应商
4. **谈判依据**：以时效数据为依据与供应商谈判价格和交货期
