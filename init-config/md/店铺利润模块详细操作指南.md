# 店铺利润模块详细操作指南

## 一、模块简介

店铺利润模块是 Wimoor ERP 系统中用于**按店铺统计亚马逊销售利润**的核心功能模块。通过该模块，您可以清晰地了解每个店铺/站点的销售业绩、费用支出和最终利润情况。

### 主要功能

| 功能 | 说明 |
| :--- | :--- |
| **店铺利润统计** | 按店铺维度统计销售利润 |
| **多币种切换** | 支持人民币、美元、站点币种展示 |
| **日期筛选** | 支持自定义日期范围查询 |
| **数据导出** | 支持Excel格式导出报表 |

---

## 二、访问路径

### 2.1 进入模块

1. 登录 Wimoor ERP 系统
2. 在左侧导航栏中找到 **亚马逊** -> **财务** -> **利润统计**
3. 点击进入后，默认显示"店铺利润"标签页

## 三、详细操作步骤

### 3.1 选择店铺/站点

1. 点击页面上方的**店铺选择器**（显示为下拉框或分组选择组件）
2. 在弹出的选择界面中：
   - 选择**店铺分组**（可选，用于批量选择多个店铺）
   - 选择**具体站点**（如美国站、英国站等）
3. 选择完成后，系统会自动刷新数据

### 3.2 设置日期范围

1. 点击**日期选择器**
2. 选择日期类型：
   - **结算日期**：按亚马逊结算报告日期统计
   - **转账日期**：按实际转账日期统计
3. 选择具体的日期范围：
   - 点击开始日期，选择起始时间
   - 点击结束日期，选择终止时间
4. 系统默认显示**近7天**的数据

### 3.3 切换币种显示

1. 点击**币种选择器**下拉框
2. 选择目标币种：
   - **人民币**：所有金额转换为人民币显示
   - **美元**：所有金额转换为美元显示
   - **站点币种**：保持各站点原币种显示
3. 选择后系统会自动刷新数据并按新币种重新计算

### 3.4 查看利润数据

数据表格包含以下列：

| 列名 | 说明 |
| :--- | :--- |
| **店铺** | 亚马逊店铺/站点名称 |
| **币种** | 当前显示的币种 |
| **销售额** | 商品销售总金额 |
| **销售佣金** | 亚马逊收取的销售佣金 |
| **FBA费用** | FBA配送费用 |
| **退款金额** | 退款订单金额 |
| **仓储费** | 亚马逊仓储费用 |
| **广告费** | 广告投放费用 |
| **其它** | 其他杂项费用 |
| **结算收入** | 最终结算收入 |
| **采购成本** | 商品采购成本 |
| **利润** | 结算收入 - 采购成本 |

**注意**：表格中**背景灰色**的行为**合计行**，显示该店铺分组下所有站点的汇总数据。

### 3.5 导出数据

1. 确认筛选条件已设置好
2. 点击右上角的**导出**按钮
3. 系统会自动下载Excel文件
4. 打开Excel文件查看详细数据

## 四、常见问题解答

### 4.1 为什么数据为空？

**可能原因**：
- 未选择任何店铺/站点
- 日期范围内没有结算数据
- 亚马逊授权过期导致无法同步数据

**解决方法**：
1. 检查店铺选择器是否已选择店铺
2. 调整日期范围，尝试选择更长的时间段
3. 检查亚马逊授权状态，重新授权

### 4.2 为什么某些费用项显示为0？

**可能原因**：
- 该费用类型在所选时间段内没有发生
- 数据尚未同步完成

**解决方法**：等待数据同步完成，或选择包含更多数据的日期范围。

### 4.3 币种转换如何计算？

系统根据**实时汇率**进行币种转换，汇率数据每日更新。转换公式：

```
目标金额 = 原金额 × 汇率
```

### 4.4 如何查看单个SKU的利润？

店铺利润模块仅显示店铺级别的汇总数据。如需查看SKU级别的利润分析，请切换到"SKU利润"相关模块。

## 五、操作技巧

### 5.1 快速查看上周数据

1. 点击日期选择器
2. 选择快捷选项"上周"（如果有）
3. 系统自动设置上周一至周日的日期范围

### 5.2 对比不同时间段数据

1. 先设置第一个时间段，记录关键数据
2. 修改日期范围为第二个时间段
3. 对比两次查询结果

### 5.3 批量导出多店铺数据

1. 在店铺选择器中选择多个店铺或整个分组
2. 设置合适的日期范围
3. 点击导出按钮，一次性获取所有选中店铺的数据

## 六、数据更新说明

### 6.1 数据同步频率

- **自动同步**：系统每日自动同步亚马逊结算报告
- **手动同步**：可在数据管理模块手动触发同步

### 6.2 数据延迟说明

由于亚马逊结算报告生成需要时间，通常存在**1-3天**的数据延迟。
