# 产品管理模块帮助文档

## 1. 模块概述

产品管理模块是Wimoor系统的核心功能之一，用于管理企业的产品信息，包括产品基本信息、SKU、品牌、品类、采购信息、规格信息等。该模块支持产品的添加、编辑、删除、查询等操作，为企业的采购、库存、销售等业务提供基础数据支持。

#### 核心代码示例：

```java
@Api(tags = "产品接口")
@RestController
@RequestMapping("/api/v1/material")
@SystemControllerLog("产品信息")
@Slf4j
@RequiredArgsConstructor
public class MaterialController {
    
    final IMaterialService iMaterialService;
    final IStepWisePriceService stepWisePriceService;
    final IDimensionsInfoService dimensionsInfoService;
    final IAssemblyService assemblyService;
    final IInventoryService inventoryService;
    // 其他服务注入...
    
    @ApiOperation("根据本地产品ID查询产品详情")
    @GetMapping("/getMaterialInfo")
    public Result<MaterialInfoVO> getMaterialInfoAction(@ApiParam("产品ID")@RequestParam String id) {
        MaterialInfoVO vo=new MaterialInfoVO();
        MaterialVO data = iMaterialService.findMaterialById(id);
        if (data == null) {
            return Result.success(vo);
        }
        // 加载产品相关信息
        DimensionsInfo itemDim = null;
        DimensionsInfo pkgDim = null;
        DimensionsInfo boxDim = null;
        if(data.getItemdimensions()!=null) {
            itemDim = dimensionsInfoService.getById(data.getItemdimensions());
        }
        // 加载阶梯价格
        List<StepWisePrice> priceList = stepWisePriceService.selectByMaterial(data.getId());
        // 加载组装信息
        List<AssemblyVO> assemblyList = assemblyService.selectByMainmid(data.getId());
        // 加载供应商信息
        List<MaterialSupplierVO> supplierList = iMaterialSupplierService.selectSupplierByMainmid(data.getId());
        // 加载其他信息...
        
        vo.setMaterial(data);
        vo.setAssemblyList(assemblyList);
        vo.setBoxDim(boxDim);
        vo.setItemDim(itemDim);
        vo.setPkgDim(pkgDim);
        vo.setStepWisePrice(priceList);
        vo.setSupplierList(supplierList);
        // 设置其他信息...
        
        return Result.success(vo);
    }
    
    @SystemControllerLog("产品信息修改")
    @PostMapping(value="/saveData",consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    @Transactional
    public Result<Map<String, Object>> saveAction(String infostr,@RequestParam(value="file",required=false)MultipartFile file,
                                                  @RequestParam(value="pkgfile",required=false)MultipartFile pkgfile){
        UserInfo userinfo = UserInfoContext.get();
        ObjectMapper mapper = new ObjectMapper();
        MaterialInfoVO vo=null;
        try {
            vo = mapper.readValue(infostr, MaterialInfoVO.class);
        } catch (JsonMappingException e) {
            e.printStackTrace();
        } catch (JsonProcessingException e) {
            e.printStackTrace();
        }
        String id=iMaterialService.saveAllInfo(vo,file,pkgfile,userinfo);
        Map<String, Object> maps=new HashMap<String, Object>();
        maps.put("id", id);
        return Result.success(maps);
    }
    
    // 其他方法...
}
```

## 4. 数据流程

### 4.1 产品添加流程

1. 用户进入产品管理页面，点击"添加产品"按钮
2. 系统打开产品编辑页面，用户填写产品基本信息（名称、SKU、品牌、品类等）
3. 用户填写采购信息（供应商、阶梯价格等）
4. 用户填写规格信息（尺寸、重量等）
5. 用户点击"提交"按钮
6. 前端验证表单数据
7. 前端将数据发送到后端 `/api/v1/material/saveData` 接口
8. 后端处理数据，保存产品信息到数据库
9. 后端返回保存结果
10. 前端显示保存成功提示，并跳转到产品详情页面

### 4.2 产品编辑流程

1. 用户在产品列表页面点击"编辑"按钮
2. 系统打开产品编辑页面，加载产品现有信息
3. 用户修改产品信息
4. 用户点击"提交"按钮
5. 前端验证表单数据
6. 前端将数据发送到后端 `/api/v1/material/saveData` 接口
7. 后端处理数据，更新产品信息到数据库
8. 后端返回保存结果
9. 前端显示保存成功提示，并跳转到产品详情页面

### 4.3 产品查询流程

1. 用户在产品列表页面设置查询条件
2. 前端发送查询请求到后端 `/api/v1/material/list` 接口
3. 后端根据查询条件查询产品列表
4. 后端返回查询结果
5. 前端显示产品列表

### 5.1 产品基本信息管理

- **产品名称**：产品的名称，用于识别产品

- **组装数量**：每个组合产品需要的子产品数量

## 8. 常见问题及解决方案

### 8.1 产品添加失败

**问题**：点击提交按钮后，产品添加失败
**原因**：
- 表单数据验证失败
- 后端处理异常
- 网络连接问题

**解决方案**：
- 检查表单数据是否填写完整
- 检查网络连接是否正常
- 查看浏览器控制台是否有错误信息
- 联系系统管理员查看后端日志

### 8.2 产品编辑后信息不更新

**问题**：编辑产品信息后，保存成功但页面显示的信息没有更新
**原因**：
- 缓存问题
- 页面刷新问题

**解决方案**：
- 刷新页面
- 清除浏览器缓存
- 重新进入产品管理页面

### 8.3 阶梯价格设置错误

**问题**：设置阶梯价格后，价格计算不正确
**原因**：
- 阶梯价格设置顺序错误
- 阶梯价格重复

**解决方案**：
- 确保阶梯价格按数量从小到大设置
- 确保每个阶梯的数量不重复
- 检查阶梯价格的计算逻辑

### 8.4 产品图片上传失败

**问题**：上传产品图片时失败
**原因**：
- 图片格式不正确
- 图片大小超过限制
- 网络连接问题

**解决方案**：
- 确保图片格式为jpg、png等支持的格式
- 确保图片大小不超过系统限制
- 检查网络连接是否正常

产品管理模块是Wimoor系统的核心功能之一，通过该模块，企业可以有效地管理产品信息，包括基本信息、采购信息、规格信息等。该模块提供了丰富的功能，支持产品的添加、编辑、删除、查询等操作，为企业的采购、库存、销售等业务提供了基础数据支持。

前端使用Vue 3和Element Plus构建了美观、易用的用户界面，后端使用Spring Boot和MyBatis-Plus构建了高效、可靠的API接口。通过前后端的紧密配合，实现了产品信息的全生命周期管理。

该模块的设计考虑了企业的实际需求，提供了灵活的配置选项和丰富的功能，能够满足不同企业的产品管理需求。
