# FBA仓储费模块帮助文档

## 1. 模块概述

FBA仓储费模块是专为财务人员设计的工具，用于统计和分析亚马逊FBA仓储费和超龄库存附加费。通过本模块，财务人员可以：

- 查看详细的仓储费统计数据
- 分析超龄库存附加费情况
- 导出报表用于财务核算
- 导入亚马逊官方仓储费报告

## 2. 功能说明

### 2.1 仓储费统计

**功能描述**：统计指定月份的FBA仓储费用，包括基本费率、总体积、仓储利用率等详细信息。

**操作步骤**：
1. 选择店铺分组和市场
2. 选择要查询的月份
3. 可选择输入ASIN或平台SKU进行搜索
4. 系统会显示该月份的仓储费详细数据
5. 点击"导出"按钮可将数据导出为Excel文件
6. 点击"导入"按钮可导入亚马逊官方仓储费报告

**数据说明**：
- **平台商品信息**：包含商品名称、SKU、ASIN、币种、月份和收货中心
- **尺寸**：商品的长、宽、高、尺寸等级和重量
- **日均**：日均存储、日均购买和日均移除数量
- **仓储利用率**：商品尺寸分段在当月最后一天的仓储利用率
- **仓储费**：每个ASIN的预计月度仓储费，包括基本费率和总体积
- **仓储费奖励**：适用于此ASIN的月度仓储费奖励金额和类型
- **体积**：商品的体积和单位
- **附加费金额**：适用于此ASIN的仓储利用率附加费金额

### 2.2 超龄库存附加费统计

**功能描述**：统计指定时间段内的超龄库存附加费用，帮助财务人员了解库存积压情况。

**操作步骤**：
1. 选择店铺分组和市场
2. 选择要查询的时间范围
3. 可选择输入ASIN或平台SKU进行搜索
4. 系统会显示该时间段内的超龄库存附加费详细数据
5. 点击"导出"按钮可将数据导出为Excel文件

**数据说明**：
- **平台商品信息**：包含商品名称、SKU和ASIN
- **费用时间**：费用产生的日期
- **数量**：被收取附加费的商品数量
- **扣费**：实际扣除的费用金额
- **每单位体积**：商品的单位体积
- **费率附加费**：适用的费率附加费
- **附加费天数**：商品超龄的天数等级
- **币种**：费用的货币类型

## 3. 操作指南

### 3.1 基本操作流程

1. **进入模块**：从左侧菜单导航到"亚马逊 > 报表 > FBA仓储费"
2. **选择标签页**：根据需要选择"仓储费"或"超龄库存附加费"标签页
3. **设置筛选条件**：
   - 选择店铺分组和市场
   - 选择查询时间范围
   - 输入搜索关键词（可选）
4. **查看数据**：系统会自动加载并显示符合条件的数据
5. **导出报表**：点击"导出"按钮下载Excel格式的报表
6. **导入报告**：在"仓储费"标签页点击"导入"按钮，上传亚马逊官方仓储费报告

### 3.2 常见问题及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 数据为空 | 选择的时间范围内没有仓储费数据 | 检查时间范围是否正确，或等待亚马逊报告生成 |
| 导入失败 | 上传的文件格式不正确 | 确保上传的是亚马逊官方的仓储费报告 |
| 导出无反应 | 网络连接问题或数据量过大 | 检查网络连接，或尝试缩小数据范围 |
| 数据不准确 | 报告数据尚未更新 | 等待亚马逊更新仓储费报告后重新导入 |

## 4. 数据解读

### 4.1 仓储费数据解读

- **仓储利用率**：数值越高表示库存周转越慢，可能需要考虑优化库存管理
- **基本费率**：根据商品尺寸和存储时间计算的基础费用
- **总体积**：现有库存的平均数量减去待移除商品的平均数量，所得的值乘以商品体积
- **仓储费奖励**：亚马逊可能会根据库存表现给予的费用减免

### 4.2 超龄库存附加费数据解读

- **附加费天数**：表示商品在亚马逊仓库中存放的时间长度，超过一定时间会产生附加费
- **费率附加费**：根据商品存放时间和尺寸计算的附加费用率
- **扣费**：实际扣除的超龄库存附加费金额

## 5. 最佳实践

1. **定期查看**：建议每月初查看上月的仓储费数据，及时了解费用变化
2. **对比分析**：将不同月份的仓储费数据进行对比，分析费用变化趋势
3. **库存优化**：根据超龄库存附加费数据，及时处理长期积压的库存
4. **报表归档**：定期导出并归档仓储费报表，用于财务核算和审计
5. **结合其他模块**：将仓储费数据与店铺利润模块结合分析，全面了解运营成本

## 6. 注意事项

1. 本模块数据来源于亚马逊官方仓储费报告，数据更新可能存在延迟
2. 导入报告时，请确保上传的是正确的亚马逊官方报告文件
3. 导出的报表包含敏感财务数据，请妥善保管
4. 如发现数据异常，请联系技术支持进行排查
5. 系统会根据用户权限控制数据访问范围，请确保您有相应的权限查看所需数据
