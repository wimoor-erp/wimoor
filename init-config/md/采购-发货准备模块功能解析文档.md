# 采购-发货准备模块功能解析文档

## 模块概述

采购-发货准备模块位于左侧菜单「ERP」→「采购」→「发货准备」，主要包含两大功能板块：成品包装和批量组装。该模块旨在帮助用户高效管理产品的包装和组装过程，确保产品能够及时、准确地发货。

## 前端实现

### 核心文件结构

1. **主入口文件**：`wimoor666/wimoor-ui/src/views/erp/purchase/consumable/index.vue`
   - 模块的主入口组件，包含两个标签页：成品包装和批量组装。
   - 使用 Vue 3 Composition API 实现组件逻辑。

2. **成品包装组件**：`wimoor666/wimoor-ui/src/views/erp/purchase/consumable/components/consumable_material.vue`
   - 实现成品包装的核心功能，包括仓库选择、时间范围设置、SKU搜索等。
   - 使用 `StorageDetailTable` 组件展示入库明细数据。

3. **批量组装组件**：`wimoor666/wimoor-ui/src/views/erp/purchase/process/index.vue`
   - 实现批量组装的核心功能，包括组装单查询、状态管理、批量操作等。
   - 使用 `GlobalTable` 组件展示组装单列表。

### 技术实现

1. **Vue 3 Composition API**：使用 `setup` 语法糖，结合 `ref`、`reactive` 等响应式 API 实现组件状态管理。

2. **Element Plus**：使用 Element Plus 组件库实现 UI 界面，包括标签页、表格、表单、日期选择器等。

3. **API 调用**：
   - 成品包装：调用 `consumableApi`、`preprocessApi`、`warehouseApi` 等 API 获取数据。
   - 批量组装：调用 `assemblyApi`、`preprocessApi` 等 API 获取数据。

4. **组件通信**：使用 props 和 emit 实现组件间通信，如 `PrePlan` 组件与父组件的通信。

5. **路由导航**：使用 `useRouter` 实现页面跳转，如跳转到组装单详情页、调库单页面等。

### 核心功能

1. **成品包装**：
   - 仓库选择：支持级联选择仓库。
   - 时间范围设置：支持快速选择近7天、近一个月、近3个月。
   - SKU搜索：支持按SKU或订单编号搜索。
   - 发货准备计划：创建和管理发货准备运行单。
   - 包装配货单生成：生成PDF格式的包装配货单。

2. **批量组装**：
   - 组装单查询：支持按仓库、状态、关键词等条件查询。
   - 组装单详情：查看组装单的详细信息。
   - 批量组装：选择多个组装单进行批量组装操作。
   - 调库操作：将组装好的产品调往指定仓库。
   - 组装单下载：下载组装单的详细信息和PDF格式的组装配货单。

## 后端实现

### 核心文件结构

1. **PreprocessFormServiceImpl**：`wimoor666/wimoor-erp/erp-boot/src/main/java/com/wimoor/erp/purchase/service/impl/PreprocessFormServiceImpl.java`
   - 实现发货准备模块的核心业务逻辑，包括创建运行单、管理SKU列表、生成配货单等。

2. **AssemblyFormService**：`wimoor666/wimoor-erp/erp-boot/src/main/java/com/wimoor/erp/purchase/service/IAssemblyFormService.java`
   - 实现组装单的核心业务逻辑，包括组装单的创建、更新、查询等。

3. **相关API接口**：
   - `preprocessApi.js`：前端调用的发货准备相关API。
   - `assemblyApi.js`：前端调用的组装相关API。

### 技术实现

1. **Spring Boot 2.x**：使用 Spring Boot 框架实现后端服务。

2. **MyBatis-Plus**：使用 MyBatis-Plus 实现数据库操作，简化 SQL 编写。

3. **业务逻辑**：
   - 发货准备运行单管理：创建、查询、更新发货准备运行单。
   - SKU管理：根据运行单ID获取SKU列表，添加或移除SKU。
   - 配货单生成：生成PDF格式的包装配货单和组装配货单。
   - 批量组装：处理批量组装操作，更新组装单状态。

4. **数据库设计**：
   - `preprocess_form`：存储发货准备运行单信息。
   - `purchase_form_receive`：存储采购入库信息。
   - `assembly_form`：存储组装单信息。
   - `material_consumable`：存储耗材信息。

### 核心功能

1. **发货准备运行单管理**：
   - `getRun` 方法：创建或获取发货准备运行单，区分成品和组合产品。
   - `editRecItem` 方法：编辑运行单中的SKU项目。
   - `clearRunsForm` 方法：清空运行单中的项目。

2. **SKU管理**：
   - `getSkuListByRunid` 方法：根据运行单ID获取SKU列表。
   - `getItemsFormConsumable` 方法：获取运行单中的项目信息。

3. **配货单生成**：
   - `setPDFConsumableFormDetail` 方法：生成包装配货单PDF。
   - `setPDFAssemblyFormDetail` 方法：生成组装配货单PDF。

4. **批量组装**：
   - `doneAssemblyForm` 方法：完成组装单的组装操作。
   - `updateForm` 方法：更新运行单的状态。

## 核心流程

### 成品包装流程

1. **进入模块**：用户进入「ERP」→「采购」→「发货准备」→「成品包装」标签页。

2. **选择仓库**：用户选择要操作的仓库。

3. **设置时间范围**：用户设置查询的时间范围。

4. **搜索SKU**：用户输入SKU或订单编号进行搜索。

5. **创建发货准备计划**：
   - 系统调用 `getRun` 方法创建发货准备运行单。
   - 用户选择需要包装的SKU，点击「添加到发货准备」按钮。
   - 系统调用 `editRecItem` 方法将SKU添加到运行单。

6. **生成包装配货单**：
   - 用户选择已添加到发货准备的SKU，点击「生成包装配货单」按钮。
   - 系统调用 `setPDFConsumableFormDetail` 方法生成PDF格式的包装配货单。

### 批量组装流程

1. **进入模块**：用户进入「ERP」→「采购」→「发货准备」→「批量组装」标签页。

2. **查询组装单**：
   - 用户选择仓库、组装单状态、搜索关键词等条件进行查询。
   - 系统调用 `listForm` 方法获取组装单列表。

3. **查看组装单详情**：
   - 用户点击组装单列表中的「详情」按钮。
   - 系统跳转到组装单详情页，展示组装单的详细信息。

4. **批量组装**：
   - 用户选择多个状态为「处理中」的组装单，点击「批量组装」按钮。
   - 系统调用 `doneAssemblyForm` 方法完成组装操作，更新组装单状态为「已完成」。

5. **调库操作**：
   - 用户选择已完成的组装单，点击「调库」按钮。
   - 系统跳转到调库单页面，用户可以将组装好的产品调往指定仓库。
