# 财务-账期末结算模块功能解析文档

## 1. 模块概述

财务-账期末结算模块（回款记录）是财务管理系统中用于查询、管理和分析亚马逊平台结算记录的核心功能模块。该模块采用前后端分离架构，前端使用Vue 3 + Element Plus实现，后端基于Spring Boot + MyBatis-Plus构建，提供了全面的结算记录查询、分析和管理功能。

### 1.1 核心功能

- **结算状态筛选**：支持筛选已结算和未结算的记录
- **多维度查询**：支持按店铺、国家、结算日期、转账日期等多维度筛选
- **币种切换**：支持在原币种、CNY、USD之间切换查看
- **数据汇总**：提供未结算金额、已结算金额、总金额等汇总数据
- **账期汇总**：支持校验账期汇总和强制重算功能
- **明细下载**：支持下载单个账期的明细和导出所有账期数据
- **账期详情**：查看账期的详细费用构成
- **状态可视化**：清晰展示结算记录的不同状态（未转账、转账失败、转账中、已转账）

### 1.2 技术架构

- **前端**：Vue 3、Composition API、Element Plus、Axios
- **后端**：Spring Boot 2.x、MyBatis-Plus、Spring Security
- **数据库**：MySQL
- **API风格**：RESTful API
- **数据传输**：JSON格式
- **汇率转换**：内置汇率转换服务
- **外部接口**：亚马逊Settlement API

## 2. 前端实现分析

### 2.1 页面结构

结算记录模块的前端实现位于`wimoor-ui/src/views/amazon/payment/record/index.vue`，主要包含以下部分：

1. **查询条件区域**：包含结算状态选择、店铺分组选择、币种选择、日期类型选择和日期范围选择
2. **汇总数据区域**：显示未结算金额、已结算金额、总金额和转账失败金额
3. **结算记录表格**：展示结算记录列表，包含店铺、国家、结算周期、状态、存款总计、转账日期、转账状态、汇总状态和操作
4. **账期详情对话框**：显示账期的详细费用构成

### 2.2 核心组件

#### 2.2.1 主页面组件 (`payment/record/index.vue`)

**核心功能**：
- 结算状态筛选
- 多维度查询条件设置

- 汇总数据展示
- 账期汇总操作

## 3. 后端实现分析

### 3.1 控制器实现

结算记录模块的后端控制器位于`wimoor-amazon/amazon-boot/src/main/java/com/wimoor/amazon/finances/controller/AmzSettlementAccReportController.java`，主要提供以下接口：

#### 3.1.1 获取结算记录列表

**接口**：`/amazon/api/v1/fin/settlementAccReport/getSettlementAccReport`

**功能**：根据筛选条件获取结算记录列表，支持分页。

**参数**：
- `fatype`：结算状态（finsett：已结算，other：未结算）
- `groupid`：店铺分组ID
- `marketplaceid`：市场ID
- `currency`：币种
- `datetype`：日期类型（空：结算日期，acc：转账日期）
- `fromDate`：开始日期
- `endDate`：结束日期
- `page`：页码
- `size`：每页大小

**返回值**：分页的结算记录列表，包含店铺、国家、结算周期、状态、存款总计、转账日期、转账状态、汇总状态等信息。

#### 3.1.2 获取汇总数据

**接口**：`/amazon/api/v1/fin/settlementAccReport/getSettlementAccReportSum`

**功能**：根据筛选条件获取汇总数据。

**参数**：与获取结算记录列表相同。

**返回值**：包含未结算金额、已结算金额、总金额和转账失败金额的汇总数据。

#### 3.1.3 校验账期汇总

**接口**：`/amazon/api/v1/fin/settlementAccReport/checkSettlementSummary`

**功能**：校验账期汇总数据的准确性。

**参数**：与获取结算记录列表相同。

**返回值**：操作结果。

#### 3.1.4 强制重算账期汇总

**接口**：`/amazon/api/v1/fin/settlementAccReport/doSettlementSummary`

**功能**：强制重算账期汇总数据。

**参数**：与获取结算记录列表相同。

**返回值**：操作结果。

#### 3.1.5 下载单个账期明细

**接口**：`/amazon/api/v1/settlement/downSettlementExcel`

**功能**：下载单个账期的明细数据。

**参数**：
- `amazonauthid`：亚马逊授权ID
- `settlementId`：结算ID
- `marketplaceName`：市场名称

**返回值**：Excel文件。

#### 3.1.6 导出所有账期数据

**接口**：`/amazon/api/v1/fin/settlementAccReport/downloadAllSettlementExcel`

**功能**：导出所有符合条件的账期数据。

**参数**：与获取结算记录列表相同。

**返回值**：Excel文件。

#### 3.1.7 获取账期详情

**接口**：`/amazon/api/v1/fin/settlementAccReport/getSettmentDetail`

**功能**：获取账期的详细费用构成。

**参数**：
- `settlement_id`：结算ID
- `currency`：币种
- `amazonAuthId`：亚马逊授权ID

**返回值**：包含销售汇总和详细费用构成的账期详情数据。

### 3.2 服务层

#### 3.2.1 结算报告服务 (`IAmzSettlementAccReportService`)

**核心方法**：
- `getSettlementAccReport`：获取结算记录列表

- `downSettlementExcel`：下载单个账期明细

#### 3.2.2 结算记录服务 (`IAmzSettlementReportService`)

**核心方法**：

- `downLoadSettlementRpt`：下载结算报告

### 3.3 数据访问层

#### 3.3.1 结算报告Mapper (`AmzSettlementAccReportMapper`)

**核心方法**：

## 4. 业务逻辑流程

### 4.1 结算记录查询流程

1. **前端发起请求**：用户设置筛选条件，点击搜索按钮
2. **参数处理**：前端处理筛选参数，确保参数完整性
3. **API调用**：前端调用`getSettlementAccReport`API发送POST请求
4. **后端处理**：
   - 控制器接收请求，处理日期参数
   - 调用服务层方法获取结算记录列表
   - 服务层调用Mapper层查询数据库
   - 服务层处理汇率转换
   - 控制器返回分页结果
5. **前端处理**：
   - 接收响应数据
   - 格式化日期和金额
   - 更新表格数据
   - 同时调用`getSettlementAccReportSum`API获取汇总数据
   - 更新汇总卡片

### 4.2 账期汇总流程

1. **前端发起请求**：用户点击"汇总"按钮，选择"校验账期汇总"或"强制重算"
2. **确认操作**：前端弹出确认对话框（仅导出时）
3. **API调用**：前端调用相应的API发送POST请求
4. **后端处理**：
   - 控制器接收请求，处理参数
   - 调用服务层方法执行汇总操作
   - 服务层调用Mapper层更新数据
   - 控制器返回操作结果
5. **前端处理**：
   - 接收响应数据
   - 显示操作结果提示
   - 刷新结算记录列表

### 4.3 明细下载流程

1. **前端发起请求**：用户点击"下载明细"按钮
2. **API调用**：前端调用`downSettlementExcel`API发送POST请求
3. **后端处理**：
   - 控制器接收请求，处理参数
   - 调用服务层方法生成Excel文件
   - 服务层调用Mapper层查询数据
   - 服务层生成Excel文件
   - 控制器返回Excel文件
4. **前端处理**：
   - 接收Excel文件
   - 触发文件下载

### 4.4 账期详情查看流程

1. **前端发起请求**：用户点击"查看详情"按钮
2. **API调用**：前端调用`getSettmentDetail`API发送POST请求
3. **后端处理**：
   - 控制器接收请求，处理参数
   - 调用服务层方法获取账期详情
   - 服务层调用Mapper层查询数据
   - 服务层处理数据格式
   - 控制器返回账期详情数据
4. **前端处理**：
   - 接收响应数据
   - 弹出详情对话框
   - 展示销售汇总和详细费用构成
