# 结算记录模块功能解析文档

## 1. 模块概述

结算记录模块是财务管理系统中用于查询、管理和分析亚马逊平台结算记录的核心功能模块。该模块采用前后端分离架构，前端使用Vue 3 + Element Plus实现，后端基于Spring Boot + MyBatis-Plus构建，提供了全面的结算记录查询、分析和管理功能。

### 1.1 核心功能

- **结算状态筛选**：支持筛选已结算和未结算的记录
- **多维度查询**：支持按店铺、国家、结算日期、转账日期等多维度筛选
- **币种切换**：支持在原币种、CNY、USD之间切换查看
- **数据汇总**：提供未结算金额、已结算金额、总金额等汇总数据
- **账期汇总**：支持校验账期汇总和强制重算功能
- **明细下载**：支持下载单个账期的明细和导出所有账期数据
- **状态可视化**：清晰展示结算记录的不同状态（未转账、转账失败、转账中、已转账）

### 1.2 技术架构

- **前端**：Vue 3、Composition API、Element Plus、Axios
- **后端**：Spring Boot 2.x、MyBatis-Plus、Spring Security
- **数据库**：MySQL
- **API风格**：RESTful API
- **数据传输**：JSON格式
- **汇率转换**：内置汇率转换服务

## 2. 前端实现分析

### 2.1 页面结构

结算记录模块的前端实现位于`wimoor-ui/src/views/amazon/payment/record/index.vue`，主要包含以下部分：

1. **筛选区域**：包含结算状态选择器、店铺选择器、国家选择器、币种选择器、日期类型选择器和日期选择器
2. **操作区域**：包含汇总下拉菜单（校验账期汇总、强制重算）和导出所有账期按钮
3. **汇总数据区域**：包含未结算金额、已结算金额、总金额、转账失败金额等汇总卡片
4. **数据展示区域**：使用GlobalTable组件展示结算记录列表
5. **详情组件**：集成Detail组件，用于查看结算记录详情

### 2.2 核心组件

#### 筛选组件

- **结算状态选择器**：使用`el-radio-group`组件，支持选择已结算和未结算
- **店铺选择器**：使用自定义的Group组件，支持选择店铺
- **国家选择器**：集成在Group组件中，支持选择国家
- **币种选择器**：使用`el-select`组件，支持选择原币种、CNY、USD
- **日期类型选择器**：使用`el-select`组件，支持选择结算日期和转账日期

public void initAmount(String shopid, Map<String, Object> item, String currency, String tocurrency) {
