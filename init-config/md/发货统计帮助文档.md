# 发货统计模块帮助文档

## 1. 模块概述

c

### 1.1 主要功能

- **多维度分组汇总**：支持按物流承运商、FBA仓库、本地仓库、店铺、站点、SKU、货件等维度进行数据分组
- **饼图可视化**：直观展示各维度的数据分布情况，支持切换不同的汇总指标
- **丰富的统计指标**：包含计划发货、实际发货、实际接收、接收差值、待发货、待接收、货值、重量、费用、时效等
- **灵活的筛选功能**：支持按店铺、仓库、日期类型、日期范围、承运商、物流渠道、SKU等条件筛选
- **数据导出**：支持导出Excel格式的统计数据

### 1.2 适用场景

- 分析不同承运商的物流表现
- 统计各店铺/站点的发货量
- 监控物流费用和时效
- 分析SKU级别的发货和接收差异
- 制作发货数据报表

## 2. 界面说明

### 2.1 页面布局

页面采用标准的报表布局，包含以下主要区域：

1. **分组条件区**：通过复选框选择数据分组维度
2. **筛选条件区**：包含店铺、仓库、日期类型、日期范围、承运商、物流渠道、SKU搜索等
3. **图表展示区**：饼图展示各维度的数据分布
4. **数据表格区**：展示详细的统计数据
5. **操作按钮区**：包含导出、自定义列、帮助文档等按钮

### 2.2 分组条件说明

| 分组维度 | 说明 | 适用场景 |
|---------|------|---------|
| 物流承运商(汇总) | 按物流承运商分组 | 分析不同承运商的物流表现 |
| FBA仓库(汇总) | 按亚马逊FBA仓库分组 | 分析不同FBA仓库的接收情况 |
| 本地仓库(汇总) | 按本地发货仓库分组 | 分析不同本地仓库的发货量 |
| 店铺(汇总) | 按亚马逊店铺分组 | 分析不同店铺的发货表现 |
| 站点(汇总) | 按亚马逊站点分组 | 分析不同站点的发货情况 |
| SKU(汇总) | 按产品SKU分组 | 分析单个产品的发货和接收差异 |
| 货件(汇总) | 按货件分组 | 分析单个货件的详细情况 |

### 2.3 统计指标说明

#### 2.3.1 发货信息指标

| 指标名称 | 说明 | 计算方式 |
|---------|------|---------|
| 计划发货 | 计划发货的总数量 | SUM(产品数量) |
| 实际发货 | 已发货出库的发货数量 | 已发货货件上的发货数量 |
| 实际接收 | 亚马逊已接收的产品数量 | 包括正在接收和接收完成的货件 |
| 接收差值 | 实际发货与接收的差值 | 实际发货 - 实际接收 |
| 待发货 | 尚未发货的数量 | 计划发货 - 实际发货 |
| 待接收 | 尚未被亚马逊接收的数量 | 实际发货 - 实际接收 |
| 实际发货货值 | 已发货产品的货值 | SUM(产品数量 × 产品单价) |

#### 2.3.2 运输信息指标

| 指标名称 | 说明 | 计算方式 |
|---------|------|---------|
| 预估运输重量(KG) | 预估的运输重量 | SUM(产品重量 × 发货数量) |
| 发货运输重量(KG) | 货件计费重量（KG） | 物流系统记录的计费重量 |
| 发货运输重量(CBM) | 货件计费体积（CBM） | 物流系统记录的计费体积 |
| 货件箱数 | 货件包含的箱子数量 | COUNT(箱子数量) |
| 运输费用 | 货件运费 | SUM(运输费用) |
| 关税/其他费用 | 货件的税费和其他费用 | SUM(其他费用) |
| 平均物流时效(天) | 发货到接收的平均天数 | AVG(接收日期 - 发货日期) |

#### 2.3.3 货件信息指标

| 指标名称 | 说明 | 计算方式 |
|---------|------|---------|
| 货件票数 | 货件总数 | COUNT(DISTINCT 货件ID) |
| 异常货件票数 | 有问题的货件数量 | COUNT(状态异常的货件) |

## 3. 操作指南

### 3.1 设置数据分组

1. 在页面顶部的分组条件区，找到需要分组的维度复选框
2. 勾选一个或多个分组维度（可同时选择多个）
3. 系统会自动根据选定的分组维度重新计算和展示数据
4. 建议：
   - 分析承运商表现：选择"物流承运商(汇总)"
   - 分析店铺表现：选择"店铺(汇总)"
   - 分析SKU表现：选择"SKU(汇总)"
   - 综合分析：选择多个维度组合

### 3.2 设置筛选条件

#### 3.2.1 选择店铺

1. 点击"店铺"下拉选择框
2. 选择要查看的店铺
3. 系统会自动加载该店铺的统计数据

#### 3.2.2 选择仓库

1. 点击"仓库"下拉选择框
2. 选择要查看的仓库
3. 默认为"全部"，显示所有仓库的数据

#### 3.2.3 选择日期类型和范围

1. 点击"日期类型"下拉选择框，选择"创建日期"或"发货日期"
2. 点击日期选择器，选择开始日期和结束日期
3. 系统会自动筛选指定日期范围内的数据

#### 3.2.4 选择承运商

1. 点击"承运商"下拉选择框
2. 选择特定的承运商
3. 系统会显示该承运商的数据

#### 3.2.5 选择物流渠道

1. 选择承运商后，"物流渠道"下拉框会被激活
2. 选择特定的物流渠道
3. 系统会显示该渠道的数据

#### 3.2.6 搜索SKU

1. 在SKU搜索框中输入SKU编码
2. 点击搜索按钮或按回车键
3. 系统会显示包含该SKU的数据

### 3.3 查看饼图

1. 在"汇总指标"下拉框中选择要展示的指标
2. 系统会显示对应指标的饼图分布
3. 饼图会按不同维度（如承运商、仓库等）展示数据占比
4. 鼠标悬停在饼图上可以查看具体数值

### 3.4 查看数据表格

1. 数据表格会根据选定的分组维度动态显示列
2. 点击列标题可以按该列进行排序
3. 表格底部会显示合计行
4. 使用分页控件浏览更多数据

### 3.5 导出数据

1. 点击"导出"按钮
2. 系统会生成Excel文件并自动下载
3. 导出文件包含当前筛选条件下的所有数据

## 4. 常见问题

### 4.1 为什么看不到数据？

可能的原因：
- 选择的日期范围内没有发货数据
- 筛选条件过于严格
- 没有相应的权限

解决方法：
- 扩大日期范围
- 检查并调整筛选条件
- 确认是否有相应店铺/仓库的权限

### 4.2 饼图不显示？

可能的原因：
- 没有选择汇总指标
- 数据为空

解决方法：
- 在"汇总指标"下拉框中选择一个指标
- 检查筛选条件是否有数据

### 4.3 数据与预期不符？

可能的原因：
- 日期类型选择错误
- 筛选条件不正确
- 数据同步延迟

解决方法：
- 确认日期类型是"创建日期"还是"发货日期"
- 检查筛选条件设置
- 等待数据同步完成后再查看

### 4.4 如何查看特定承运商的数据？

1. 在"承运商"下拉框中选择该承运商
2. 勾选"物流承运商(汇总)"分组维度
3. 系统会显示该承运商及其各渠道的数据

### 4.5 如何分析SKU级别的数据？

1. 勾选"SKU(汇总)"分组维度
2. 选择日期范围
3. 系统会显示每个SKU的发货和接收数据
4. 可以看到每个SKU的接收差值

### 4.6 导出文件打不开？

可能的原因：
- 文件损坏
- Excel版本不兼容

解决方法：
- 重新导出
- 使用较新版本的Excel打开

## 5. 最佳实践

### 5.1 日常监控

1. **定期查看**：建议每周查看发货统计报表
2. **关注异常**：重点关注接收差值较大的数据
3. **时效监控**：监控平均物流时效的变化趋势

### 5.2 数据分析

1. **承运商对比**：选择"物流承运商(汇总)"分组，对比不同承运商的费用和时效
2. **成本分析**：选择"运输费用"作为汇总指标，分析物流成本构成
3. **店铺分析**：选择"店铺(汇总)"分组，分析各店铺的发货表现
4. **SKU优化**：选择"SKU(汇总)"分组，找出接收差值较大的SKU进行分析

### 5.3 报表制作

1. **定期导出**：定期导出数据保存
2. **多维度组合**：根据需要选择不同的分组维度组合
3. **筛选保存**：设置好筛选条件后导出，确保数据完整性

## 6. 注意事项

### 6.1 数据同步

- 货件数据通过亚马逊API同步，可能存在延迟
- 建议在发货后24小时再查看统计数据
- 接收数据可能需要更长时间才能同步完成

### 6.2 性能考虑

- 避免选择过大的日期范围
- SKU分组会显著增加数据量
- 大量数据导出可能需要较长时间

### 6.3 权限控制

- 用户只能查看有权限的店铺和仓库数据
- 权限由系统管理员设置
