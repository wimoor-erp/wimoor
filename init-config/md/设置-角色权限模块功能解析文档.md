# 设置-角色权限模块功能解析文档

## 1. 模块架构

### 1.1 前端架构

角色权限模块采用Vue 3 Composition API开发，结合Element Plus UI组件库，实现了模块化、响应式的权限管理界面。模块位于`wimoor-ui/src/views/sys/ruleAuth`目录下，主要包含以下文件：

| 文件路径 | 功能描述 |
|---------|--------|
| `index.vue` | 主角色权限管理组件，包含整体布局和组件集成 |
| `components/role_list.vue` | 角色列表组件，实现角色的CRUD操作 |
| `components/menu_list.vue` | 菜单列表组件，实现菜单权限的分配 |
| `components/page_list.vue` | 页面权限组件，实现功能权限和字段权限的分配 |

### 1.2 后端架构

后端采用Spring Boot框架实现，提供RESTful API接口。相关代码位于`wimoor-admin`模块中，主要包含以下部分：

| 模块 | 功能描述 |
|---------|--------|
| `RoleController.java` | 角色管理的RESTful API接口 |
| `ISysRoleService.java` | 角色管理的业务逻辑接口 |
| `SysRoleServiceImpl.java` | 角色管理的业务逻辑实现 |
| `ISysRoleMenuService.java` | 角色菜单权限的业务逻辑接口 |
| `ISysRolePermissionService.java` | 角色功能权限的业务逻辑接口 |
| `SysRole.java` | 角色实体类 |
| `SysRoleMenu.java` | 角色菜单关联实体类 |
| `SysRolePermission.java` | 角色权限关联实体类 |

**核心代码**：
```java
@ApiOperation(value = "列表分页")
@GetMapping("/page")
public Result<List<SysRole>> pageList(Integer page, Integer limit, String name) {
    UserInfo user = UserInfoContext.get();
    LambdaQueryWrapper<SysRole> queryWrapper = new LambdaQueryWrapper<SysRole>()
            .like(StrUtil.isNotBlank(name), SysRole::getName, name)
            .eq(SysRole::getShopid, user.getCompanyid())
            .ne(SysRole::getType, "admin")
            .orderByAsc(SysRole::getName);
    Page<SysRole> result = iSysRoleService.page(new Page<>(page, limit), queryWrapper);
    return Result.success(result.getRecords(), result.getTotal());
}
```

**后端实现**：
- 在`RoleController.java`中，通过`add`方法处理添加角色请求

- 设置角色类型和所属公司

**核心代码**：
```java
@ApiOperation(value = "新增角色")
@PostMapping
public Result<?> add(@RequestBody SysRole role) {
    UserInfo user = UserInfoContext.get();
    long count = iSysRoleService.count(new LambdaQueryWrapper<SysRole>()
                .eq(SysRole::getName, role.getName())
                .eq(SysRole::getShopid, user.getCompanyid())
    );
    Assert.isTrue(count == 0, "角色名称重复，请检查！");
    role.setShopid(new BigInteger(user.getCompanyid()));
    role.setType("actor");
    role.setIssystem(false);
    boolean result = iSysRoleService.save(role);
    if (result) {
        iSysPermissionService.refreshPermRolesRules();
    }
    return Result.success(role);
}
```

#### 2.1.3 删除角色

**前端实现**：
- 在`role_list.vue`中，通过`handleDelete`函数处理角色删除
- 使用Element Plus的`ElMessageBox`组件显示确认对话框
- 调用`deleteRoles`接口删除角色

**核心代码**：
```javascript
function handleDelete(row){

    ElMessageBox.confirm(`确认删除已选中的数据项?`, '警告', {
      confirmButtonText: '确定',
      cancelButtonText: '取消',

          })
          .catch(() => {

          });
      })
      .catch(() => ElMessage.info('已取消删除'));
}
```

**后端实现**：
- 在`RoleController.java`中，通过`delete`方法处理角色删除请求
- 检查角色是否已被用户使用
- 删除角色及其关联的菜单权限和功能权限

**核心代码**：
```java
@ApiOperation(value = "删除角色")
@DeleteMapping("/{ids}")
public Result<?> delete(@PathVariable String ids) {
    List<String> list = Arrays.asList(ids.split(","));
    List<BigInteger> idsList=new LinkedList<BigInteger>();
    for(String id:list) {
        long count=iSysUserRoleService.count(new LambdaQueryWrapper<SysUserRole>().eq(SysUserRole::getRoleId, id));
        if(count>0) {
            throw new BizException("该角色已被用户使用，无法删除。请先将用户上面对应角色权限去掉");
        }
        if(StrUtil.isBlankOrUndefined(ids)) {
            throw new BizException("该角色已被用户使用，无法删除。请先将用户上面对应角色权限去掉");
        }
        idsList.add(new BigInteger(id));
    }
    boolean result = iSysRoleService.delete(idsList);
    if (result) {
        iSysPermissionService.refreshPermRolesRules();
    }
    return Result.judge(result);
}
```

```java
@ApiOperation(value = "修改角色菜单")
@PutMapping(value = "/{id}/menus")
public Result<?> updateRoleMenu(
        @PathVariable("id") BigInteger roleId,
        @RequestBody SysRole role) {

    List<BigInteger> menuIds = role.getMenuIds();
    boolean result = iSysRoleMenuService.update(roleId, menuIds);
    if (result) {
        iSysPermissionService.refreshPermRolesRules();
        menuService.cleanCache();

    return Result.judge(result);
}
```

### 2.3 功能权限管理

#### 2.3.1 功能权限分配

**前端实现**：
- 在`page_list.vue`中，通过`loadData`函数加载角色的功能权限
- 使用Element Plus的`el-checkbox-group`组件显示功能权限
- 通过`permChange`函数处理权限变更

**核心代码**：
```javascript
function loadData(role,menu){
    title.value="-"+menu.name;
    state.loading=true;
    state.formData.roleId=role.id;
    state.formData.menuId=menu.id;
    
    listPerms({'menuId':menu.id}).then(res=>{
        let perms=res.data;
        getRoleResources(role.id,{'menuId':menu.id}).then(res=>{
            let selectids=res.data;
            state.formData.permissionIds=selectids;
            state.formData.permissionNames=[];
            if(selectids&&perms){
                perms.forEach(item=>{
                    if(selectids.includes(item.id)){
                        item.hasPerm=true;
                        state.formData.permissionNames.push(item.name);
                    }else{
                        item.hasPerm=false;
                    }
                    if(item&&item.btnPerm.includes("field")){
                        item.isfield=true;
                    }else{
                        item.isfield=false;
                    }
                })
            }
            state.permList=perms;
            state.loading=false;
        })
    })
}
function permChange(perms){
    state.formData.permissionIds=[];
    state.permList.forEach(item=>{
        if(state.formData.permissionNames.includes(item.name)){
            state.formData.permissionIds.push(item.id);
        }
    });
    updateRoleResource(state.formData.roleId,state.formData).then(res=>{
        ElMessage.success('修改成功');
    })
}
```

**后端实现**：
- 在`RoleController.java`中，通过`updateRolePermission`方法处理功能权限更新
- 调用`ISysRolePermissionService`的`update`方法更新角色权限关联

**核心代码**：
```java
@ApiOperation(value = "修改角色权限")
@PutMapping(value = "/{id}/permissions")
```

## 4. 技术要点

### 4.1 前端技术

1. **Vue 3 Composition API**：使用`ref`、`reactive`、`toRefs`等组合式API管理状态
2. **Element Plus**：使用`el-tree`、`el-checkbox-group`、`el-dropdown`等组件构建界面
3. **组件通信**：使用`emit`和`ref`实现组件间通信
4. **响应式设计**：通过Vue的响应式系统实现数据与视图的双向绑定
5. **表单验证**：使用Element Plus的表单验证功能确保数据有效性

### 4.2 后端技术

1. **Spring Boot**：构建RESTful API接口
2. **MyBatis-Plus**：简化数据库操作，提供分页查询功能
3. **权限缓存**：使用`refreshPermRolesRules`刷新权限缓存
4. **事务管理**：确保权限操作的原子性
5. **数据校验**：使用`Assert`和`BizException`进行数据校验

## 5. 安全考虑

1. **权限控制**：基于角色的权限控制（RBAC）模型，确保用户只能访问授权的资源
2. **权限继承**：菜单权限支持层级继承，简化权限管理
3. **权限缓存**：权限信息缓存，提高权限验证效率
4. **数据校验**：前端和后端双重数据校验，确保数据安全性
5. **防误操作**：删除角色前检查是否已被用户使用，防止误操作

## 6. 性能优化

1. **前端优化**：
   - 使用Vue 3的Composition API减少模板代码
   - 合理使用组件拆分，提高代码复用性
   - 权限变更时自动保存，减少用户操作步骤

2. **后端优化**：
   - 使用MyBatis-Plus的分页查询，提高大数据量场景下的性能
   - 权限信息缓存，减少数据库查询
   - 批量操作支持，减少API调用次数
