# 设置-部门管理模块帮助文档

## 功能概述

部门管理模块位于左侧菜单「设置」→「系统」→「部门管理」，主要用于管理企业内部的部门结构，支持部门的增删改查操作，以及部门层级关系的维护。

## 操作指南

### 进入模块

1. 点击左侧菜单「设置」→「系统」→「部门管理」，进入部门管理页面。

### 查看部门列表

1. 部门管理页面默认显示所有正常状态的部门，以树形表格的形式展示部门层级结构。
2. 可以通过搜索框输入部门名称进行搜索。
3. 可以通过状态下拉框筛选不同状态的部门（正常/禁用）。
4. 点击「重置」按钮可以清空搜索条件，恢复默认显示。

### 新增部门

1. **新增顶级部门**：点击页面顶部的「新增」按钮，在弹出的对话框中选择「顶级部门」作为上级部门，填写部门名称、显示排序和部门状态，然后点击「确定」按钮。

2. **新增子部门**：在部门列表中，找到要添加子部门的父部门，点击该部门操作列中的「新增」按钮，在弹出的对话框中会自动设置该部门为上级部门，填写部门名称、显示排序和部门状态，然后点击「确定」按钮。

### 修改部门

1. 在部门列表中，找到要修改的部门，点击该部门操作列中的「编辑」按钮。
2. 在弹出的对话框中修改部门信息（上级部门、部门名称、显示排序、部门状态）。
3. 点击「确定」按钮保存修改。

### 删除部门

1. **删除单个部门**：在部门列表中，找到要删除的部门，点击该部门操作列中的「删除」按钮，在弹出的确认对话框中点击「确定」按钮。

2. **批量删除部门**：在部门列表中，勾选多个要删除的部门，点击页面顶部的「删除」按钮，在弹出的确认对话框中点击「确定」按钮。

> 注意：删除部门时，会同时删除该部门的所有子部门。

## 技术实现

### 前端实现

#### 核心文件

- **主组件**：`wimoor666/wimoor-ui/src/views/sys/dept/index.vue`
  - 实现部门管理的主界面，包括部门列表展示、搜索、新增、修改、删除等功能。
  - 使用 Element Plus 组件库实现 UI 界面，包括表格、表单、对话框等。
  - 使用 Vue 3 Composition API 实现组件逻辑。

- **API 文件**：`wimoor666/wimoor-ui/src/api/sys/admin/dept.js`
  - 封装部门管理相关的 API 调用，包括获取部门列表、新增部门、修改部门、删除部门等。

#### 核心功能

1. **部门列表展示**：使用 Element Plus 的树形表格组件展示部门层级结构。
2. **部门搜索**：支持按部门名称和状态进行搜索。
3. **部门新增**：支持新增顶级部门和子部门。
4. **部门修改**：支持修改部门的基本信息和上级部门。
5. **部门删除**：支持单个删除和批量删除部门。
6. **部门下拉选择**：使用树形选择组件选择上级部门。

### 后端实现

#### 核心文件

- **控制器**：`wimoor666/wimoor-admin/admin-boot/src/main/java/com/wimoor/admin/controller/DeptController.java`
  - 实现部门管理的 RESTful API 接口，包括获取部门列表、新增部门、修改部门、删除部门等。

- **服务接口**：`wimoor666/wimoor-admin/admin-boot/src/main/java/com/wimoor/admin/service/ISysDeptService.java`
  - 定义部门管理的业务逻辑接口。

- **服务实现**：`wimoor666/wimoor-admin/admin-boot/src/main/java/com/wimoor/admin/service/impl/SysDeptServiceImpl.java`
  - 实现部门管理的业务逻辑，包括部门列表查询、部门保存、部门删除等。

#### 核心功能

1. **部门列表查询**：
   - `listTable` 方法：获取部门表格层级列表，支持按状态和名称过滤。
   - `listSelect` 方法：获取部门下拉列表，用于选择上级部门。

2. **部门保存**：
   - `saveDept` 方法：保存部门信息，包括新增和修改操作，自动处理部门的树形路径。

3. **部门删除**：
   - `deleteByIds` 方法：删除部门及其子部门，使用树形路径进行级联删除。

4. **部门树形结构处理**：
   - 使用递归方法生成部门的树形结构，支持表格展示和下拉选择。
   - 通过 `treePath` 字段维护部门的层级关系，便于级联操作。

## 常见问题

1. **如何添加子部门？**
   - 在部门列表中，找到要添加子部门的父部门，点击该部门操作列中的「新增」按钮，在弹出的对话框中会自动设置该部门为上级部门。

2. **如何修改部门的上级部门？**
   - 点击部门操作列中的「编辑」按钮，在弹出的对话框中通过树形选择器选择新的上级部门。

3. **删除部门时会删除子部门吗？**
   - 是的，删除部门时会同时删除该部门的所有子部门。

4. **部门的显示排序有什么作用？**
   - 部门的显示排序决定了部门在列表中的显示顺序，数值越小，显示越靠前。

5. **如何禁用部门？**
   - 点击部门操作列中的「编辑」按钮，在弹出的对话框中将部门状态设置为「禁用」。

## 注意事项

1. **部门层级**：部门支持多级嵌套，通过树形结构展示部门层级关系。

2. **部门状态**：部门有「正常」和「禁用」两种状态，禁用的部门不会在下拉选择中显示。

3. **权限控制**：操作部门管理模块需要具有相应的权限，请确保您有足够的权限进行操作。

4. **数据完整性**：删除部门前，请确保该部门下没有关联的用户或其他数据，避免数据丢失。

5. **操作顺序**：建议先创建顶级部门，然后再创建子部门，保持部门结构的清晰性。
