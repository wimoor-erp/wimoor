# 财务-其它费用模块功能解析文档

## 1. 模块架构

### 1.1 技术栈

- **前端**：Vue 3 + Composition API + Element Plus
- **后端**：Spring Boot 2.x + MyBatis-Plus
- **数据库**：MySQL
- **API设计**：RESTful API

### 1.2 模块结构

```
前端：
- wimoor-ui/src/views/amazon/payment/costSharing/index.vue  # 主页面
- wimoor-ui/src/views/amazon/payment/costSharing/components/table.vue  # 费用表格组件
- wimoor-ui/src/views/amazon/payment/costSharing/components/otherFee.vue  # 费用类型管理组件
- wimoor-ui/src/api/amazon/finances/financesItemDataApi.js  # API调用

后端：
- wimoor-amazon/amazon-boot/src/main/java/com/wimoor/amazon/finances/controller/AmzFinUserItemDataController.java  # 控制器
- wimoor-amazon/amazon-boot/src/main/java/com/wimoor/amazon/finances/service/impl/AmzFinUserItemDataServiceImpl.java  # 服务实现
- wimoor-amazon/amazon-boot/src/main/java/com/wimoor/amazon/finances/pojo/entity/AmzFinUserItemData.java  # 实体类
- wimoor-amazon/amazon-boot/src/main/java/com/wimoor/amazon/finances/mapper/AmzFinUserItemDataMapper.java  # Mapper接口
- wimoor-amazon/amazon-boot/src/main/resources/mapper/finances/AmzFinUserItemDataMapper.xml  # SQL映射文件
```

## 5. 功能流程

### 5.1 数据加载流程

1. 用户进入其它费用页面
2. 前端默认加载费用类型列表
3. 用户选择筛选条件（店铺、站点、费用类型、日期、SKU）
4. 前端调用 `handleQuery()` 方法，将筛选参数传递给表格组件
5. 表格组件调用 `loadTableData()` 方法，向后端请求数据
6. 后端 `AmzFinUserItemDataController` 接收请求，设置查询参数
7. 后端调用 `AmzFinUserItemDataService.getFinDataList()` 方法查询数据
8. 服务层调用 Mapper 执行 SQL 查询
9. 数据库返回查询结果
10. 后端将结果封装为分页数据返回给前端
11. 前端接收到数据后更新表格显示

### 5.2 费用添加流程

1. 用户点击「添加费用」按钮，调用 `handleAdd()` 方法
2. 前端打开费用添加对话框
3. 用户填写费用信息（店铺、站点、费用类型、日期、货币、金额等）
4. 点击「确定」按钮，前端调用 `saveFinItemData()` API向后端发送请求
5. 后端 `AmzFinUserItemDataController` 接收请求，验证参数
6. 后端调用 `AmzFinUserItemDataService.saveFinItemData()` 方法保存数据
7. 服务层调用 Mapper 执行插入操作
8. 数据库插入新记录
9. 后端返回成功结果
10. 前端显示成功提示，关闭对话框，重新加载数据

### 5.3 费用导入流程

1. 用户点击「导入分摊费」按钮，调用 `openUpload()` 方法
2. 前端打开导入对话框，提供模板下载功能
3. 用户下载模板，填写费用数据
4. 用户上传填写好的Excel文件
5. 前端调用 `uploadFile()` API向后端发送请求
6. 后端 `AmzFinUserItemDataController` 接收文件，调用 `AmzFinUserItemDataService.loadFile()` 方法处理文件
7. 服务层解析Excel文件，批量插入费用数据
8. 数据库批量插入记录
9. 后端返回成功结果
10. 前端显示成功提示，关闭对话框，重新加载数据

### 5.4 费用导出流程

1. 用户点击「导出」按钮，选择导出格式（每天详细或按月汇总）
2. 前端调用 `downloadExcel()` 方法，向后端发送导出请求
3. 后端 `AmzFinUserItemDataController` 接收请求，设置查询参数
4. 后端调用 `AmzFinUserItemDataService.getFinDataList()` 方法查询数据
5. 后端根据导出格式，调用相应的方法生成Excel文件
6. 后端将Excel文件写入响应流
7. 前端下载Excel文件

## 6. 性能优化

- **分页查询**：使用 MyBatis-Plus 的分页功能，避免一次性加载大量数据
