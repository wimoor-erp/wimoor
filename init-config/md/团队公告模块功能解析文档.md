# 团队公告模块功能解析文档

## 1. 模块架构

### 1.1 前端架构

团队公告模块采用Vue 3 + Element Plus开发，实现了模块化、响应式的公告管理界面。模块位于`wimoor-ui/src/views/sys/notify`目录下，主要包含以下文件：

| 文件路径 | 功能描述 |
|---------|--------|
| `index.vue` | 主通知管理组件，实现公告发布功能 |
| `@/api/sys/admin/notify.js` | 通知相关的API调用接口 |

### 1.2 后端架构

后端采用Spring Boot框架实现，提供RESTful API接口。相关代码位于`wimoor-admin`模块中，主要包含以下部分：

| 模块 | 功能描述 |
|---------|--------|
| `NotifyController.java` | 通知管理的RESTful API接口 |
| `NotifyHandlerServiceImpl.java` | 通知管理的业务逻辑实现 |
| `SystemNotify.java` | 系统通知实体类 |
| `SystemUserNotify.java` | 用户通知关联实体类 |
| `SystemNotifyMapper.java` | 系统通知数据访问接口 |
| `SystemUserNotifyMapper.java` | 用户通知关联数据访问接口 |

## 2. 核心功能实现

#### 2.1.2 后端实现

**后端实现**：
- 在`NotifyController.java`中，通过`addAnnounceAction`方法处理公告发布请求
- 调用`notifyHandlerService.createAnnounce`方法保存公告信息
- 在`NotifyHandlerServiceImpl.java`中，实现公告的创建和保存逻辑

**核心代码**：
```java
@PostMapping("/addAnnounce")
public Result<?> addAnnounceAction(@RequestBody SystemNotify notify) {
    notifyHandlerService.createAnnounce(notify);
    return Result.success();
}

public Integer createAnnounce(SystemNotify notify) {
    notify.setCreatedat(new Date());
    notify.setType(NotifyType.Announce.getValue());
    int result = this.baseMapper.insert(notify);
    return result;
}
```

### 2.2 公告查询

#### 2.2.1 前端实现

**前端实现**：
- 通过`getAnnounce` API获取公告列表
- 支持分页查询和公告阅读状态管理

**核心代码**：
```javascript
function getAnnounce(data){
    return request.post('/admin/api/v1/notify/getAnnounce',data );
}
```

#### 2.2.2 后端实现

**后端实现**：
- 在`NotifyController.java`中，通过`getAnnounceAction`方法处理公告查询请求
- 调用`notifyHandlerService.getUserNotify`方法获取用户公告列表
- 调用`notifyHandlerService.read`方法标记公告为已读

**核心代码**：
```java
@PostMapping("/getAnnounce")
public Result<IPage<Map<String, Object>>> getAnnounceAction(@RequestBody BasePageQuery dto) {
    UserInfo userInfo = UserInfoContext.get();
    IPage<Map<String, Object>> list = notifyHandlerService.getUserNotify(dto.getPage(), userInfo, NotifyType.Announce, null);
    List<String> ids = new LinkedList<String>();
    for (Map<String, Object> map : list.getRecords()) {
        ids.add(map.get("id").toString()) ;
    }
    notifyHandlerService.read(userInfo, ids);
    Integer num = notifyHandlerService.findNoReadNumberByUser(userInfo.getId());
    if(list != null && list.getRecords().size() > 0) {
         list.getRecords().get(0).put("msgcount", num);
    }
    return Result.success(list);
}

public IPage<Map<String, Object>> getUserNotify(Page<?> page, UserInfo user, NotifyType type, String ftype) {
    HashMap<String, Object> map = new HashMap<String, Object>();
    map.put("userid", user.getId());
    map.put("ftype", type.getValue());
    map.put("targettype", ftype);
    return systemUserNotifyMapper.findNotifyByUser(page, map);
}

public void read(UserInfo user, List<String> notifyIDs) {
    for (String notifyid : notifyIDs) {
        if (StrUtil.isNotEmpty(notifyid)) {
            systemUserNotifyMapper.updateRead(notifyid, user.getId());
        }
    }
}
```

### 2.3 公告推送

**后端实现**：
- 在`NotifyHandlerServiceImpl.java`中，通过`pullMessage`方法实现公告的推送
- 查询用户的订阅信息，获取用户的目标通知列表
- 根据用户的订阅记录，推送相关的公告信息

**核心代码**：
```java
public void pullMessage(UserInfo user) {
    Date lasttime = systemUserNotifyMapper.findLasttimeByUser(user.getId());
    List<String> targetlist = getTarget(user);
    Map<String, Object> maps = new HashMap<String, Object>();
    maps.put("lasttime", lasttime);
    maps.put("targetlist", targetlist);
    maps.put("userid", user.getId());
    maps.put("shopid", user.getCompanyid());
    List<SystemNotify> notifylist = this.baseMapper.findAnnounceAndRemaind(maps);
    if (notifylist.size() > 0) {
        for (SystemNotify notify : notifylist) {
            SystemUserNotify userNotify = new SystemUserNotify();
            userNotify.setCreatedat(new Date());
            userNotify.setNotify(notify.getId());
            userNotify.setUserid(user.getId());
            userNotify.setIsread(false);
            systemUserNotifyMapper.insert(userNotify);
        }
    }
}
```

## 3. API接口设计

### 3.1 公告管理API

| API路径 | 方法 | 功能描述 | 请求参数 | 成功返回 |
|---------|------|---------|---------|----------|
| `/admin/api/v1/notify/addAnnounce` | POST | 发布公告 | title, content, action, target | 发布结果 |
| `/admin/api/v1/notify/getAnnounce` | POST | 获取公告列表 | page, size | 公告列表及总数 |
| `/admin/api/v1/notify/findNitofyNums` | GET | 获取未读公告数量 | 无 | 未读公告数量 |
| `/admin/api/v1/notify/updateAllTargets` | GET | 标记所有公告为已读 | 无 | 操作结果 |
| `/admin/api/v1/notify/pullMessage` | GET | 拉取新公告 | 无 | 操作结果 |

### 3.2 通知订阅API

| API路径 | 方法 | 功能描述 | 请求参数 | 成功返回 |
|---------|------|---------|---------|----------|

## 4. 技术要点

### 4.1 前端技术

1. **Vue 3 Composition API**：使用`ref`、`reactive`等组合式API管理状态
2. **Element Plus**：使用表格、表单、对话框等组件构建界面
3. **TinyMCE编辑器**：集成富文本编辑器，支持公告内容的格式化编辑
4. **响应式设计**：通过Vue的响应式系统实现数据与视图的双向绑定
5. **表单验证**：使用Element Plus的表单验证功能确保数据有效性

### 4.2 后端技术

1. **Spring Boot**：构建RESTful API接口
2. **MyBatis-Plus**：简化数据库操作，提供CRUD功能
3. **用户上下文**：使用`UserInfoContext`获取当前用户信息
4. **消息类型管理**：通过`NotifyType`枚举管理不同类型的通知
5. **目标用户管理**：支持按用户类型（全部用户、付费用户、免费用户）分发通知
6. **通知订阅**：实现用户对特定类型通知的订阅管理

## 5. 数据模型

### 5.1 核心实体类

| 实体类 | 描述 | 主要字段 |
|---------|------|----------|
| `SystemNotify` | 系统通知实体 | id, title, content, type, target, action, createdat |
| `SystemUserNotify` | 用户通知关联实体 | id, notify, userid, isread, createdat |
| `SystemSubscription` | 用户订阅实体 | id, userid, target, action, disable, createdat |
| `SystemTarget` | 通知目标实体 | id, name, code, description |

### 5.2 枚举类

| 枚举类 | 描述 | 主要值 |
|---------|------|----------|
| `NotifyType` | 通知类型 | Announce(公告), Remind(提醒), Message(消息) |
| `NotifyTarget` | 通知目标 | amazonAuth(亚马逊授权), system(系统), etc. |

## 6. 安全考虑

1. **权限控制**：公告发布功能仅对系统管理员开放，普通用户只能查看公告
2. **数据验证**：前端和后端双重验证，确保公告内容的合法性
3. **用户隐私**：根据用户类型分发公告，避免信息泄露
4. **通知频率**：对相同内容的通知设置发送频率限制，避免重复通知打扰用户

## 7. 性能优化

1. **前端优化**：
   - 使用Vue 3的Composition API减少模板代码
   - 合理使用组件拆分，提高代码复用性
   - 表格数据分页加载，提高大数据量场景下的性能

2. **后端优化**：
   - 使用MyBatis-Plus的分页查询，减少数据库压力
   - 对通知发送频率进行限制，避免频繁插入重复通知
   - 批量处理通知阅读状态更新，减少数据库操作次数
