# 财务-批次成本模块功能解析文档

## 1. 模块概述

财务-批次成本模块是财务管理系统中用于管理和分析亚马逊FBA发货批次成本的核心功能模块。该模块采用前后端分离架构，前端使用Vue 3 + Element Plus实现，后端基于Spring Boot + MyBatis-Plus构建，提供了全面的批次成本管理功能。

### 1.1 核心功能

- **批次成本查询**：查询FBA发货的批次成本数据，包括采购成本和头程成本
- **成本重置**：重置和调整批次成本
- **接收日期修改**：修改FBA发货的接收日期
- **运费重算**：重新计算货件的运费分摊
- **扣减明细查看**：查看批次成本的扣减明细
- **数据导出**：导出批次成本数据为Excel文件

### 1.2 技术架构

- **前端**：Vue 3、Composition API、Element Plus、Axios
- **后端**：Spring Boot 2.x、MyBatis-Plus、Spring Security
- **数据库**：MySQL
- **API风格**：RESTful API
- **数据传输**：JSON格式

## 2. 前端实现分析

5. **接收日期修改对话框**：用于修改接收日期

### 2.2 核心组件

#### 2.2.1 主页面组件 (`jobCosting/index.vue`)

**核心功能**：
- 批次成本查询条件设置

## 3. 后端实现分析

### 3.1 控制器实现

批次成本模块的后端控制器位于`wimoor-amazon/amazon-boot/src/main/java/com/wimoor/amazon/inbound/controller/ShipInboundItemController.java`，主要提供以下接口：

#### 3.1.1 获取批次成本列表

**接口**：`/amazon/api/v1/shipInboundItem/getInboundItemBatchList`

**功能**：根据筛选条件获取批次成本列表，支持分页。

**参数**：
- `fromDate`：开始日期
- `toDate`：结束日期
- `search`：搜索关键词
- `searchType`：搜索类型（sku/number/shipmentid）
- `status`：状态
- `groupid`：店铺分组ID
- `marketplaceid`：市场ID
- `owner`：产品负责人ID
- `page`：页码
- `size`：每页大小

**返回值**：分页的批次成本列表，包含SKU、发货日期、接收日期、状态、单据号、货件号、数量、成本等信息。

#### 3.1.2 导出批次成本数据

**接口**：`/amazon/api/v1/shipInboundItem/downloadList`

**功能**：导出批次成本数据为Excel文件。

**参数**：与获取批次成本列表相同。

**返回值**：Excel文件。

#### 3.1.3 获取扣减明细

**接口**：`/amazon/api/v1/shipInboundItem/findDetailByShipment`

**功能**：获取批次成本的扣减明细。

**参数**：
- `shipmentid`：货件号
- `sku`：SKU

**返回值**：扣减明细列表，包含订单号、发货数量、售价、日期等信息。

#### 3.1.4 更新批次成本

**接口**：`/amazon/api/v1/shipInboundItem/updateFeeByShipment`

**功能**：更新批次成本。

**参数**：`ShipInboundItem`对象列表，包含成本信息。

**返回值**：操作结果。

#### 3.1.5 更新接收日期

**接口**：`/amazon/api/v1/shipInboundItem/updateReceiveDate`

**功能**：更新FBA发货的接收日期。

**参数**：
- `itemlist`：要更新的项目列表
- `date`：新的接收日期（具体日期模式）
- `day`：增加的天数（增加天数模式）

**返回值**：操作结果。

#### 3.1.6 重算运费

**接口**：`/amazon/api/v1/shipInboundItem/handleResetFee`

**功能**：重新计算货件的运费分摊。

**参数**：
- `shipmentid`：货件号

**返回值**：操作结果。

### 3.2 服务层

#### 3.2.1 批次成本服务 (`IShipInboundItemService`)

**核心方法**：
- `getShipinboundItemBatchCondition`：获取批次成本列表
- `findDetailByShipment`：获取扣减明细
- `updateFeeByShipment`：更新批次成本
- `downloadBatchListList`：导出批次成本数据
- `updateReceiveDate`：更新接收日期
- `saveShipmentsFee`：保存货件运费（重算运费）

**关键逻辑**：
- 运费分摊计算：`saveShipmentsFee`方法根据货件的重量计算每个SKU的运费分摊
- 成本更新：`updateFeeByShipment`方法更新批次成本并同步更新订单费用
- 接收日期更新：`updateReceiveDate`方法更新接收日期

### 3.3 数据访问层

#### 3.3.1 批次成本Mapper (`ShipInboundItemMapper`)

**核心方法**：
- `getShipinboundItemBatchCondition`：查询批次成本列表
- `selectByShipmentid`：根据货件号查询项目
- `summaryShipmentItem`：汇总货件项目
- `findDetailByShipment`：查询扣减明细

## 4. 业务逻辑流程

### 4.1 批次成本查询流程

1. **前端发起请求**：用户设置筛选条件，点击查询按钮
2. **参数处理**：前端处理筛选参数，确保参数完整性
3. **API调用**：前端调用`getInboundItemBatchList`API发送POST请求
4. **后端处理**：
   - 控制器接收请求，处理参数
   - 调用服务层方法获取批次成本列表
   - 服务层调用Mapper层查询数据库
   - 控制器返回分页结果
5. **前端处理**：
   - 接收响应数据
   - 更新表格数据

### 4.2 成本重置流程

1. **前端发起请求**：用户选择记录，点击"重置成本"按钮
2. **API调用**：前端调用`updateFeeByShipment`API发送POST请求
3. **后端处理**：
   - 控制器接收请求，处理参数
   - 调用服务层方法更新批次成本
   - 服务层更新数据库中的成本数据
   - 服务层同步更新订单费用
   - 控制器返回操作结果
4. **前端处理**：
   - 接收响应数据
   - 显示操作结果提示
   - 刷新批次成本列表

### 4.3 接收日期修改流程

1. **前端发起请求**：用户选择记录，点击"修改接收日期"按钮，设置新的接收日期
2. **API调用**：前端调用`updateReceiveDate`API发送POST请求
3. **后端处理**：
   - 控制器接收请求，处理参数
   - 调用服务层方法更新接收日期
   - 服务层更新数据库中的接收日期
   - 控制器返回操作结果
4. **前端处理**：
   - 接收响应数据
   - 显示操作结果提示
   - 刷新批次成本列表

### 4.4 运费重算流程

1. **前端发起请求**：用户点击"重算运费"按钮
2. **API调用**：前端调用`handleResetFee`API发送GET请求
3. **后端处理**：
   - 控制器接收请求，获取货件信息
   - 调用服务层方法重新计算运费
   - 服务层根据货件重量重新计算每个SKU的运费分摊
   - 服务层更新数据库中的运费数据
   - 控制器返回操作结果
4. **前端处理**：
   - 接收响应数据
   - 显示操作结果提示
   - 刷新批次成本列表

### 4.5 扣减明细查看流程

1. **前端发起请求**：用户点击"扣减明细"按钮
2. **API调用**：前端调用`findDetailByShipment`API发送GET请求
3. **后端处理**：
   - 控制器接收请求，处理参数
   - 调用服务层方法获取扣减明细
   - 服务层调用Mapper层查询数据库
   - 控制器返回扣减明细数据
4. **前端处理**：
   - 接收响应数据
   - 弹出明细对话框
   - 展示扣减明细数据

### 4.6 数据导出流程

1. **前端发起请求**：用户点击"导出"按钮
2. **API调用**：前端调用`downloadList`API发送POST请求
3. **后端处理**：
   - 控制器接收请求，处理参数
   - 创建Excel工作簿
   - 调用服务层方法生成Excel数据
   - 服务层调用Mapper层查询数据
   - 服务层将数据写入Excel文件
   - 控制器设置响应头并返回Excel文件
4. **前端处理**：
   - 接收Excel文件
   - 触发文件下载
