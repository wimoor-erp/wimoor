# 团队公告模块帮助文档

## 1. 模块概述

团队公告模块是Wimoor系统中的消息通知管理模块，主要用于系统管理员向用户发布重要通知、系统更新信息和操作指南等内容。该模块支持多种消息类型和目标用户群体的设置，确保信息能够准确传达给相关用户。通过该模块，管理员可以实现系统通知的统一管理和分发，提高信息传递的效率和准确性。

## 2. 功能结构

团队公告模块包含以下主要功能：

| 功能模块 | 子功能 | 描述 |
|---------|--------|------|
| 公告发布 | 发布常规消息 | 发布普通的系统公告 |
| 公告发布 | 发布即时消息 | 发布需要立即通知用户的重要信息 |
| 公告发布 | 选择目标用户 | 选择公告的接收用户群体（全部用户、付费用户、免费用户） |
| 公告管理 | 公告列表查看 | 查看系统中已发布的所有公告 |
| 公告管理 | 公告详情查看 | 查看公告的详细内容 |
| 公告管理 | 未读公告提醒 | 提醒用户有未读的公告 |
| 公告管理 | 公告阅读状态 | 标记公告为已读状态 |

## 3. 操作指南

### 3.1 公告发布

#### 3.1.1 发布公告

1. **操作步骤**：
   - 进入系统管理模块，点击「通知管理」菜单
   - 在通知管理页面，填写公告信息：
     - 消息类型：选择「常规消息」或「即时消息」
     - 用户类型：选择「全部用户」、「付费用户」或「免费用户」
     - 标题：填写公告的标题
     - 内容：使用富文本编辑器填写公告的详细内容，支持格式编辑、图片插入等功能
   - 点击「提交」按钮发布公告
   - 系统提示「提交成功」后，公告将被发布到目标用户群体

2. **注意事项**：
   - 公告标题不能为空
   - 公告内容应清晰明确，避免使用模糊或歧义的表述
   - 对于重要信息，建议选择「即时消息」类型，确保用户能够及时收到通知
   - 应根据公告的适用范围选择合适的用户类型，避免信息打扰无关用户

### 3.2 公告查看

#### 3.2.1 查看系统公告

1. **操作步骤**：
   - 登录系统后，系统会自动提醒有未读的公告
   - 点击系统顶部的通知图标，查看公告列表
   - 点击公告标题，查看公告的详细内容
   - 查看后，公告会自动标记为已读状态

2. **注意事项**：
   - 系统会保存用户的公告阅读状态
   - 已读的公告可以在通知历史中查看
   - 重要公告建议及时查看，以免错过关键信息

### 3.3 公告管理

#### 3.3.1 管理公告列表

1. **操作步骤**：
   - 进入系统管理模块，点击「通知管理」菜单
   - 系统显示已发布的公告列表
   - 可以根据需要查看或编辑公告内容

2. **注意事项**：
   - 只有系统管理员有权限发布和管理公告
   - 公告发布后，系统会自动推送给目标用户群体

## 4. 公告类型说明

| 公告类型 | 描述 | 适用场景 |
|---------|------|----------|
| 常规消息 | 普通的系统公告，用户登录后可查看 | 系统更新、功能说明、操作指南等 |
| 即时消息 | 重要的系统通知，会立即提醒用户 | 紧急维护、重要变更、安全提示等 |

## 5. 目标用户说明

| 用户类型 | 描述 |
|---------|------|
| 全部用户 | 系统中的所有用户 |
| 付费用户 | 已付费的系统用户 |
| 免费用户 | 未付费的系统用户 |

## 6. 常见问题

### 6.1 公告发布失败

**问题**：点击提交按钮后提示提交失败
**解决方案**：
- 检查公告标题是否已填写
- 检查公告内容是否为空
- 检查网络连接是否正常
- 联系系统管理员确认权限是否正确

### 6.2 公告未收到

**问题**：发布公告后，部分用户未收到
**解决方案**：
- 检查目标用户类型是否选择正确
- 确认用户是否已登录系统
- 检查用户的通知设置是否开启
- 尝试重新发布公告

### 6.3 公告内容显示异常

**问题**：公告内容显示不完整或格式异常
**解决方案**：
- 检查公告内容是否包含特殊字符
- 确认图片链接是否有效
- 尝试使用简单的文本格式重新编辑公告

### 6.4 公告无法标记为已读

**问题**：查看公告后，仍然显示为未读状态
**解决方案**：
- 刷新页面后再次查看
- 尝试重新登录系统
- 联系系统管理员处理

## 7. 最佳实践

1. **公告内容规范**：
   - 标题应简洁明了，准确反映公告内容
   - 内容应条理清晰，重点突出
   - 避免使用过于技术性的术语，确保所有用户都能理解

2. **公告发布时机**：
   - 系统更新前发布预告公告
   - 系统更新后发布操作指南
   - 重要变更应提前发布通知，给用户足够的准备时间

3. **公告类型选择**：
   - 常规系统更新使用「常规消息」
   - 紧急维护或重要变更使用「即时消息」

4. **目标用户选择**：
   - 通用信息选择「全部用户」
   - 付费功能相关信息选择「付费用户」
   - 免费用户专属活动选择「免费用户」

5. **公告管理**：
   - 定期清理过期或失效的公告
   - 保持公告内容的准确性和时效性
   - 建立公告发布的审批流程，确保信息的真实性和权威性
